## 서문

이 글에서는 시각적 포지셔닝 모듈의 소프트웨어 아키텍처, ROS SDK 구성 방법, 기본 매핑 및 포지셔닝 작업 프로세스를 소개합니다.

이 위치 결정 모듈의 개략도는 위에 나와 있습니다. 그 중 카메라에 내장된 포지셔닝 프로그램은 UDP를 사용하여 호스트 컴퓨터의 ROS SDK와 상호 작용합니다. 또한 포지셔닝 프로그램에는 포지셔닝 시각화를 실현할 수 있는 구성 가능한 WEB 프런트 엔드가 함께 제공됩니다(모든 주류 브라우저를 사용하여 열 수 있으며 기본 액세스 IP는 192.168.100.201:9998입니다). 프런트 엔드에는 두 가지 주요 기능이 있습니다.

1. 전체 DEMO를 실행하여 매핑 및 위치 지정 프로세스를 완료합니다.

2. ROS SDK와 통신하기 위한 타겟 ID와 PORT를 구성합니다. 마찬가지로 ROS SDK를 시작하기 전에 대상 카메라의 IP와 PORT를 설정해야 합니다.

위의 구성이 완료되면 매핑, 재배치, 지도 다운로드 등 WEB 프런트엔드 인터페이스의 모든 기능을 ROS SDK를 통해 호출할 수 있습니다.

## 컴파일

### 프로그램 실행 환경 구성

1. QT 설치

호스트 컴퓨터에 QT5가 설치되어 있는지 확인하십시오. QT5에는 코어, 네트워크 및 기타 기능 패키지가 필요합니다. Linux 버전에 따라 설치 방법이 다를 수 있습니다. 설치 예는 다음과 같습니다.

```bash
sudo apt install qtbase5-dev qtbase5-dev-tools libqt5network5
```

2. libzip 기능 패키지 설치

설치 튜토리얼 https://blog.csdn.net/iamhuanyu/article/details/129603632

3. ROS 함수 패키지를 컴파일합니다. `/V2-ROS-SDK-service` 경로(`/src`와 동일한 레벨)에서 `catkin_make`를 실행합니다.

### 디버깅 중

1\. ROS 디버깅 방법

새 터미널을 열고 `/V2-ROS-SDK-service` 경로를 입력하고 `source devel/setup.bash`를 실행한 후 디버깅을 시작합니다.

에이. 메시지 구독

예: 현재 점유된 카메라 저장 공간을 가져옵니다.

디버깅 진입 후 터미널에서 `rostopic echo /lx_localization_node/LxCamera_Capacity_Status`를 실행합니다.

다음 메시지를 수신한다는 것은 전체 통신 링크가 정상임을 의미합니다. 동시에 이 메시지는 2hz 주파수의 하트비트 패킷으로도 사용될 수 있습니다.

</Frame>

2\. 서비스 인터페이스 호출

예: 현재 선택된 지도 이름 보기

터미널에서 `rosservice call /lx_localization_node/LxCamera_GetCurMapname_srv "get_cur_mapname: true"`를 실행합니다.

다음 메시지를 받았다는 것은 전체 통신 링크가 정상임을 의미하며, 반환된 메시지는 현재 지도 이름입니다.

## 기본 작업 과정

카메라 전원을 켜고 약 30초 정도 기다린 후 데이터 전달 노드를 시작하여 사용을 시작합니다.

디버깅 터미널에 들어가세요

***1. 이슈 맵 구축 지침***

rosservice 호출 /lx\_localization\_node/LxCamera\_Mapping\_srv "상태: true 맵 이름: 'test\_map'"

이 명령은 매핑 작업을 시작하는 데 사용됩니다. "test\_map"이라는 이름의 맵 생성을 준비합니다. 성공적으로 발행되면 true를 반환합니다. 이때 차를 당겨 지도를 스캔할 수 있습니다.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8a77a9c8-f9df-4c71-a4c9-a4e9d66d6892.avif" alt="기본 작업 절차와 관련된 그림" />
</Frame>

***2. 매핑을 종료하려면 명령을 실행하세요***

rosservice 호출 /lx\_localization\_node/LxCamera\_Mapping\_srv "상태: 거짓 맵 이름: 'test\_map'"

이 명령은 매핑 작업을 종료하는 데 사용됩니다. 전달이 성공하면 true를 반환합니다. 이때, 매핑이 완료될 때까지 기다린 후 반환값을 받을 수 있습니다.

### 자주 묻는 질문(FAQ) 및 솔루션

***1\.ROS*** 프로그램을 컴파일할 수 없습니다.

\-종속 라이브러리 환경을 확인하여 모든 종속 라이브러리가 완전히 설치되었는지 확인하세요.

***2.ROS SDK*** 메시지를 보낸 후 응답이 없습니다.

\-카메라와 기기가 동일한 네트워크 세그먼트에 있는지 확인하거나 카메라 호스트 컴퓨터를 사용하여 카메라를 열어 이미지를 볼 수 있는지 확인하세요.

\-카메라 구성 파일에 문제가 없는지 확인하고, ROS 구성 파일에 문제가 없는지 확인하십시오.

***3.ROS*** 디버그 메시지를 보낼 수 없습니다

\-ROS 메시지에는 엄격한 형식 요구 사항이 있습니다. 디버깅 시 지침 문서의 형식을 완전히 따르고 값만 수정하십시오.

디버깅을 변경합니다.

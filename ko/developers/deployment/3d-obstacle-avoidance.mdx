## 1. 개요

시각적 장애물 회피는 Meyer Micro Vision이 자체 개발한 S 시리즈 카메라와 TOF 원리를 기반으로 하는 RGBD 카메라를 기반으로 합니다. 딥 러닝과 기존 3D 비전 알고리즘을 통합하여 풍부한 장면, 정확하고 실시간, 단일 기계 구성 가능, 다중 기계 협업에 적합하고 어느 정도의 주도권과 예측 가능성을 갖춘 장애물 회피 시스템을 만듭니다.

시스템에는 실시간 동적 감지, 적응형 지상 감지, 감지 대상의 동적 추적, 다중 신호 출력 지원, 다중 모드 전환 지원, 5가지 모드 지원 및 다중 기계 공동 작업 지원 등의 특성이 있습니다.

| 2 | 호스트 컴퓨터 및 API 설정 | 기본 매개변수가 설정되어 있으며 정전의 영향을 받지 않습니다. |

5. IO 통신(IO가 일시 중지되지 않음) 중에는 우선순위 2의 통신 방법을 사용하여 현재 매개변수를 수정할 수 없지만 기본 매개변수와 임시 매개변수는 수정할 수 있습니다. IO 통신이 끊어지면(IO가 일시 중지됨) 마지막 설정 방법에 따라 특정 기본 매개변수 또는 임시 매개변수가 사용됩니다. 마지막으로 485 또는 UDP 설정을 사용하면 임시 매개 변수가 사용됩니다. 마지막으로 API 호출이나 호스트 컴퓨터 설정이 사용되면 기본 매개변수가 사용됩니다. 논리는 다음과 같습니다.

### 7.1 API 호출 방법

API 호출 방법은 C++, C#, JAVA, ROS1, ROS2 및 기타 Windows, Linux, arm 및 기타 환경을 지원할 수 있습니다. Windows 환경의 SDK가 호스트 컴퓨터 소프트웨어에 설치된 후 SDK 파일과 샘플 프로그램 코드는 모두 설치 경로(예: D:\\Program Files\\Lanxin-MRDVS)에 있습니다. Linux 환경 SDK는 Meier Microvision에서 제공하는 관련 기술 지원 또는 공식 웹사이트에서 다운로드할 수 있습니다. S10 카메라의 매개변수를 전환하는 데 이 방법을 사용하지 않는 것이 좋습니다. 매개변수를 전환하려면 UDP 또는 IO를 사용하는 것이 좋습니다.

- 문서 폴더에는 SDK 및 호스트 컴퓨터 사용 설명서가 포함되어 있습니다.

- FirmWare 폴더는 카메라 펌웨어 패키지가 저장된 디렉터리입니다.

- Sample 폴더는 샘플코드 소스코드가 저장되는 디렉터리로, 개발환경에 따라 선택 가능합니다.

- SDK 폴더는 SDK 라이브러리 파일이 저장되는 디렉터리입니다. 개발 환경에 따라 환경변수를 구성할 수 있습니다.

- 장애물 회피 애플리케이션은 Sample 디렉토리에 있는 C/application\_obstacled의 장애물 회피 API 소스 코드 프로그램을 매개변수화할 수 있습니다.

### 7.3 UDP 통신 방법

장애물 회피 기능 유형 필드 의미:

| 번호 | 전송 방향 | 메시지 | 장애물 회피 결과 |
| --- | --- | --- | --- |
| 1 | 카메라->본체 | ACED0410XX000000 | 무장애, XX는 현재 장애물 회피 매개변수를 나타냅니다 |
| 2 | | ACED0410XX000001 | 조기경보감속 |
| 3 | | ACED0410XX000002 | 알람 정지 |
| 4 | 본체->카메라 | ACED040000000000 | UDP 연결 설정 |
| 5 | | ACED041A00000000 | 장애물 회피 매개변수 0 |
| 6 | | ACED041A00000001 | 장애물 회피 매개변수 1 |
| 7 | | ACED041A00000002 | 장애물 회피 매개변수 2 |
| 8 | | ACED041A00000003 | 장애물 회피 매개변수 3 |
| 9 | | ... | (매개변수 19까지 계속) |
| 10 | 본체->카메라 | ACED041B00000000 | 현재 매개변수를 기본 매개변수로 설정 |
| 11 | 카메라->본체 | ACED041B00000001 | 기본 매개변수가 성공적으로 설정되었습니다 |

1. UDP를 통해 연결하는 경우 네트워크 디버깅 도우미를 사용하여 카메라와의 통신 연결을 테스트하고 데이터 수신 및 전송을 위해 16진수 전송 및 수신을 선택합니다. 첫 번째 통신에는 4번 명령을 사용하는 것이 좋습니다. UDP 연결이 설정된 후 장애물 회피 결과가 자동으로 전송됩니다. 전송 빈도는 현재 이미지 프레임 속도와 관련이 있습니다. 매개변수를 수정한 후, 카메라가 바디로 보내는 메시지 메시지를 통해 수정 성공 여부를 확인할 수 있습니다. 메시지 번호 1의 XX가 현재 매개변수입니다.

2. DP에서 설정한 매개변수는 전원이 차단되거나 다른 통신 방법을 사용하여 수정되지 않는 한 카메라를 다시 시작할 때까지 유효합니다. 정전 후에도 계속 적용하려면 메시지 번호 10을 사용하여 기본 매개변수로 저장할 수 있습니다. 기본 매개변수로 저장하는 기능은 UDP 통신 CAN, 485 고유의 기능입니다.

3. 로컬 호스트 주소와 로컬 호스트 포트를 설정하고 클릭하여 엽니다. 원격 호스트는 카메라입니다. 카메라 IP 포트를 192.168.100.12:6688로 입력하세요. 카메라 IP가 변경된 경우 카메라 IP:6688이 됩니다. 현재 매개변수 19를 예로 들어 매개변수 0으로 수정합니다. 클릭하면 5번 데이터가 전송됩니다. 성공적인 설정 표시는 아래와 같이 5번째 바이트가 0x13(파라미터 19)에서 0x00(파라미터 0)으로 변경된 것입니다.

| 3 | | ACED0410XX000002 | 알람 정지 |

| 10 | 카메라->본체 | ACED041B00000001 | 기본 매개변수가 성공적으로 설정되었습니다 |

1. TCP를 통해 연결할 때 네트워크 디버깅 도우미를 사용하여 카메라와의 통신 연결을 테스트하고 데이터 수신 및 전송을 위해 16진수 전송 및 수신을 선택합니다. TCP 연결이 설정된 후 장애물 회피 결과가 자동으로 전송되며 전송 빈도는 현재 이미지 프레임 속도와 관련됩니다. 매개변수를 수정한 후, 카메라가 바디로 보내는 메시지 메시지를 통해 수정 성공 여부를 확인할 수 있습니다. 메시지 번호 1의 XX가 현재 매개변수입니다.

2. TCP에 의해 설정된 매개변수는 전원이 차단되거나 다른 통신 방법을 사용하여 수정되지 않는 한 카메라가 다시 시작될 때까지 유효합니다. 정전 후에도 계속 적용하려면 메시지 번호 10을 사용하여 기본 매개변수로 저장할 수 있습니다. 기본 매개변수로 저장하는 기능은 TCP 통신 고유의 기능으로 CAN이나 485에서는 지원하지 않습니다.

3. 보조자를 통해 카메라를 테스트할 수 있습니다. 프로토콜은 TCP Client, IP는 카메라 IP, 포트는 6688을 선택하세요. TCP 통신을 사용하기 전에 현재 사용 중인 장애물 회피 영역이 호스트 컴퓨터에 설정되어 있는지 확인해야 합니다.

### 7.5 CAN 통신 방식

1. CAN 통신은 S2-MAX 카메라만 지원합니다. 통신 프로토콜은 CanOpen을 사용합니다. CanOpen 프로토콜은 표준 CAN 버스에서 실행되는 애플리케이션 계층 프로토콜입니다. 통신 모드는 산업용 통신 프로토콜에서 일반적으로 사용되는 "마스터-슬레이브" 모드 통신이기도 합니다. 즉, 네트워크에는 마스터 스테이션 + 여러 슬레이브 스테이션이 있습니다. 슬레이브 스테이션은 서로 직접 통신하지 않습니다. 모든 통신은 마스터 스테이션과 슬레이브 스테이션 사이에서 이루어집니다. 마스터 스테이션은 "클라이언트"라고도 하며, 슬레이브 스테이션은 "서버"라고도 합니다.

2. 기본 통신 방식은 CAN 표준 프레임 형식을 사용합니다. 즉, CAN ID는 11비트(0x000~7FF)이고 데이터는 8바이트입니다.

3. CAN ID의 사용 규칙: 11비트 ID를 4비트 기능 코드와 7비트 노드 ID로 나누며, CAN ID는 COB ID라고도 합니다.

## 8. 자주 묻는 질문

### 8.1 검은 물체

TOF 원리로 인해 S 시리즈 카메라는 검정색 팔레트의 반사율에 영향을 받습니다. 검정색 팔레트의 반사율이 5%인 S2max 카메라는 2m 범위 내에서 이미징을 달성하는 데 사용할 수 있습니다. 디버깅 중에 블랙 팔레트의 이미징 효과는 호스트 컴퓨터 소프트웨어의 높은 통합 시간과 낮은 신호 임계값을 통해 최적화될 수도 있습니다. 장애물 회피 장면에 검정색 포크 암이 있는 경우 포크 팁 앞의 작은 부분에 다른 색상의 페인트 표면을 뿌리는 등 장애물 회피를 돕기 위해 포크 팁에 더 높은 반사율을 추가하는 것이 좋습니다.

### 8.2 저지대 물체

지상에 있는 물체는 장애물을 피할 수 없습니다. 장애물 회피 애플리케이션에서는 일반적으로 접지를 외부 매개변수 구성의 기초로 사용합니다. 지상에 낮은 장애물이 있을 때 장애물을 피할 수 없는 경우 다음 해결 방법을 참조할 수 있습니다.

- 호스트 컴퓨터의 포인트 클라우드 이미지에는 장애물이 선택되지 않은 것으로 표시됩니다. 장애물 회피 높이(하단)가 양수 값인지 확인해야 합니다. 양수 값인 경우 일시적으로 0으로 설정하여 장애물 감지 가능 여부를 판단하는 것이 좋습니다. 그래도 감지되지 않으면 카메라 설치 높이를 조정하는 것이 좋습니다.

- 호스트 컴퓨터의 포인트 클라우드 이미지는 장애물 객체를 선택할 수 있지만 장애물 회피 결과 출력은 0입니다. 이러한 현상은 카메라 설치 높이를 잘못 구성하여 발생할 수 있습니다. 사용자가 카메라 설치 높이를 조정하는 것이 좋습니다.

### 8.3 장애물 회피 매개변수가 적용되지 않습니다.

- 매개변수 설정 시 설정 완료를 클릭하지 않았습니다.

- 잘못된 매개변수를 수정하여 설정에 실패했습니다.

- 카메라 연결이 불안정하고 연결이 끊겨서 매개변수 전달이 실패하는 현상이 발생합니다.

### 8.4 설치 구조물 간섭

- 설치 시 카메라 주변에 카메라 시야를 차단하는 구조물이 있어 비정상적인 카메라 데이터가 발생할 수 있습니다.

- 구조 설계 시 카메라 설계가 후방 시야를 방해하여 카메라 데이터 이상 현상이 발생합니다.

- 카메라 미러의 보호필름이 제거되지 않아 카메라 데이터가 비정상적입니다.

- 카메라 미러에 기타 보호유리가 설치되어 있어 데이터 이상 현상이 발생합니다.

### 카메라 연결

1. 카메라는 24V로 전원이 공급되며 카테고리 6e/카테고리 7 네트워크 케이블을 통해 연결됩니다. 카메라를 설치하고 배포한 후 노트북을 통해 창고 위치 감지 호스트 컴퓨터 소프트웨어를 열어 카메라의 외부 매개변수를 보정하고 감지 범위를 설정합니다. 알고리즘은 카메라 내부에서 실시간으로 실행됩니다.

2. 여러 대의 카메라를 배포하는 경우 기가비트 스위치를 통해 연결해야 합니다. CAD에서 계획한 설치 높이에 따라 3D 비전 카메라가 마운팅 브라켓을 통해 고정 위치에 설치되어 카메라가 단단히 고정되도록 합니다. 카메라를 설치한 후 24V 조정 DC 전원 공급 장치에 연결하고 해당 네트워크 케이블을 사용하여 스위치 또는 메인 제어 장치에 연결합니다.

3. 준비가 완료되면 카메라 시야각 내 창고에 물품을 배치하고 소프트웨어 배포를 진행합니다.

### 카메라 네트워크 구성

1. Win10 시스템을 예로 들어 제어판을 두 번 클릭합니다.

2. 제어판에 진입한 후 "**네트워크 상태 및 작업 보기**"를 클릭합니다.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/d0ad7fd5-7f9c-43fb-8c8e-fb45f83fcf98.avif" alt="카메라 네트워크 구성 관련 그림" />
</Frame>

3. 네트워크 및 공유 센터 인터페이스로 들어가서 "**어댑터 설정 변경**"을 클릭하세요.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/4c7f569d-7ce6-4c16-a926-03cfd5e42d13.avif" alt="카메라 네트워크 구성 관련 다이어그램 2" />
</Frame>

4. 네트워크 연결 인터페이스로 들어가서 마우스를 두 번 클릭하여 카메라가 연결된 이더넷을 선택합니다.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8a8ff254-6ffc-4a84-9cbe-19c42f984c93.avif" alt="카메라 네트워크 구성 관련 다이어그램 3" />
</Frame>

5. 이더넷 상태(기가비트 네트워크 포트 사용 권장)를 확인하고 "**속성**" 버튼을 클릭하세요.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/b59f551e-1741-4f24-b4ff-8b77bbf16981.avif" alt="카메라 네트워크 구성 관련 다이어그램 4" />
</Frame>

6. 이더넷 속성 인터페이스로 들어가서 "**인터넷 프로토콜 버전 4(TCP/IPv4)**"를 찾아 두 번 클릭하여 엽니다.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/b166daac-7f5c-45c6-9595-2861ff9f2633.avif" alt="카메라 네트워크 구성 관련 다이어그램 5" />
</Frame>

7. 인터넷 프로토콜 버전 4(TCP/IPv4) 인터페이스로 들어가서 "**다음 IP 주소 사용**"을 선택하여 IP 주소, 서브넷 마스크 및 기본 게이트웨이를 작성합니다.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/1798a3b3-d233-4cd1-90a8-77416cf0160f.avif" alt="카메라 네트워크 구성 관련 다이어그램 6" />

8. IP 주소 설정: 192.168.100.xxx 서브넷 마스크: 255.255.255.0 기본 게이트웨이: 192.168.100.1. 아래와 같이 IP를 설정합니다. 설정이 완료되면 "확인" 버튼을 클릭하세요.

<Frame>

9. 확인 버튼을 클릭하면 자동으로 이전 인터페이스로 돌아갑니다. 다시 "**확인**"을 클릭하여 네트워크 구성을 완료하세요.

### 알고리즘 업그레이드

참고: 알고리즘 펌웨어를 업그레이드하기 전에 카메라 펌웨어가 MDS 버전인지 확인해야 합니다.

### 전역 함수

카메라에서 알고리즘이 활성화되었는지, 정상적인 사용 조건에서 카메라 호스트 컴퓨터를 통해 매개변수를 구성한 후 카메라가 작동하지 않는지 확인하십시오. 다음 상황에서는 자동 새로 고침 기능을 활성화해야 합니다.

(1) 장치 목록에서 해당 카메라의 배경이 노란색으로 표시되면 영상 획득 과정에서 알고리즘이 막히는 것을 나타냅니다.

(2) 장치 목록에서 해당 카메라의 배경이 녹색인데 영상이 업데이트되지 않는 경우에도 영상 획득 과정에서 알고리즘이 차단되었음을 의미합니다. 통신은 정상이지만 시스템은 캐시된 데이터를 계속해서 보냅니다.

## 8. 알고리즘 배포 프로세스

참고: 알고리즘 호스트 컴퓨터 소프트웨어를 연 후 포인트 클라우드 보기로 전환하고 마우스를 이미지 표시 영역으로 이동한 다음 마우스 휠을 밀어 포인트 클라우드 데이터를 표시합니다.

### 8.1 인간 탐지 배포 프로세스

#### 인체 감지 - 외부 매개변수 교정

자세한 내용은 6.3 수평 설치를 위한 외부 매개변수 보정을 참조하십시오.

캘리브레이션이 완료된 후 X축(빨간색 화살표)과 Y축(녹색 화살표)이 지면 포인트 클라우드와 일치하고, Z축(파란색 화살표)이 지면 포인트 클라우드와 수직이며 위쪽을 가리키는지 확인합니다.

1. 인간 감지를 클릭하여 매개변수 설정을 입력하고 주로 전역 회전 감지를 조정하며 다른 매개변수는 기본값으로 둡니다. 회전 감지는 카메라 설치 각도와 관련이 있으며, 카메라의 시계 방향 회전은 0°, 90°, 180°, 270°로 제한됩니다.

2. 전체 구성 설정이 완료되면 [**저장 및 보내기**] 버튼을 클릭하여 카메라에 매개변수를 저장합니다.

#### 인간 감지 - 지역 구성

1. 감지 범위를 설정하고 감지 길이와 너비(xy)를 조정하려면 먼저 위쪽 및 아래쪽 보기로 전환하는 것이 좋습니다. 다음은 각 보기에 대한 단축키입니다. ![사람 감지 - 지역 구성과 관련된 그림](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/0c7d1df9-b6a4-43d8-9495-0068463e3a55.avif)

2. [**감지 영역 추가**] 버튼을 클릭하세요. 감지 영역 구성에 감지 영역 수와 감지 영역의 ID 이름이 추가됩니다. ![123](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/d97865c8-086f-46d1-b41f-c3a63ed994ff.avif)

3. 생성된 감지 영역의 최대 XYZ 축 크기를 수정하면 감지 영역의 크기를 직접 확인할 수 있습니다. 아래 그림과 같이 1000\*1000\*2000의 감지 영역이 생성됩니다(단위 mm). ![인체 감지 영역 생성](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/a1d33db3-1c36-4cb6-8235-7ecbb302fe0e.avif)

#### 저장 위치 감지 - 전역 설정

1. 위치 감지를 클릭하면 위치 감지 매개변수 설정 페이지로 전환됩니다. 글로벌 구성에서는 정렬 각도, 정렬 효과, 최소 포인트 수에 주의해야 합니다.

|  |  |
| --- | --- |

| 개체 없음 | 0 |
| 개체가 있습니다 | 1 |
| 물체의 각도가 정렬 각도를 초과함(정렬 효과 켜기) | 2 |

1. 창고 내 중소형 물품 식별 시 식별 우려가 있거나 오판의 위험이 있는 경우 "최소 포인트 수량" 매개변수를 사용하여 심사하는 것이 좋습니다. 이 매개변수는 객체를 인식하는 데 필요한 최소 점 구름 수 임계값을 지정합니다. 물체가 감지되었으나 해당 포인트 클라우드 수가 이 임계값에 도달하지 못한 경우 해당 물체는 인식 결과에서 제외됩니다.

2. 전체 구성 설정이 완료되면 [**저장 및 보내기**] 버튼을 클릭하여 카메라에 매개변수를 저장합니다.

### modbus-tcp(라이브러리 위치 감지)

탐지 결과는 위치 ID에 따라 자동으로 정렬되며 처음 16개의 위치 결과만 반환됩니다.

#### 슬레이브 구성 매개변수

##### 기본 통신 구성

1. \****포트 번호***\*: 16502

2. \****수신 주소***\*: 모든 네트워크 인터페이스(0.0.0.0)

3. \****레지스터 수***\*: 16

4. \****시작 주소 등록***\*: 0x100

5. \****바이트 순서***\*: Big-Endian 모드(Big-Endian)

##### 저장 규칙 등록

|  |  |  |
| --- | --- | --- |
| 주소등록**** | 저장 내용**** | 데이터 형식**** |
| *0x100 + ID* | 위치 감지 결과 + 마커 감지 결과 | *2바이트(uint16_t)* |
| *분할 규칙* | | |
| *바이트 0(상위 8비트)* | 위치 상태 코드 | *0-2(아래 정의 참조)* |
| *바이트 1(하위 8비트)* | 마커 상태 코드(아직 열리지 않았으며 내용이 위치 상태 코드와 일치함) | *0-2(아래 정의 참조)* |

## 1. 개요

LxCameraViewer는 Blue Core가 자체 개발한 카메라 제품과 카메라 SDK C 또는 C++ 개발 키트를 기반으로 개발된 Windows 호스트 컴퓨터 소프트웨어(이하 이 소프트웨어)입니다.

이 소프트웨어는 카메라 사용자에게 보기, 카메라 설정, 알고리즘 설정, IP 구성, 포인트 클라우드 보기, 패킷 캡처 도구, 통신 도구, 방화벽 구성 및 기타 기능과 같은 간단하고 빠른 기능을 제공합니다. 현재 이 소프트웨어는 여러 장치의 열기 및 보기를 지원하고 깊이 이미지, 강도 이미지, 포인트 클라우드 이미지 및 RGB 이미지의 출력 보기를 지원합니다. 또한 기본 카메라 기능, 알고리즘 기본 구성 등 매개변수 읽기 및 설정도 지원합니다.

이 애플리케이션을 통해 사용자는 카메라에서 얻은 고정밀 이미지 데이터, 카메라 구성, 알고리즘 세부 데이터 등을 확인하고 저장할 수 있습니다.

### 2.2 IP 주소 및 게이트웨이 구성

- 연결방식은 고정 IP 주소 연결입니다. IP를 수정해야 하는 경우 이 소프트웨어를 통해 수정할 수 있습니다.

- 카메라의 기본 IP 주소는 192.168.100.82입니다.

- 기본 게이트웨이는 비어 있을 수 있습니다.

- 서브넷 마스크는 Linux 플랫폼에서 255.255.0.0, 255.255.255.0입니다.

- 호스트 컴퓨터의 카메라에 연결된 이더넷의 IP 주소를 192.168.100.x로 설정하세요. x는 3에서 254 사이의 숫자를 나타냅니다. 호스트 컴퓨터 IP와 카메라 IP는 동일할 수 없습니다.

- 예를 들어 카메라 IP가 192.168.100.12인 경우, 아래 그림과 같이 호스트 컴퓨터 IP 주소는 192.168.100.89, 서브넷 마스크는 255.255.0.0으로 설정됩니다.

### 2.3 호스트 컴퓨터 설치

- 두 번 클릭하여 호스트 컴퓨터 설치 패키지 Lanxin-MRDVS-x.x.x.xxxx.exe를 엽니다.

- 앱이 기기를 변경할 수 있도록 허용하려면 클릭하세요.

- 설치 중에 사용할 언어를 선택합니다(그림 1).

- 설치 경로를 선택하세요. 기본 설치 경로를 사용하는 것이 좋습니다. (그림 2)

- [바탕화면 바로가기 생성]을 체크하세요. (그림 3)

- 설치가 완료되면 [Run Lanxin-MRDVS] 체크를 해제하고 Finish를 클릭합니다(그림 4).

- 호스트 컴퓨터를 처음 열 때 관리자 권한을 사용해야 열 수 있습니다. 데스크톱 소프트웨어 아이콘을 마우스 오른쪽 버튼으로 클릭하고 [관리자 권한으로 실행]을 클릭합니다(그림 5).

## 3. LxCameraViewer 사용 지침

### 3.1 소프트웨어 인터페이스 개요

#### 3.1.1 소프트웨어 인터페이스 개요(카메라를 열지 않은 상태)

이 소프트웨어의 초기 인터페이스는 3가지 주요 섹션으로 구분됩니다.

- 목록 표시줄(1번 파란색 영역):

- 장치 목록

- 현재정보

- 기기 정보

- 메뉴 표시줄(2번 녹색 영역):

- 세 개의 오른쪽 도구 모음 버튼

- 인터페이스 언어 전환 버튼

- 이미지 표시줄(빨간색 4번 영역):

- 포인트 클라우드 표시

- 깊이 지도 표시

- 강도 그래프 표시

- RGB 이미지 디스플레이

- 메뉴 표시줄 영역을 두 번 클릭하여 최대화/최소화하고, 마우스 왼쪽 버튼을 누른 채 마우스를 움직여 인터페이스를 드래그합니다.

#### 3.1.2 소프트웨어 인터페이스 개요(카메라 켜짐)

- 다음 기능을 제공하는 기본 도구:

- 이미지 표시 설정

- 2D 설정

- 3D 설정

- 필터링

- 하드웨어 인터페이스

- 컬렉션 설정

- 기능 설정

- 알고리즘을 적용하여 다음 기능을 제공합니다.

- 작업 모드 설정

- 알고리즘 모드 설정

- 알고리즘 버전 표시

- 알고리즘 세부 설정

- 기타 다음 기능을 제공합니다.

- 로컬 포인트 클라우드 로드

- IP 구성 도구

- 패킷 캡처 도구

- 네트워크 카드 구성 도구

- 커뮤니케이션 도구

- GPU 지원

- 로그 수준

### 3.2 기기 켜기 및 끄기

#### 3.2.1 장치 목록

- 장치 목록은 인터페이스의 왼쪽 상단에 있습니다(그림 1의 빨간색 상자).

- [새로 고침] 버튼은 장치 목록 옆에 있습니다. (그림 2의 빨간색 상자)

- 장치 목록 축소/확장 버튼은 새로 고침 버튼 오른쪽에 있습니다(그림 2의 녹색 상자). 축소 후 그림 3을 참조하세요.

- 장치 목록 표시줄에 세 가지 속성이 표시됩니다.

- 카메라가 켜져 있는지 여부(그림 2의 붉은색 상자에서 장치 1은 켜져 있지만 장치 2는 켜지지 않음)

- 카메라 ID(그림 2의 노란색 상자 내부)

- 카메라 IP를 보려면 카메라 ID 위에 마우스를 올리면 됩니다. (그림 4의 빨간색 상자에 표시된 것은 두 번째 카메라의 IP입니다.)

- 장치 목록에는 검색된 각 카메라에 대한 버튼이 설정되어 있습니다. 카메라를 켜거나 끄려면 [켜기] 또는 [닫기] 텍스트를 클릭합니다(그림 2의 파란색 상자). 버튼을 누르면 카메라의 현재 상태 정보도 표시됩니다(그림 5).

- 장치 목록 표시줄에서 선택한 카메라의 글꼴은 파란색이고, 다른 카메라의 글꼴은 흰색입니다(그림 6에서와 같이 두 번째 카메라는 선택되었지만 열리지 않았고, 첫 번째 카메라는 열렸지만 선택되지 않았습니다)

- 장치 목록에서 여러 장치를 동시에 열 수 있지만(그림 7 참조) 네트워크 대역폭이 충분한지 주의해야 합니다. 카메라 ID를 클릭하면 다른 카메라 보기로 전환됩니다.

### 3.3 신청정보

- 장치 목록은 인터페이스 왼쪽 중앙(그림 1의 빨간색 상자)에 있습니다.

- 장치 목록에는 다음 정보가 표시됩니다.

- 카메라에 대한 모든 정보(그림 2의 빨간색 상자)

- 소프트웨어 작동(그림 2의 녹색 상자)

- 연산 결과(그림 2의 파란색 상자)

- 흐름 획득 결과(그림 4의 루즈 색상 상자)

- 텍스트가 너무 길면 아래의 파란색 스크롤 막대를 드래그하여 전체 텍스트를 볼 수 있습니다. (그림 3과 같이 하단에 추가 파란색 스크롤 막대가 있음)

- 현재 정보는 관련 정보를 최대 25개까지 표시하며, 마우스 휠을 이용해 스크롤할 수 있습니다. 이전 정보를 덮어쓰게 됩니다.

- 현재 정보를 축소하려면 축소 버튼을 클릭합니다(그림 2의 노란색 상자).

- 미러 버전: 기기 내장 미러 버전(카메라를 열면 표시됨)

#### 3.5.2 이미지(흐름) 제어

- [흐름 시작/중지] 버튼: 클릭하면 장치의 흐름을 시작하거나 중지합니다.

- [강도 그래프] 체크박스 : 강도 그래프 표시(켜기) 여부

- [Point Cloud] 체크박스 : 포인트 클라우드 이미지를 표시(켜기)할지 여부

- [RGB] 체크박스 : RGB 이미지 활성화 여부

##### 3.7.2.2 2D 설정

- [RGB 너비] RGB 이미지 픽셀 너비는 장치 모델에 따라 결정되며 설정할 수 없습니다.

- [RGB 높이] RGB 이미지 픽셀 높이, 장치 모델에 따라 결정되며 설정할 수 없음

- [픽셀 비닝] 1\*1 또는 2\*2 또는 N\*N을 눌러 매트릭스의 픽셀 값을 추가하고 2\*2 또는 N\*N으로 나누어 비닝 이미지를 얻습니다(이미지 크기는 수평 및 수직으로 비닝 픽셀 크기로 나눈 원래 크기가 됩니다).

- [왜곡 방지] 내부 매개변수에서 왜곡 매개변수를 호출하여 인터페이스 디스플레이에서 이미지의 왜곡을 제거합니다.

- [왜곡 방지 계수] 왜곡 계수를 조정하여 차폐된 이미지의 왜곡을 강화하거나 약화시킵니다.

- [자동 노출] 카메라를 켜거나 끄면 RGB 노출 값이 자동으로 설정됩니다.

- [노출] 수동 노출, 자동 노출이 꺼진 경우 설정 가능

- [게인] 수동 게인, 자동 노출이 꺼진 경우 설정 가능

##### 3.7.2.3 3D 설정

- [3D 너비] 포인트 클라우드 이미지의 너비는 장치 모델에 따라 결정되며 설정할 수 없습니다.

- [3D 높이] 포인트 클라우드 이미지의 높이는 장치 모델에 따라 결정되며 설정할 수 없습니다.

- [픽셀 비닝] 1\*1 또는 2\*2 또는 N\*N을 눌러 행렬의 픽셀 값을 추가하고 2\*2 또는 N\*N으로 나누어 비닝 이미지를 얻습니다. 이는 흐름을 중지한 후 수정할 수 있습니다. (이미지 크기는 원래 크기가 되고 수평 및 수직으로 비닝 픽셀 크기로 나뉩니다.)

- [RGB 정렬] 깊이 맵, 강도 맵, 포인트 클라우드 및 RGB 이미지를 정렬합니다. 정렬 후 깊이 맵과 강도 맵은 RGB 이미지와 동일한 너비와 높이를 갖게 됩니다. 포인트 클라우드 데이터는 증가하며 흐름을 중지한 후 수정할 수 있습니다.

- [멀티 카메라 모드] 여러 대의 카메라를 동시에 사용할 때 켜면 다른 카메라의 간섭을 제거할 수 있으며, 흐름을 중단한 후 수정할 수 있습니다.

- [다중 기계 알고리즘 감도] 장치에 내장된 매개 변수는 장치 모델에 따라 결정되며 설정할 수 없습니다.

- [왜곡 방지] 내부 매개변수에서 왜곡 매개변수를 호출하여 인터페이스 디스플레이에서 이미지의 왜곡을 제거합니다.

- [왜곡 방지 계수] 왜곡 계수를 조정하여 차폐된 이미지의 왜곡을 강화하거나 약화시킵니다.

- [주파수 모드] 장치에 내장된 매개 변수는 장치 모델에 따라 결정됩니다. (일부 카메라의 주파수는 고정되어 설정할 수 없습니다.)

- [HDR 활성화] HDR 활성화를 켜면 빛과 어둠의 변화가 심한 장면에서 사진 선명도가 향상됩니다.

- [자동 노출] 켜져 있으면 장치가 주변 조명 강도에 따라 노출 매개 변수를 자동으로 조정합니다.

- [대상 강도] 깊이 영상 획득 시 강도 임계값(설정할 수 없음)

- [자동 노출 상한] 자동 노출의 상한을 설정합니다.

- [자동 노출 하한] 자동 노출의 하한을 설정합니다(설정할 수 없음).

- [Gain]은 강도 값을 배수로 증가시킬 수 있습니다. (설정할 수 없습니다)

- [x] ROI 영역 세트의 왼쪽 상단 모서리의 x 좌표로, 흐름이 중지된 후 수정될 수 있습니다.

- [y] ROI 영역의 왼쪽 상단 모서리의 y 좌표를 설정합니다. 이는 흐름이 중지된 후 수정될 수 있습니다.

- [고통분] 고통분 시간(증가할수록 카메라의 거리 측정 범위가 넓어짐)

- [낮은 통합] 낮은 통합 시간(주로 가까운 거리에서 거리 측정을 제어하며 일반적으로 변경할 필요가 없음)

- [x offset] 너비 방향의 오프셋 값으로, 흐름을 중지한 후 수정할 수 있습니다.

- [y 오프셋] 흐름이 중지된 후 수정될 수 있는 높은 방향의 오프셋 값입니다.

- [너비] 설정된 ROI 영역의 너비로 흐름을 중지한 후 수정할 수 있습니다.

- [높음] ROI 영역 설정 높이로, 흐름을 중지한 후 수정할 수 있습니다.

- [3D 프레임 속도] 장치 프레임 속도 범위 내에서 포인트 클라우드 프레임 속도를 설정합니다.

##### 3.7.2.4 필터 설정

- [최소 깊이] 깊이 맵의 가색상 또는 거리의 하한입니다. 이 값 이하에서는 가색상이 검정색으로 표시되며 강제로 0으로 설정됩니다.

- [최대 깊이] 깊이 맵의 가색상 또는 거리의 상한입니다. 이 값 이상에서는 가색상이 검정색으로 표시되며 강제로 0으로 설정됩니다.

- [낮은 신호 임계값] 강도 값의 하한입니다. 이 값 아래에서는 깊이 맵의 해당 픽셀이 강제로 0으로 설정됩니다(강도 맵에는 효과적이지 않음).

- [높은 신호 임계값] 강도 값의 상한입니다. 이 값보다 높으면 깊이 맵의 해당 픽셀이 강제로 0으로 설정됩니다(강도 맵에는 영향을 주지 않음).

- [소신호 측정] 소신호 감지 레벨을 설정합니다. 레벨이 높을수록 감지하기가 더 쉽습니다.

- [필터 모드] 필터 차폐 수준 설정

- [평활화 수준] 공간 필터링의 평활화 강도를 제어하여 노이즈를 평활화하는 동시에 개체 가장자리 세부 정보를 유지합니다.

- [노이즈 레벨] 노이즈 필터링 임계값을 설정합니다. (값이 클수록 노이즈가 많아집니다.)

- [시간적 수준] 시간적 필터링의 강도를 제어하고 과거 다중 프레임 데이터를 사용하여 깊이 데이터의 연속성을 향상시킵니다.

- 참고: 스무딩 수준, 노이즈 수준 및 시간 도메인 수준을 0으로 설정하면 필터링이 꺼지는 것을 의미합니다.

##### 3.7.2.6 컬렉션 설정

- [프레임 동기화] 이미지 저장 시 각 이미지 캡처 유형의 환경 조건이 일관되도록 제어합니다.

- [트리거 모드] 흐름 모드를 연속, 수동 또는 능동으로 제어합니다.

- [포인트 클라우드 저장 방법]

- 일반(정렬되지 않은 포인트 클라우드): 포인트 클라우드의 개수는 변경되지 않고 그대로 유지됩니다. 너비 = 너비\*높이, 높이 = 1, 그림 3과 같이

- 정렬된 포인트 클라우드: 포인트 클라우드의 수는 그림 4와 같이 변경되지 않고 너비 = TOF 너비, 높이 = TOF 높이로 유지됩니다.

- [포인트 클라우드 저장 형식] 포인트 클라우드 저장을 위한 데이터 형식을 설정합니다.

- BINARY: 바이너리 형식으로 저장

- ASCII : ASCII 코드 형식으로 저장, 즉 텍스트 모드로 저장

- [포인트 클라우드 값 단위] 포인트 클라우드 이미지의 데이터 측정 단위를 설정합니다.

- [사진 저장 시작] 현재 선택한 유형의 사진을 계속 저장합니다.

- [저장 경로 열기] [단일 저장] 기능으로 사진을 저장할 경로를 열어줍니다.

참고: 640\*480 이미지의 경우 ASCII 저장 시간은 약 6.6초이고 바이너리 저장 시간은 약 32ms입니다. 바이너리 저장 속도는 ASCII의 206배입니다. 바이너리 형식으로 저장하는 것이 좋습니다.

##### 3.7.2.7 기능 설정

- [파일로 구성 내보내기] 현재 카메라 매개변수 구성을 json 형식 파일로 패키징하고 내보내기

3.7.3.1 장애물 회피 알고리즘

- 적용 알고리즘을 장애물 회피 알고리즘으로 설정하면 장애물 회피 관련 구성이 표시됩니다.

- [차체 표시] 포인트 클라우드 이미지에 차체의 위치를 ​​표시하는 자동차 모델을 끄거나 켤 수 있습니다.

- [장애물 회피 지수] 장애물 회피는 여러 구성이 가능하며 이 옵션을 통해 설정됩니다.

- [카메라 각도 롤(x축)] 카메라 외부 매개변수, 롤 각도, 기본 카메라는 180입니다.

- [카메라 각도 피치(y축)] 카메라 외부 매개변수, 피치 각도, 기본 수직 설치는 90입니다.

- [카메라 각도 요(z축)] 카메라 외부 매개변수, 방향 각도 요, 기본값은 90입니다.

- [카메라 위치(거리 x)] 카메라 외부 매개변수, x 방향과 원점 사이의 거리, 단위: 미터.

- [카메라 위치(폭 y)] 카메라 외부 파라미터, y 방향과 원점 사이의 거리, 중앙에 설치하면 0 입니다. 단위: 미터.

- [카메라 위치(높이 z)] 카메라의 외부 매개변수인 z 방향과 원점 사이의 거리를 지면에서 카메라까지의 높이(단위: 미터)로 이해할 수 있습니다.

- [조기경보거리(x거리)] 장애물 회피범위, 차량 중심에서 x방향 거리. 이 거리 너머의 장애물은 안전한 것으로 간주됩니다. 조기 경보 거리와 경보 거리 사이의 거리는 조기 경보 감속으로 간주되며 인터페이스는 노란색 영역을 표시합니다. 단위: 미터.

- [경보 거리(x 중심)] 장애물 회피 범위, 차량 중심에서 x 방향의 거리입니다. 이 거리 미만의 장애물은 경보 중지로 간주되며 인터페이스에 빨간색 영역이 표시됩니다. 사전 경고 거리와 경보 거리 사이의 거리는 조기 경고 감속으로 간주되며 인터페이스는 노란색 영역을 표시합니다. 단위: 미터.

- [차폐 거리(x 부근)] 장애물 회피 범위, 차량 중심에서 x 방향의 거리. 이 거리 내의 장애물은 무시됩니다. 단위: 미터.

- [장애물 회피 범위 좌측(y left)] 장애물 회피 범위, y방향 좌측에서 차량 중심까지의 거리. 단위: 미터.

- [장애물 회피 범위 오른쪽(y right)] 장애물 회피 범위, y방향 오른쪽에서 차량 중심까지의 거리. 단위: 미터.

- [장애물 회피 높이 하한(z lower)] 장애물 회피 범위, 지면으로부터의 높이, 이 높이 이하는 처리되지 않습니다. 설정이 너무 작으면 범위 오류로 인해 잘못된 경보가 발생할 수 있습니다. 설정이 너무 크면 지상의 큰 장애물이 필터링되어 잘못된 경보가 발생할 수 있습니다. 단위: 미터.

- [장애물 회피 높이 상한(z 상한)] 장애물 회피 범위, 지면으로부터의 높이, 이 높이보다 높은 것은 처리되지 않습니다. 단위: 미터.

- [장애물 회피 출력] 장애물 회피 결과 출력

- [카메라 외부 매개변수 재설정] 카메라에 기록되지 않은 모든 매개변수를 복원합니다.

- [외부 매개변수 설정] 모든 매개변수를 카메라에 쓰기

- 자세한 장애물 회피 설정은 장애물 회피 문서를 참조하세요.

##### 3.7.3.3 팔레트 위치 지정

- 응용 알고리즘을 팔레트 포지셔닝으로 설정하면 장애물 회피 관련 구성이 표시됩니다.

- [프로젝트 이름] 현재 팔레트 인식 매개변수 구성의 이름을 지정하세요.

- [빠른 구성] 카메라가 기본적으로 제공하는 여러 구성

- [카메라 각도 롤(x축)] X축을 기준으로 한 카메라의 기울기 각도

- [카메라 각도 피치(y축)] Y축을 기준으로 한 카메라의 기울기 각도

- [카메라 각도 요(z축)] Z축을 기준으로 한 카메라의 기울기 각도

- [카메라 위치(거리 x)] 카메라 광학 중심과 지게차 회전 중심 사이의 X 방향 상대 거리

- [카메라 위치(폭 y)] 카메라 광학 중심과 지게차 회전 중심 간의 Y 방향 상대 거리

- [카메라 위치(높이 z)] 카메라 광학 중심과 지게차 회전 중심 사이의 Z 방향 상대 거리

- [지상 높이] 카메라의 광학 중심에서 지면까지의 거리

- [최대 팔레트 다리 너비] 팔레트 유형의 팔레트 다리 최대 너비와 호환되어야합니다.

- [최소 팔레트 다리 폭] 팔레트 타입의 팔레트 다리 최소 폭과 호환되어야 합니다.

- [트레이로부터의 거리] 삽입이 완료된 후 카메라의 광학 중심에서 트레이까지의 거리

- [설치방법] 카메라의 고정방향에 따라 설정하며, 기본값은 정식설치(0)입니다.

- [최소 Z 값] 팔레트의 가장 가까운 깊이 거리를 식별합니다.

- [최대 Z 값] 팔레트의 가장 먼 깊이 거리를 식별합니다.

- [최소 X 값] 팔레트의 측면 경계(카메라를 중심으로 한 왼쪽 경계)를 식별합니다.

- [최대 X 값] 팔레트의 측면 경계(카메라를 중심으로 한 오른쪽 경계)를 식별합니다.

- [팔레트 알고리즘 출력]은 팔레트 알고리즘의 결과를 표시합니다.

- [캘리브레이션 안내] 지면이 평탄하고 파렛트 앞에 장애물이 없을 때 자동으로 캘리브레이션을 진행합니다. 팔레트는 표준 크기입니다.

- [기본 매개변수 복원] 모든 매개변수를 공장 기본값으로 복원합니다.

- [카메라에서 외부 매개변수 읽기] 카메라에서 모든 매개변수를 읽고 전송되지 않은 모든 매개변수를 덮어씁니다.

- [매개변수 보내기] 모든 매개변수를 카메라에 씁니다.

- 팔레트 감지에 대한 자세한 설정은 팔레트 감지 기능 문서를 참고하세요.

## 자주 묻는 질문(FAQ) 및 솔루션

|  |  |
| --- | --- |
| 문제 설명 | 솔루션 |
| 카메라를 찾을 수 없습니다 | * 카메라 전원이 연결되어 있는지 확인하세요. 전원을 켠 후 카메라 전면의 파란색 표시등이 항상 켜져 있습니다. * IP 주소와 서브넷 마스크가 올바른지 확인하세요. * 호스트 컴퓨터의 네트워크 접속이 금지되어 있는지 확인 |
| 카메라를 열 수 있지만 스트리밍이 실패합니다 | 호스트 컴퓨터가 관리자 권한으로 열렸는지 확인하세요 |
| 이미지 지연 | 네트워크 카드, 네트워크 케이블, 스위치가 기가비트인지 확인하세요. RGB 정렬을 켜면 100M 네트워크에서 어느 정도 지연이 발생합니다 |
| 카메라가 자주 다시 연결됩니다 | 공장 설정 복원, 펌웨어 업그레이드 후 또는 100M 네트워크에서 처음으로 카메라를 켠 후 카메라는 약 30초 동안 자체 조정/재시작됩니다. 30초 이내에 호스트 컴퓨터는 카메라가 안정될 때까지 카메라 연결을 계속 시도합니다 |
| 깊이 데이터 없음, 포인트 클라우드 데이터 | * 카메라가 가려졌는지 확인하세요 * 물체가 카메라 촬영 범위 안에 있는지 확인하세요 * 최대/최소 깊이 값이 올바른지 확인하세요 * 필터 매개변수가 올바른지 확인하세요 |
| 포인트 클라우드가 사라지고 인터페이스에는 흰색 프레임만 표시됩니다 | 포인트 클라우드 이미지를 마우스로 클릭하고 'm' 키를 눌러 포인트 클라우드 안정화 |
| 포인트 클라우드 표시가 비정상입니다 | 포인트 클라우드 단축키가 작동되었는지 확인하고 포인트 클라우드 안정성 켜기 |
| Linux에서 카메라를 연 후 포인트 클라우드가 표시되지 않습니다 | 마우스를 사용하여 포인트 클라우드 이미지를 누르고 드래그하여 표시하세요 |
| 응용 프로그램 표시: VTK 로딩 실패, 포인트 클라우드가 표시되지 않습니다. 그래픽 카드 구성 및 그래픽 카드 드라이버를 확인하십시오 | * 그래픽 카드 구성이 너무 낮은지 확인하세요 * 그래픽 카드 드라이버가 정상적으로 실행되고 있는지 확인하세요 * 포인트 클라우드 디스플레이에는 독립된 비디오 디스플레이 메모리와 해당 드라이버가 있는 그래픽 카드가 필요합니다 |

## 인터페이스 설명

빨간색 메뉴 표시줄: 클릭하면 매핑 시작, 재배치, 지도 관리 등과 같은 해당 기능 작업 세부 정보를 입력할 수 있습니다.

파란색 상자: 로드된 2D 등고선 지도와 자동차 위치 표시 아이콘을 사용하여 표시됩니다.

녹색 상자: 탑뷰 카메라의 실시간 RGB 이미지를 표시합니다.

노란색 상자: 재배치 신호 강도의 실시간 표시(녹색: 매우 좋음, 파란색: 좋음, 빨간색: 낮음) 자동차의 현재 위치 좌표, 자동차의 현재 이동 속도, 마우스 위치 좌표를 실시간으로 표시

매핑을 시작하는 페이지입니다

그런 다음 AGV를 움직여 작업 영역의 지도를 스캔합니다. 스캔이 완료된 후 메뉴바를 클릭하면 지도 구축이 종료됩니다. 인내심을 갖고 기다려 주십시오. (대기 시간은 스캔 영역의 크기와 관련이 있습니다.)

매핑이 성공하면 지도 관리 항목에서 매핑 데이터를 확인할 수 있습니다.

증분 매핑

기능: 기존 지도를 기반으로 지도 확장, 사각지대 보완 등 신규 지도 영역 추가

전제조건: 증분 맵을 구축할 때 웹페이지의 재배치가 완료되었는지, 추가해야 할 맵으로 전환되었는지 확인해야 합니다.

(당분간 수동 HTTP 요청이 발행됩니다)

주소 [http://192.168.100.201:9998/tool/rob/sendCMD](http://192.168.100.201:9998/tool/rob/sendCMD "http://192.168.100.201:9998/tool/rob/sendCMD")

매개변수 JSON

```json
{

    "cmd":905, // 904 starts incremental mapping; 905 stops incremental mapping

    "attach":{

        "sub_command":0,

        "param":""

}

}
```

## 2. 지도 관리

![2. 지도관리 관련 그림](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/9b855ff4-d2b8-4a0b-9031-3b1343a848df.avif) 그림 2 지도 관리의 개략도

지도 생성이 완료되면 메뉴바에서 **지도 관리**를 클릭하세요. 호스트 맵 목록에서 방금 생성한 맵의 이름을 확인할 수 있습니다. 적용 버튼을 클릭하면 지도로 전환됩니다. 마찬가지로 전환할 지역 지도가 여러 개인 경우 지도 관리 기능을 사용하여 전환해야 합니다. 해당 지도를 삭제하려면 삭제 버튼을 클릭하세요. **맵 적용:** 적용을 클릭한 후 맵이 적용되고 적용되기 전에 호스트를 다시 시작해야 합니다. 그렇지 않으면 재배치가 불가능합니다.

## 3. 이전

재배치를 클릭한 후 레이저 포인트 클라우드와 재배치를 위한 레이저 등고선 맵의 일치 프롬프트를 기반으로 마우스를 드래그하여 대략적인 재배치 포즈를 제공할 수 있습니다.

</Frame>

그림 4 재배치 기능의 개략도

재배치가 성공하면 **위치 지정 시간 초과** 예외가 사라집니다.

QR 코드 재배치

지도를 작성할 때 재배치를 위해 일부 위치에 Apritag 코드를 붙여넣도록 선택할 수 있습니다. 완전한 QR 코드가 장치 관점에 나타나면 코드가 어느 지도와 위치에 있는지 자동으로 식별하고 자동으로 지도를 전환하고 위치를 변경합니다.

QR 코드 생성 웹사이트 [https://chaitanyantr.github.io/apriltag](https://chaitanyantr.github.io/apriltag "https://chaitanyantr.github.io/apriltag")

## 7. 펌웨어 업데이트

패키지 로컬 보기: 업데이트해야 하는 로컬 펌웨어(섀시, 액추에이터, 카메라 펌웨어)를 가져옵니다.

패키지 복원: 마지막 업데이트 이전의 펌웨어 버전을 기록합니다. 다시 돌아가고 싶다면 클릭하기만 하면 됩니다.

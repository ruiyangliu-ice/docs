## 인터페이스

- TCP 소켓 서버. 카메라는 서버 역할을 합니다. 기본 포트는 `5501`입니다.

- 메시지 내용: ASCII를 지원하는 텍스트 형식입니다. JSON을 권장합니다.

## 요청과 응답

**1. 로봇(PLC) —> 비전 시스템**

```json
{"action": "3D"}
```

이 요청은 3D 비전 작업을 트리거합니다.

**2. 비전 시스템 —> 로봇(PLC)**

카메라는 다음 필드와 함께 감지 결과를 반환합니다.

| 필드 | 필수 | 설명 |
| --- | --- | --- |
| 결과 | 예 | 정수 감지 상태. `0`는 현지화에 실패했으며 결과가 없음을 나타냅니다. 다른 값은 성공을 나타내며 반환되는 결과 수를 지정합니다. |
| 위치 | 예 | 로봇 좌표계에서 감지된 물체에 대한 X, Y, Z, RX, RY 및 RZ 포즈 값입니다. 단위는 밀리미터와 도입니다. `-180° ≤ RZ ≤ 180°`. 감지 실패 시 모든 값은 `0`로 설정됩니다. |

**예**

| 감지 성공 | `{"result":2,"pos":[100,107.3,1530,179,89,89,112,432.1,1500,179,89,75]}` |
| --- | --- |
| 감지 실패 | `{"result":0,"pos":[0,0,0,0,0,0]}` |

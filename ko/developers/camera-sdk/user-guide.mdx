## 1. **개요**

이 문서는 주로 카메라 장치 액세스와 관련된 개발자를 대상으로 합니다(소프트웨어 개발 기능 필요). 현재 M4/M4 Mega/M4 Pro/V1Pro/S2/S2/H3 시리즈 제품을 지원합니다. 이 문서의 주요 목적은 SDK에서 제공하는 API를 사용하는 방법을 설명하는 것입니다. Camera SDK는 x86 기반 Windows 운영 체제, X86 및 Arm 기반 Linux 운영 체제에서의 실행을 지원합니다.

카메라 SDK는 통합 액세스 제어 및 매개변수 구성 인터페이스를 제공하여 장치 데이터(깊이 이미지, 강도 이미지, 포인트 클라우드 이미지, RGB 이미지 포함)의 동기 또는 비동기 획득을 지원합니다. 일부 카메라 모델은 장애물 회피, 도킹, 위치 지정 및 기타 데이터를 지원합니다.

본 문서는 최대한의 개발 지원을 제공하며, 제공된 내용에는 결함이 있거나 불완전할 수 있습니다.

##### 2.2.3.2 Linux 시스템 설치:

1. 설치 스크립트의 압축을 풀고 실행합니다.

- tar xvf Lanxin-MRDVS-xxx.tar.gz

- CD Lanxin-MRDVS

2. 설치에는 루트 권한이 필요하므로 현재 사용자가 루트가 아닌 경우 루트 사용자로 전환해야 합니다.

- sudo su 루트

3. 그런 다음 설치를 수행합니다.

- chmod +x install.sh

- ./install.sh

설치 스크립트는 /opt/Lanxin-MRDVS 디렉터리에 SDK를 설치하고 환경 변수를 설정합니다. 스크립트에 설정된 환경변수는 현재 사용자의 환경변수라는 점에 유의하시기 바랍니다. 실행 중인 프로그램이 루트 사용자가 아닌 경우 install.sh 스크립트를 다시 실행하여 환경 변수를 설정하거나 현재 사용자의 ~/.bashrc 끝에 내보내기 LD\_LIBRARY\_PATH=/opt/Lanxin-MRDVS/lib/:$LD\_LIBRARY\_PATH를 수동으로 추가해야 합니다.

설치 전에 열린 터미널 세션의 경우 프로그램을 실행하기 전에 ~/.bashrc 소스를 실행하여 LD\_LIBRARY\_PATH 환경 변수를 업데이트해야 합니다.

## 4. **카메라 SDK 호출 참고사항**

### 4.1 Camera SDK 인터페이스 호출의 일반 원칙

1. LX\_INT 접두사가 붙은 열거 유형의 경우 DcSetIntValue 또는 DcGetIntValue를 사용하여 INT 유형 매개변수를 설정하거나 가져옵니다.

2. LX\_FLOAT 접두사가 붙은 열거 유형의 경우 DcSetFloatValue 또는 DcGetFloatValue를 사용하여 FLOAT 유형 매개변수를 설정하거나 가져옵니다.

3. LX\_BOOL 접두사가 붙은 열거 유형의 경우 DcSetBoolValue 또는 DcGetBoolValue를 사용하여 BOOL 유형 매개변수를 설정하거나 가져옵니다.

4. LX\_STRING 접두사가 붙은 열거 유형의 경우 DcSetStringValue 또는 DcGetStringValue를 사용하여 문자열 유형 매개변수를 설정하거나 가져옵니다.

5. DcSetCmd를 사용하여 LX\_CMD 접두사가 붙은 열거 유형에 대한 명령을 실행합니다.

6. LX\_PTR 접두사가 붙은 열거 유형의 경우 DcGetPtrValue를 사용하여 포인터 유형 결과를 얻습니다.

### 4.2 Camera SDK 인터페이스 부분 호출 전제조건

**1. 흐름이 시작되거나 작업 모드(LX\_INT\_WORK\_MODE)가 일반적으로 열려 있는 모드인 경우 다음 기능을 설정할 수 없습니다.**

-DcSetRoi

- LX\_INT\_3D\_BINNING\_MODE;

- LX\_INT\_2D\_BINNING\_MODE;

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH;

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR;

- LX\_BOOL\_ENABLE\_MULTI\_MACHINE;

- LX\_INT\_TRIGGER\_MODE;

- LX\_INT\_CALCULATE\_UP;

- LX\_BOOL\_ENABLE\_HDR;

- LX\_BOOL\_ENABLE\_TOF\_DEPTH\_CALIBE;

- LX\_BOOL\_ENABLE\_SYNC\_FRAME

- LX\_INT\_SAVE\_PARAMS\_GROUP

**2. 작업 모드(LX\_INT\_WORK\_MODE)가 정상적으로 열려 있어도 설정이 허용되지 않습니다.**

- LX\_INT\_TOF\_FREQMODE<br />

**3. 3D 자동 노출(LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE)이 켜져 있으면 다음 기능 열거 작업이 허용되지 않습니다.**

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR ;

- LX\_INT\_FIRST\_EXPOSURE;

- LX\_INT\_SECOND\_EXPOSURE;

- LX\_INT\_THIRD\_EXPOSURE;

- LX\_INT\_GAIN;<br />

**4.3D 자동 노출(LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE)이 꺼지면 다음 기능 열거가 작동할 수 없습니다.**

- LX\_INT\_3D\_AUTO\_EXPOSURE\_LEVEL

3D 자동 노출 최대값(LX\_INT\_3D\_AUTO\_EXPOSURE\_MAX) 및 최소값 설정은 허용되지 않습니다.

- (LX\_INT\_3D\_AUTO\_EXPOSURE\_MIN)

**5. 2D 자동 노출(LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE)이 켜져 있으면 다음 기능 열거 작업이 허용되지 않습니다.**

- LX\_INT\_2D\_MANUAL\_EXPOSURE

<br />**6. 2D 자동 노출(LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE)이 꺼진 경우 다음 기능 열거는 작동이 허용되지 않습니다.**

- LX\_INT\_2D\_AUTO\_EXPOSURE\_LEVEL;

<br />7.**특수 필터링 모드에서는 다음 설정이 허용되지 않습니다.**

- LX\_INT\_FILTER\_SMOOTH\_LEVEL LX\_INT\_FILTER\_NOISE\_LEVEL

- LX\_INT\_FILTER\_TIME\_LEVEL

<br />8.**2D 왜곡 방지를 켜면 RGBD 정렬을 먼저 켜야 합니다.**

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH

<br />9.**S2 유형이 아닌 장치의 경우 내장 알고리즘을 켤 때 먼저 tof 알고리즘을 설정해야 합니다. 알고리즘 매개변수는 내장 알고리즘(LX\_INT\_ALGORITHM\_MODE)이 켜진 경우에만 설정할 수 있습니다.**

- (LX\_STRING\_ALGORITHM\_PARAMS)

<br />10.**포인트 클라우드 데이터(LX\_PTR\_XYZ\_DATA)를 얻으려면 먼저 깊이 맵을 열어야 합니다**

- (LX\_BOOL\_ENABLE\_3D\_DEPTH\_STREAM)

| 키워드 | 설명 |

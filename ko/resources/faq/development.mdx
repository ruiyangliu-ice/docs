## FAQ

<AccordionGroup>
<Accordion title="LX_ERROR = -1">

**가능한 이유**

알 수 없는 오류

**해결**

결과가 비정상이므로 처리를 위해 장치 로그를 가져와 MRDVS로 보내야 합니다.

</Accordion>

<Accordion title="LX_E_NOT_SUPPORT = -2">

**가능한 이유**

지원되지 않는 기능

**해결**

현재 장치는 이 기능을 지원하지 않습니다

</Accordion>

<Accordion title="LX_E_NETWORK_ERROR = -3">

**가능한 이유**

네트워크 통신 오류

**해결**

네트워크 통신 장애, 신호 전송 패킷 손실 등의 문제, IP 구성 확인, 네트워크 케이블 및 스위치의 정상 여부 확인, 패킷 손실 문제 해결

</Accordion>

<Accordion title="LX_E_INPUT_ILLEGAL = -4">

**가능한 이유**

인삼을 복용하는 것은 불법이다

**해결**

전달된 매개변수가 잘못되었습니다. 호출 인터페이스의 매개변수를 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_재연결 중 = -5">

**가능한 이유**

장치 재연결 중

**해결**

1. 장치가 다시 연결 중입니다. 잠시 기다렸다가 다시 연결이 성공할 때까지 다시 시도하십시오<br />2. 카메라가 정상적으로 켜져 있는지, 네트워크 연결이 정상적인지 확인하세요.

</Accordion>

<Accordion title="LX_E_DEVICE_ERROR = -6">

**가능한 이유**

장치 오류 또는 장치 응답 오류

**해결**

기기 이상, 프로그램이나 카메라를 다시 시작하고 다시 시도해 보세요.

</Accordion>

<Accordion title="LX_E_DEVICE_NEED_UPDATE = -7">

**가능한 이유**

기기 버전이 너무 낮아 업그레이드가 필요합니다.

**해결**

펌웨어 버전 업그레이드

</Accordion>

<Accordion title="LX_E_API_NEED_UPDATE = -8">

**가능한 이유**

API 버전이 너무 낮습니다.

**해결**

SDK 업데이트

</Accordion>

<Accordion title="LX_E_CTRL_PERMISS_ERROR = -9">

**가능한 이유**

독점 제어 권한이 실패했습니다.

**해결**

카메라는 하나의 기본 권한 클라이언트 연결만 지원합니다. 카메라가 비정상적으로 종료된 경우 권한이 해제될 때까지 3~5초 정도 기다려야 합니다.

</Accordion>

<Accordion title="LX_E_GET_DEVICEINFO_ERROR = -10">

**가능한 이유**

장치 정보를 가져오지 못했습니다.

**해결**

장치 펌웨어 버전을 확인하거나 MRDVS 지원팀에 문의하세요.

</Accordion>

<Accordion title="LX_E_IMAGE_SIZE_ERROR = -11">

**가능한 이유**

이미지 크기가 일치하지 않아 다시 열어야 합니다.

**해결**

획득한 이미지의 크기가 실제 코드 스트림의 크기와 일치하지 않습니다. 장치를 닫았다가 다시 열어야 합니다.

</Accordion>

<Accordion title="LX_E_IMAGE_PARTITION_ERROR = -12">

**가능한 이유**

이미지 구문 분석에 실패했습니다(pixformat이 올바르지 않음). 다시 열어야 합니다.

**해결**

기기를 다시 켜세요

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_CONNECTED = -13">

**가능한 이유**

카메라가 연결되지 않음

**해결**

잘못된 유형의 매개변수 입력입니다. 카메라가 켜지지 않은 상태에서 카메라를 조작하면 이 오류가 반환됩니다. 카메라가 성공적으로 켜졌는지 확인하려면 호출 코드 로직을 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_DEVICE_INIT_FAILED = -14">

**가능한 이유**

카메라 초기화 실패

**해결**

카메라를 다시 켜세요

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_FOUND = -15">

**가능한 이유**

카메라를 찾을 수 없음(또는 일치하는 카메라를 찾을 수 없음)

**해결**

잘못된 입력 매개변수의 일종입니다. 이 오류 코드는 일반적으로 인터페이스에 전달된 카메라 핸들이 불법인 경우 반환됩니다. 호출 코드 논리를 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_FILE_INVALID = -16">

**가능한 이유**

파일 오류(잘못된 파일 이름, 유형, 형식 또는 파일 열기 실패)

**해결**

파일 경로/유형이 틀리거나 파일이 존재하지 않거나 열 수 없는 경우, 들어오는 매개변수 파일 경로를 수동으로 열 수 있는지 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_CRC_CHECK_FAILED = -17">

**가능한 이유**

파일 crc 또는 md5 확인에 실패했습니다.

**해결**

파일을 업로드하거나 다운로드할 때 crc 또는 md5 확인에 실패했습니다. 다시 시도해야 합니다.

</Accordion>

<Accordion title="LX_E_TIME_OUT = -18">

**가능한 이유**

시간 초과

**해결**

다시 시도해 보세요. 네트워크상의 이유로 패킷 손실이 많거나 카메라 프레임 속도가 너무 낮은 경우가 있습니다.

</Accordion>

<Accordion title="LX_E_FRAME_LOSS = -19">

**가능한 이유**

누락된 프레임

**해결**

콜백 모드에서는 패킷 손실 및 프레임 손실이 자주 발생합니다. 소수의 경우 특별한 처리가 필요하지 않습니다. 많은 경우 네트워크 대역폭, 캐시 등을 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_ENABLE_ANYSTREAM_FAILED = -20">

**가능한 이유**

스트림을 열지 못했습니다.

**해결**

버전 2.4.9 이전에는 모든 스트림 데이터를 닫는 것이 허용되지 않습니다. 이 오류 코드는 후속 버전에서 폐지되었습니다.

</Accordion>

<Accordion title="LX_E_NOT_RECEIVE_STREAM = -21">

**가능한 이유**

스트리밍 데이터가 수신되지 않음

**해결**

코드 스트림 데이터가 수신되지 않았고 참조 카메라가 열려 있으며 데이터가 없습니다.

</Accordion>

<Accordion title="LX_E_PARSE_STREAM_FAILED = -22">

**가능한 이유**

스트림이 성공적으로 시작되었지만 스트림 데이터를 구문 분석하지 못했습니다.

**해결**

코드 스트림을 구문 분석하지 못했습니다. 펌웨어 버전을 확인하거나 MRDVS 지원팀에 문의하세요.

</Accordion>

<Accordion title="LX_E_PROCESS_IMAGE_FAILED = -23">

**가능한 이유**

이미지 계산 처리 실패

**해결**

이미지 알고리즘 라이브러리 버전 "Dataprocess" 확인

</Accordion>

<Accordion title="LX_E_SETTING_NOT_ALLOWED = -24">

**가능한 이유**

일반적으로 열린 모드에서는 설정이 허용되지 않습니다.

**해결**

작업 모드, 일반적으로 개방형 모드에서는 ROI 설정 또는 여러 카메라/정렬 설정과 같은 작업이 허용되지 않습니다. 호출 논리를 조정해야 합니다. 먼저 카메라 작동 모드를 설정한 후 설정 작업을 수행하십시오.

</Accordion>

<Accordion title="LX_E_LOAD_DATAPROCESSLIB_ERROR = -25">

**가능한 이유**

이미지 처리 알고리즘 라이브러리를 로드하는 중 오류가 발생했습니다.

**해결**

이미지 알고리즘 라이브러리 Dataprocess(동적 링크 라이브러리/공유 개체 파일)가 존재하고 라이브러리에서 로드된 환경 변수에 포함되어 있는지 확인하세요. 실행: 소스 install.sh

</Accordion>

<Accordion title="LX_E_FUNCTION_CALL_LOGIC_ERROR = -26">

**가능한 이유**

함수 호출 논리 오류

**해결**

함수 호출 순서나 사전 로직이 잘못된 경우 SDK 로그에 따라 오류를 보고하고 호출 로직을 조정해야 합니다. 일반적인 논리 문제는 다음과 같습니다:<br />1. 매개변수를 설정하면 이미지 크기가 변경되므로 먼저 흐름을 중지해야 합니다.<br />2. 정상적으로 켜진 모드에서는 이미지 크기를 변경할 수 없습니다. <br />3. 알고리즘을 위로 이동하는 것은 내장된 애플리케이션 알고리즘 <br />4를 켜는 것과 상호 배타적입니다. HDR이 활성화되면 자동 노출을 켤 수 없습니다. <br />5. 자동 노출 관련 매개변수는 자동 광폭발 상태와 관련됩니다. <br /> />6. 트리거 관련 매개변수는 트리거 모드와 관련이 있습니다.

</Accordion>

<Accordion title="LX_E_IPAPPDR_UNREACHABLE_ERROR = -27">

**가능한 이유**

IP에 연결할 수 없거나 네트워크 구성 오류

**해결**

네트워크를 사용할 수 없습니다. SDK 호스트가 시각적 장치를 ping할 수 있는지 확인해야 합니다.

</Accordion>

<Accordion title="LX_E_FRAME_ID_NOT_MATCH = -28">

**가능한 이유**

시간 초과 범위 내에서 프레임 동기화 오류가 발생함

**해결**

RGBD 프레임 동기화 기능이 켜져 있는 경우 tof 및 rgb 데이터 프레임이 실제 코드 스트림에서 동기화되지 않은 경우 이 오류가 보고됩니다. 실제 사용 시나리오(동기화의 경우 높은 우선순위 또는 데이터 실시간의 경우 높은 우선순위)에 따라 프레임 데이터를 삭제하거나 오류를 무시하도록 선택할 수 있습니다.

</Accordion>

<Accordion title="LX_E_FRAME_MULTI_MACHINE = -29">

**가능한 이유**

프레임에서 다중 기계 간섭 신호가 감지됨

**해결**

실제 사용 시나리오에 따라 프레임 데이터를 삭제하거나 오류 보고서를 무시하도록 선택합니다.

</Accordion>

<Accordion title="오류 코드: -25 또는 LxDataProcess 로드 실패">

**가능한 이유**

(Linux 운영 체제 환경에만 해당) 알고리즘 라이브러리가 환경 변수로 구성되지 않았습니다.

**해결**

소스 install.sh 실행

</Accordion>

<Accordion title="스트리밍된 이미지의 3/3를 차지합니다.">

**가능한 이유**

이미지를 획득한 후 표시된 화면 비율이 비정상적입니다.

</Accordion>

<Accordion title="ros2가 SDK 라이브러리와 충돌합니다.">

**해결**

카메라 SDK는 util 폴더에 캡슐화되어 있으며 카메라 SDK 라이브러리에 간접적으로 연결됩니다.

</Accordion>

<Accordion title="소프트웨어 수준에서 카메라가 Lite 버전인지 RGB가 포함된 표준 버전인지 확인하는 방법">

**해결**

DcGetBoolValue(handle, LX_BOOL_ENABLE_2D_STREAM, &test_rgb) 카메라가 Lite 버전인 경우 지원되지 않음을 반환합니다.

</Accordion>

<Accordion title="시간동기화 방식">

**해결**

SDK와 PTP가 동기화됩니다. 각 인터페이스 호출은 추가 트리거 없이 자동으로 동기화됩니다.

</Accordion>

</AccordionGroup>

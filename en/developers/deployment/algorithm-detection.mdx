### Camera connection

1. The camera is powered by 24V and connected through a Category 6e/Category 7 network cable. After the camera is installed and deployed, open the warehouse location detection host computer software through a laptop to calibrate the external parameters of the camera and set the detection range. The algorithm runs inside the camera in real time.

2. When deploying multiple cameras, they need to be connected through a Gigabit switch. According to the installation height planned in the CAD, the 3D vision camera is installed in a fixed position through the mounting bracket to ensure that the camera is firmly fixed. After the camera is installed, connect it to the 24V regulated DC power supply and use the corresponding network cable to connect it to the switch or main control.

3. After the preparation is completed, place the goods in the warehouse within the camera's field of view and proceed with software deployment.

Each time the state switches, a frame of data is saved, which is saved as a frame of RGB image and point cloud data at the moment of switching. The playback between different algorithms is not interoperable. The time standard is the camera's internal system time. Please do not judge based on the computer system time.

### Global functions

Please ensure that the algorithm has been activated in the camera, and that the camera fails after configuring parameters through the camera host computer under normal use conditions. The auto-refresh feature should be enabled in the following situations.

(1) If the background of the corresponding camera in the device list is displayed in yellow, this indicates that the algorithm encounters a blockage during the image acquisition process.

(2) If the background of the corresponding camera in the device list is green, but the image is not updated, this also means that the algorithm is blocked in the image acquisition process. Although the communication is normal, the system continues to send cached data.

## 7. Meaning of algorithm parameters

### Human body detection

#### Global configuration of human detection

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/7be5cd79-40be-429a-a05b-ca9ccd043fd9.avif" alt="Illustrations related to global configuration of human detection" />

|  |  |
| --- | --- |
| Human detection global configuration | Functional meaning |
| Algorithm version number | Current algorithm version information. |
| Detection frequency | Algorithm detection frequency. That is, how many ms to refresh the image + result data. |
| Time domain filtering | Filter parameter, when the parameter value is set to 3, the system requires that the detection results of three consecutive times are completely consistent before it will be judged as valid data. |
| Kernel Size | Corrosion expands the kernel size on a 2D image. Large kernels will significantly shrink the target or expand the background, while small kernels will retain more details. Change as needed. |
| Distance threshold | Used to determine whether two data points are "similar" or "belong to the same category". If the result is less than the threshold, it is considered a match or similar type. |
| Minimum number of points | The minimum number of point clouds required for semantic clustering. |
| Unsemantic minimum number of points | Unsemantic minimum number of point clouds required for clustering. |
| Domain size | Determines which points are considered "neighboring points", directly affecting the granularity and accuracy of the clustering results. Modify as needed. |
| Rotation detection | According to the installation position of the camera, the front 0° logo is at the bottom, adjust accordingly. |

### modbus-tcp (library location detection)

The detection results are automatically sorted according to the location ID, and only the first 16 location results will be returned.

#### Slave configuration parameters

##### Basic communication configuration

1. \****Port number***\*: 16502

2. \****Listening Address***\*: All network interfaces (0.0.0.0)

3. \****Number of registers***\*: 16

4. \****Register starting address***\*: 0x100

5. \****Byte order***\*: Big-Endian mode (Big-Endian)

##### Register storage rules

|  |  |  |
| --- | --- | --- |
| Register address**** | Storage content**** | Data format**** |
| *0x100 + ID* | Location detection result + marker detection result | *2 bytes (uint16_t)* |
| *Split Rules* | | |
| *Byte 0 (high 8 bits)* | Location status code | *0-2 (see definition below)* |
| *Byte 1 (lower 8 bits)* | Marker status code (not yet open, the content is consistent with the location status code) | *0-2 (see definition below)* |

#### Status code definition

##### Location status code

|  |  |  |
| --- | --- | --- |
| Encoding | Physical meaning | Trigger conditions |
| *0* | Unoccupied | *No goods/pallets* |
| *1* | Normal occupancy | *Standard goods are placed correctly* |
| *2* | Abnormal occupancy | *Cargo size/position offset* |

##### Marker status code

|  |  |  |
| --- | --- | --- |
| Encoding**** | Physical meaning**** | Trigger condition**** |
| *0* | No marker detected | |
| *1* | Marker detected | |
| *2* | Detect anomalies | |

PalletPro configures pallet-recognition and docking parameters for M Series cameras. The camera runs the algorithm on the device; PalletPro connects to the camera, checks imaging, calibrates extrinsics, adjusts recognition regions, and reads results.

<Info>
  The standard pallet dimensions, distances, and parameter ranges in this guide apply only to the standard recognition workflow. Evaluate non-standard, legless, or highly reflective pallets on site first.
</Info>

## Prepare for deployment

### Standard pallets and installation requirements

This workflow uses a standard two-leg or three-leg pallet. The original workflow uses `1.2 m × 0.8 m` or `1.2 m × 1.0 m` pallets. Confirm non-standard dimensions with MRDVS before deployment.

| Item | Requirement |
| --- | --- |
| Camera position | Keep the camera as close as possible to the center between the two forks; left-to-right offset should not exceed `50 mm`. |
| Camera attitude | Keep the camera face perpendicular to the ground with minimal pitch and roll. |
| Field of view | At the nearest recognition distance, the forks must not obscure the complete pallet. |
| Follow-up installation | Mount the camera on the fork-root panel so it moves with the forks. |
| Drop-down installation | Mount the camera below the forks so it moves with the forks at full drop. |

For fork lengths up to `1.2 m`, an empirical mounting height is `0.35–0.4 m`. For longer forks, use `0.4–0.8 m` as a starting range. Confirm the final height from the field of view, pallet-leg imaging, and docking distance.

### Equipment and software

- Power the camera with `24 VDC` and connect it to the computer with a Cat7 Ethernet cable.
- Recommended host: Windows 10 or later, Intel® Core™ i5 (7th generation) or later, `8 GB` RAM, and Intel® UHD Graphics or a higher-performance GPU.
- When installing PalletPro, accept the C++ runtime prompt. A missing runtime can prevent the application from starting.

<Warning>
  Use only the camera firmware and pallet algorithm package delivered for the project. The camera restarts during an upgrade and temporarily disappears from the device list. Back up verified calibration before upgrading.
</Warning>

## Quick deployment

<Steps>
  <Step title="Configure the network and open the camera">
    Put the computer on the `192.168.100.xxx / 255.255.255.0` subnet. Start PalletPro, discover, and open the target camera.
  </Step>
  <Step title="Verify versions and imaging">
    In LxCameraViewer, verify that the application, camera firmware, and pallet algorithm versions match the project delivery. Confirm that pallet legs are fully visible in the point cloud.
  </Step>
  <Step title="Set the initial height">
    In **Parameter settings**, use **Height adaptation** to obtain an initial value. Fine-tune it against the ground line, then select **Save parameters**.
  </Step>
  <Step title="Run near-end teaching">
    Use a standard pallet, keep it centered with no nearby obstructions, and select **Pallet teaching (near end)** when the pallet is about `1.3 m` from the camera.
  </Step>
  <Step title="Run far-end teaching">
    Keep the pallet in the same lateral position. Reverse the vehicle in a straight line by about `600–1000 mm` until the pallet is about `2.0–2.4 m` from the camera, then select **Pallet teaching (far end)**.
  </Step>
  <Step title="Save and verify">
    Save the parameters and test the near and far positions repeatedly. Check X, Y, angle output, and recognition stability.
  </Step>
</Steps>

<Tip>
  Far-end teaching calibrates camera pitch. Do not move laterally or turn between near-end and far-end teaching.
</Tip>

## Use PalletPro

### Main interface

| Area | Purpose |
| --- | --- |
| Basic operations | Extrinsic calibration, parameter saving, and camera operations. |
| Advanced applications | Advanced parameters, 3D display, and offline tests. |
| Camera settings | Discover, open or close the camera, display live data, detect, and project. |
| Parameter settings | Camera name, installation height, attitude, and installation method. |
| File import | Import calibration files, a single image, or an image folder for offline tests. |
| Logs and images | View runtime logs, versions, and live recognition results. |

### Extrinsic calibration

In **Basic operations → Extrinsic calibration**, read **Calibration notes** before teaching. Use the project name and vehicle ID for the customer name. The distance from the camera optical center to the forklift rotation center is required.

| Calibration step | Result |
| --- | --- |
| Near-end teaching | Calibrates mounting height, lateral offset, heading, and roll. |
| Far-end teaching | Calibrates pitch from the near-end result. |
| Save parameters | Sends the current calibration to the camera. |

### Advanced parameters

Change only fields required by the project and select **Save parameters** after every change.

| Parameter | Purpose |
| --- | --- |
| `R` | Camera attitude: pitch, heading, and roll. |
| `t` | Camera offset; the first value is the lateral offset. |
| `ground_y` | Camera height in `mm`. |
| `orientation` | Installation method: `0` upright, `2` inverted, `1` / `3` side-mounted. |
| `multi_level` | Multi-pallet recognition: `0` off, `1` on. |
| `quad` | Four-leg pallet recognition: `0` off, `1` on. |
| `min_z` / `max_z` | Valid range along the camera-distance axis. |
| `min_x` / `max_x` | Valid horizontal range. |
| `two_leg` | Two-leg pallet recognition: `0` off, `1` on. |

<Note>
  Do not change advanced parameters that are not listed here unless requested by MRDVS. Preserve the original configuration and validate each change if you need to adjust leg width, beam detection, or the recognition region.
</Note>

### Offline tests and logs

In **Advanced applications → Offline test**, drag in:

- the `pallet_settings.json` calibration file;
- one PNG image or a folder containing result images.

Choose a single-file or folder test and select **Start detection**. The log area shows camera API, firmware, algorithm, and recognition results. Use **Camera operations** to export software logs for troubleshooting.

## Communication integration

### TCP and UDP

In LxCameraViewer, select **Pallet localization** and set it to **Always on**. The TCP server uses port `5501` by default and UDP uses port `8000`. Messages use big-endian byte order and a standard frame length of `16` bytes.

| Direction | Hex command | Purpose |
| --- | --- | --- |
| Client → PalletPro | `60 04 00 00` | Start recognition. |
| Client → PalletPro | `60 04 00 01` | Stop recognition. |

The response frame contains a return code, length, X / Y / theta / Z signs, pallet-front-center X / Y, angle theta, and pallet height. Set extension bytes in the current protocol to `0`.

### RS485 (M4 Pro only)

Use Modbus RTU at `9600` baud and `8N1`. The default Modbus address is `0x17`; use the SDK to set an address from `1–247`.

| Operation | Function code | Register | Value |
| --- | --- | --- | --- |
| Enable / stop recognition | `0x06` | `120` (`0x0078`) | `1` enable, `0` stop |
| Read results | `0x04` | `122` (`0x007A`) | Read `8` registers |

Result fields are X, Y, yaw, return value, and height, each represented by a 4-byte field.

### CANopen (M4 / M4 Mega)

CAN uses standard 11-bit frames at `250 Kbit/s`. The controller sends `COB ID 0x212`; `MODE CTRL` `0` disables recognition and `1` enables it. The device returns recognition status, pose, position, height, and pallet dimensions through `0x192`, `0x292`, `0x392`, and `0x492`.

### Modbus TCP

This interface requires algorithm version `0.8.1` or later. The default port is `6601`; it listens on all network interfaces and uses register start address `0x100`.

| Offset | Register | Content |
| ---: | --- | --- |
| 0 | `0x100` | Absolute return value; `0` means recognition succeeded. |
| 1–4 | `0x101–0x104` | Signs for X, Y, yaw, and height; `0` positive, `1` negative. |
| 5–8 | `0x105–0x108` | Pallet-front-center X / Y, yaw, and height. |
| 9 | `0x109` | Pallet width. |
| 10 | `0x10A` | Number of pallet legs. |
| 11–19 | — | Reserved. |

## Verify and troubleshoot

### Calibration acceptance

Drive repeatedly between the near-end and far-end teaching positions. Calibration is acceptable when X and Y errors are within `10 mm` and recognition is stable without misses. For example, `x = -2000`, `y = 100`, and `yaw = 2.3` means the pallet is `2000 mm` from the camera optical center, `100 mm` to the right, and rotated `2.3°` counterclockwise in the top view.

### Common error codes

| Error code | Meaning | Priority check |
| ---: | --- | --- |
| `0` | Recognition normal | — |
| `-1` | No image data or insufficient data | Confirm that the camera outputs images. |
| `-2` / `-6` | Too few points or pallet legs not detected | Check mounting height, fork attitude, obstructions, and pallet-leg imaging. |
| `-3` | Pallet-leg dimensions outside the conditions | Adjust leg width or pallet-width limits. |
| `-4` | Camera height not set | Set and save the installation height. |
| `-5` | Beam detection error or excessive pallet tilt | Check beam imaging, flying pixels, and docking distance. |
| `-7` | Excessive difference between pallet-leg widths | Check whether reflectors, walls, or other objects affect edge-leg imaging. |
| `-8` | Pallet angle too large | Adjust pallet attitude. |

### Recognition conditions and site limits

- Standard conditions: pallet width and length `800–2000 mm`, height `80–200 mm`, leg width `40–200 mm`, opening width `300–600 mm`, beam thickness at least `30 mm`, and recognition distance `1200–3000 mm`.
- Keep the pallet fully inside the field of view. Avoid occlusion by forks, goods, or nearby objects.
- Test black, low-reflectivity, and highly reflective pallets on site. In the original test, a black pallet with `5%` reflectivity could be imaged at `2 m`; adjust integration time and the low-signal threshold when needed.
- For high-position fork pickup, keep at least `200 mm` between reflectors and pallet legs when possible.
- In live docking, use result timestamps at the application layer to determine whether data is fresh and prevent delayed corrections.

Contact MRDVS to evaluate non-standard or legless pallets.

## 1. Introduction

The volume detection algorithm is developed by Meyer Micro Vision (MRDVS) and is specifically used in volume measurement scenarios. It is used in self-developed M series cameras in logistics, express delivery, coal yards, warehousing and other scenarios. The volume detection algorithm runs inside the camera, eliminating the need for an external industrial computer. The algorithm supports static and dynamic detection.

The software only requires the user to calibrate the external parameters of the camera and draw the detection range, and then it can detect and measure single/multiple objects. External parameter calibration can automatically obtain calibration results and support multi-camera splicing detection.

This article mainly introduces the deployment and use of Meyer micro-vision volume detection function.

## 2. Volume detection application scenarios

### 2.1 Logistics industry

1. By obtaining information such as the volume and weight of goods, we can achieve refined operations, which will help improve vehicle loading rates, reduce vehicle costs, transportation costs and labor costs, and achieve cost reduction and efficiency improvement throughout the entire process.

2. It can quickly detect the volume and weight of goods/boxes and calculate freight with one click. Together with the floor scale, it can also effectively measure trucks and container trucks to ensure that the entire vehicle is not overweight.

### 2.2 Express Industry

When sending and receiving express delivery, with the help of intelligent volume measurement equipment, information such as the volume and weight of the goods can be quickly obtained. The data is connected to the internal system for easy search and review, which not only avoids the problem of data loss, but also effectively reduces manual investment and improves work efficiency.

### 2.3 Coal mining industry

Traditional methods for measuring the volume of large coal ore piles usually use total stations and RTK single-point measurement methods, and require manual extraction of feature points accurately. However, because large coal ore piles are usually inconvenient to climb, it is extremely difficult to extract precise feature points. At the same time, the single-point measurement method has low data density and cannot accurately obtain the volume of coal ore piles, resulting in extremely low operating efficiency.

### 2.4 Warehousing Industry

In logistics, warehousing and even general industrial industries, AGVs carry goods, and the volume and weight of goods need to be checked when entering and exiting the warehouse.

### 3. Solve the problem

In industrial industries such as logistics and warehousing, obtaining the volume and quantity of items and mastering the entry and exit of items are of great significance to production inventory management. However, traditional solutions always face problems such as lack of technical means, large measurement errors, low efficiency of manual management, and difficulty in supervision.

Meyer Micro Vision ToF camera volume detection has more advantages than radar, structured light, and 2D cameras. Meyer Micro Vision cameras are simple to deploy and only require camera firmware brackets, power cords, and network cables. The algorithm runs inside the camera without the need for redundant industrial computer configurations. It is more convenient to use and lower in cost than radar.

### 7.3 Volume detection external parameter calibration

1. After the camera IP configuration is completed and the firewall is closed, proceed to the second step of deployment. Open the software to calibrate the external parameters of the algorithm. It is necessary to calibrate the specific reference ground and camera installation angle deviation.

2. Click on the camera configuration and switch to the parameter calibration interface. Before performing external parameter calibration, make sure that x, y, z, roll, pitch, and yaw are 0. If they are not 0, you need to return all parameters to 0 and click the [**Save and Send**] button. After the parameters are saved successfully, re-calibrate the external parameters.

## 8. Communication protocol

### 8.1 TCP communication

1. Volume detection results can be obtained through TCP communication. Through the network debugging assistant, the remote camera IP: 192.168.100.82, port: 14951, send RESULT\_UPDATE to see the returned volume detection result information.

### 8.2 UDP communication

1. Volume detection results can be obtained through UDP communication. Through the network debugging assistant, the remote camera IP: 192.168.100.82, port: 14950, send RESULT\_UPDATE to see the returned volume detection result information.

<Frame>
  <img src="/images/legacy/17127c06-43af-4038-b287-8a42f8aa9ba7/905bafb7-3038-43f2-9c23-666327a4c4e0.avif" alt="UDP communication related icons" />
</Frame>

### 8.3 HTTP communication

1. Volume detection results can be obtained through HTTP communication, port number: 14952.

2. The upper-layer system (WMS) obtains the signal from the visual sending result, and the visual returns a string in JSON format.

| WMS -> Vision | ㅤ |
| --- | --- |
| Result | Meaning |
| RESULT_UPDATE | Trigger visual result return signal |
| Vision -> WMS | ㅤ |
| Result | Meaning |
| IP | Data source camera IP |
| TM | Volume detection status |
| angle | Angle of the selected object |
| cx | x coordinate of frame-selected object |
| cy | y coordinate of frame-selected object |
| cz | Frame selected object z coordinate |
| height | object height |
| length | object length |
| width | object width |
| valid | false if the confidence is too low, true otherwise |
| TM_DATA_TIME | Volume detection data extraction time |
| TM_RES_TIME | Volume detection result output time |
| nick_name | Volume detection camera name, configurable in the camera |
| ret_code | Return code, 0 is success |

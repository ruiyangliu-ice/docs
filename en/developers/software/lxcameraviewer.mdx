LxCameraViewer searches for and connects to cameras, displays images and point clouds, configures camera parameters, and provides application algorithm maintenance tools.

## Overview

LxCameraViewer is based on the MRDVS Camera SDK and supports Windows and Linux. Use it to view and save depth, intensity, point cloud, and RGB images; read and set camera and application algorithm parameters; and access IP configuration, packet capture, communication, and network-card configuration tools.

## Installation and network configuration

### Recommended system configuration

| Configuration item | Windows | Linux |
| --- | --- | --- |
| Operating system | Windows 10 (64-bit) or Windows 11 (64-bit) | Ubuntu (64-bit) |
| Memory | 4 GB or more | 4 GB or more |
| Network adapter and cable | Gigabit Ethernet or faster | Gigabit Ethernet or faster |

### IP address and gateway configuration

The camera uses a static IP address. Its default IP address is `192.168.100.82`, and the default gateway may be left blank. Use subnet mask `255.255.0.0` on Windows and `255.255.255.0` on Linux.

Set the host computer's Ethernet adapter connected to the camera to `192.168.100.x`, where `x` is any number from `3` to `254` and does not match the camera IP. For example, when the camera IP is `192.168.100.12`, set the host computer IP to `192.168.100.89`.

## Use LxCameraViewer

### Software interface overview

#### Camera not opened

The initial interface contains the following areas:

- **List panel**: Device list, current information, and device information.
- **Top menu bar**: Tools and interface language selection.
- **Image display area**: Point cloud, depth, intensity, and RGB views.
- Double-click the menu bar to maximize or restore the window. Drag the menu bar to move the window.

#### Camera opened

After you open a camera, the function bar appears on the right.

- **Basic tools**: Image display settings, 2D settings, 3D settings, filtering, hardware interface, collection settings, and function settings.
- **Application algorithms**: Working mode settings, algorithm mode settings, algorithm version, and detailed algorithm settings.
- **Other tools**: Load local point clouds, IP configuration, packet capture, network-card configuration, communication, GPU enablement, and log level.

- [Anti-distortion] Call the distortion parameters in the internal parameters to remove the distortion of the image in the interface display

- [Anti-distortion coefficient] Adjust the distortion coefficient to enhance or weaken the distortion of the shielded image

- [Auto Exposure] Turn on or off the camera to automatically set the RGB exposure value

- [Exposure] Manual exposure, can be set when automatic exposure is turned off

- [Gain] Manual gain, can be set when automatic exposure is turned off

##### 3D settings

- [3D Width] The width of the point cloud image is determined by the device model and cannot be set.

- [3D Height] The height of the point cloud image is determined by the device model and cannot be set.

- [Pixel binning] Press 1\*1 or 2\*2 or N\*N to add the pixel values in the matrix and divide by 2\*2 or N\*N to get the binning image, which can be modified after stopping the flow (the image size becomes the original size and is divided by the binning pixel size horizontally and vertically)

- [RGB Alignment] Align the depth map, intensity map, point cloud and RGB image. After the alignment, the depth map and intensity map will have the same width and height as the RGB image. The point cloud data will increase and can be modified after stopping the flow.

- [Multi-camera mode] Turning on when using multiple cameras at the same time can eliminate interference from other cameras, and can be modified after stopping the flow.

- [Multi-machine algorithm sensitivity] The built-in parameters of the device are determined by the device model and cannot be set.

- [Anti-distortion] Call the distortion parameters in the internal parameters to remove the distortion of the image in the interface display

- [Anti-distortion coefficient] Adjust the distortion coefficient to enhance or weaken the distortion of the shielded image

- [Frequency Mode] The built-in parameters of the device are determined by the device model (the frequency of some cameras is fixed and cannot be set)

- [HDR Enable] Turning on HDR enable can improve the picture clarity in scenes with strong changes in light and dark.

- [Auto Exposure] When turned on, the device automatically adjusts the exposure parameters according to the ambient light intensity.

- [Target intensity] Intensity threshold when acquiring depth images (cannot be set)

- [Auto exposure upper limit] Set the upper limit of automatic exposure

- [Auto exposure lower limit] Set the lower limit for automatic exposure (cannot be set)

- [Gain] can increase the intensity value by multiples (cannot be set)

- [x] The x coordinate of the upper left corner of the ROI area set, which can be modified after the flow is stopped.

- [y] Set the y coordinate of the upper left corner of the ROI area, which can be modified after the flow is stopped.

- [High integration] High integration time (increasing it allows the camera to measure distance further)

- [Low Integration] Low integration time (mainly controls distance measurement at a closer distance, generally does not need to be changed)

- [x offset] offset value in the width direction, which can be modified after stopping the flow

- [y offset] The offset value in the high direction, which can be modified after the flow is stopped.

- [Width] The width of the set ROI area, which can be modified after stopping the flow.

- [High] The height of the ROI area set, which can be modified after stopping the flow.

- [3D Frame Rate] Set the point cloud frame rate within the device frame rate range

##### Filter settings

- [Minimum depth] The lower limit of the depth map's false color or distance. Below this value, the false color is displayed as black and is forcibly set to zero.

- [Maximum depth] The upper limit of the depth map's false color or distance. Above this value, the false color is displayed as black and is forcibly set to zero.

- [Low signal threshold] The lower limit of the intensity value. Below this value, the corresponding pixels in the depth map are forcibly set to zero (not effective for the intensity map)

- [High signal threshold] The upper limit of the intensity value. If it is higher than this value, the corresponding pixels in the depth map are forcibly set to zero (does not take effect on the intensity map)

- [Small signal measurement] Set the small signal detection level. The higher the level, the easier it is to detect.

- [Filter Mode] Set filter shielding level

- [Smoothing Level] Controls the smoothing intensity of spatial filtering, retaining object edge details while smoothing noise

- [Noise Level] Set the threshold for noise filtering (the larger the value, the more noise)

- [Temporal Level] Control the intensity of temporal filtering and use historical multi-frame data to improve the continuity of depth data

- Note: Setting the smoothing level, noise level, and time domain level to 0 means turning off filtering.

##### Collection settings

- [Frame synchronization] Control the environmental conditions of each type of image capture to be consistent when saving images

- [Trigger Mode] Control the flow mode to be continuous, passive or active

- [Point cloud saving method]

    - Regular (unordered point cloud): The number of point clouds remains unchanged, width = width\*height, height = 1, as shown in Figure 3

    - Ordered point cloud: The number of point clouds remains unchanged, width = ToF width, height = ToF height, as shown in Figure 4

- [Point cloud saving format] Set the data format for saving point clouds

    - BINARY: Save in binary format

    - ASCII: Save in ascii code format, that is, save in text mode

- [Point cloud value unit] Set the measurement unit of the data in the point cloud image.

- [Start saving pictures] Continuously save the currently selected type of pictures

- [Open Save Path] Open the path to save pictures with the [Single Save] function

Note: For a 640\*480 image, the ascii saving time is about 6.6s, and the binary saving time is about 32ms. The binary saving speed is 206 times that of ascii. It is recommended to save in binary format.

##### Obstacle avoidance algorithm

- Set the application algorithm to obstacle avoidance algorithm, and obstacle avoidance related configurations will be displayed.

- [Display car body] You can turn off or turn on the car model that displays the position of the car body in the point cloud image.

- [Obstacle avoidance index] Obstacle avoidance can have multiple configurations, set through this option

- [Camera Angle Roll (x-axis)] Camera external parameter, roll angle, the default camera is 180

- [Camera angle Pitch (y-axis)] Camera external parameter, pitch angle, default vertical installation, is 90

- [Camera angle Yaw (z-axis)] Camera external parameter, heading angle yaw, default is 90

- [Camera position (distance x)] Camera external parameter, the distance between the x direction and the origin, unit: meters.

- [Camera position (width y)] Camera external parameter, the distance between the y direction and the origin, if installed in the middle, it is 0. Unit: Meter.

- [Camera position (height z)] The external parameter of the camera, the distance between the z direction and the origin, can be understood as the height of the camera from the ground, unit: meters.

- [Early Warning Distance (x far)] Obstacle avoidance range, the distance from the center of the car in the x direction. Obstacles beyond this distance are considered safe; the distance between the early warning distance and the alarm distance is considered early warning deceleration, and the interface displays a yellow area. Unit: meter.

- [Alarm distance (x center)] Obstacle avoidance range, the distance from the center of the car in the x direction. Obstacles below this distance are regarded as alarm stops, and the interface displays a red area; the distance between the pre-warning distance and the alarm distance is regarded as an early warning deceleration, and the interface displays a yellow area. Unit: meter.

- [Shielding distance (near x)] Obstacle avoidance range, the distance in the x direction from the center of the car. Obstacles within this distance will be ignored. Unit: meter.

- [Obstacle avoidance range left (y left)] Obstacle avoidance range, the distance from the left side of the y direction to the center of the car. Unit: meter.

- [Obstacle avoidance range right (y right)] Obstacle avoidance range, the distance from the right side of the y direction to the center of the car. Unit: meter.

- [Obstacle avoidance height lower (z lower)] Obstacle avoidance range, height from the ground, below this height will not be processed. If the setting is too small, false alarms may occur due to ranging errors; if the setting is too large, large obstacles on the ground may be filtered out, resulting in false alarms. Unit: meter.

- [Obstacle avoidance height upper (z upper)] Obstacle avoidance range, height from the ground, higher than this height will not be processed. Unit: Meter.

- [Obstacle avoidance output] Output obstacle avoidance results

- [Reset camera external parameters] Restore all parameters that have not been written to the camera

- [Set external parameters] Write all parameters to the camera

- For detailed obstacle avoidance settings, please see the obstacle avoidance documentation.

##### Pallet positioning

- Set the application algorithm to pallet positioning, and obstacle avoidance related configurations will be displayed.

- [Project Name] Name the current pallet recognition parameter configuration

- [Quick Configuration] Several configurations provided by the camera by default

- [Camera Angle Roll (x-axis)] The tilt angle of the camera relative to the X-axis

- [Camera angle Pitch (y-axis)] The tilt angle of the camera relative to the Y-axis

- [Camera Angle Yaw (z-axis)] The tilt angle of the camera relative to the Z-axis

- [Camera position (distance x)] The relative distance between the camera optical center and the forklift rotation center in the X direction

- [Camera position (width y)] The relative distance between the camera optical center and the forklift rotation center in the Y direction

- [Camera position (height z)] The relative distance between the camera optical center and the forklift rotation center in the Z direction

- [Ground Height] The distance from the optical center of the camera to the ground

- [Maximum pallet leg width] Must be compatible with the maximum width of pallet legs in the pallet type

- [Minimum Pallet Leg Width] Must be compatible with the minimum width of pallet legs in the pallet type

- [Distance from the tray] The distance from the optical center of the camera to the tray after the insertion is completed

- [Installation method] Set according to the fixed direction of the camera, the default is formal installation (0)

- [Minimum Z value] Identify the closest depth distance of the pallet

- [Maximum Z value] Identify the farthest depth distance of the pallet

- [Minimum X value] Identify the lateral boundary of the pallet (the left boundary with the camera as the center)

- [Maximum X value] Identify the lateral boundary of the pallet (the right boundary with the camera centered)

- [Pallet algorithm output] displays the results of the pallet algorithm

- [Calibration Instructions] Automatically calibrate when the ground is flat and there is no obstruction in front of the pallet. The pallet is of standard size.

- [Restore default parameters] Restore all parameters to factory defaults

- [Read external parameters from camera] Read all parameters from the camera, all unsent parameters will be overwritten

- [Send parameters] Write all parameters to the camera

- For detailed settings of pallet detection, please refer to the pallet detection function documentation.

## Troubleshooting

|  |  |
| --- | --- |
| Problem Description | Solution |
| Cannot find the camera | * Check whether the camera power is connected. After the power is turned on, the blue light on the front of the camera is always on * Check whether the IP address and subnet mask are correct * Check whether the host computer is prohibited from accessing the network |
| The camera can be opened but the streaming fails | Please check whether the host computer has been opened with administrator rights |
| Image lag | Check whether the network card, network cable, and switch are Gigabit. After turning on RGB alignment, the 100M network will experience a certain degree of lag |
| The camera frequently reconnects | After restoring factory settings, firmware upgrade, or when the camera is turned on for the first time under 100M network, the camera will adjust/restart itself for about 30 seconds. Within 30 seconds, the host computer will keep trying to connect to the camera until the camera is stable |
| No depth data, point cloud data | * Check whether the camera is blocked * Check whether the object is within the camera shooting range * Check whether the maximum/minimum depth values are correct * Check whether the filter parameters are correct |
| The point cloud disappears and only the white frame can be seen on the interface | Click the point cloud image with the mouse and press the 'm' key to turn on point cloud stabilization |
| Point cloud display is abnormal | Check whether the point cloud shortcut key has been operated and turn on point cloud stability |
| The point cloud is not displayed after opening the camera under Linux | Please use the mouse to press and drag the point cloud image to display |
| Application display: VTK loading failed, point cloud will not be displayed, please check the graphics card configuration and graphics card driver | * Check whether the graphics card configuration is too low * Check whether the graphics card driver is running normally * Point cloud display requires a graphics card with independent video display memory and its driver |

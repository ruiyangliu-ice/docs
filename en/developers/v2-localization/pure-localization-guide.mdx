This page describes the V2 localization-only web interface. Before building a map, switching maps, or relocalizing, make sure the vehicle, camera, and LiDAR are operating normally.

<Warning>
  Switching maps, updating firmware, and restarting the built-in application affect localization services. Perform these operations only with the vehicle stopped, the work area safe, and a recovery procedure available.
</Warning>

## Interface overview

| Area | Contents |
| --- | --- |
| Menu bar | Mapping, relocalization, map management, and maintenance functions. |
| Map view | The loaded 2D contour map and the vehicle localization icon. |
| RGB view | The live RGB image from the top-view camera. |
| Status area | Relocalization signal strength, vehicle coordinates, speed, and cursor coordinates. Signal quality is shown from best to worst as green, blue, and red. |

## Build a map

### Create a map

<Steps>
  <Step title="Start mapping">
    Click **Start Mapping** in the web interface.

    <Frame>
      <img src="/images/developers/v2-localization-pure-localization-guide-1-starting-to-build-a-map.avif" alt="Start Mapping button in the V2 localization-only interface" />
    </Frame>
  </Step>
  <Step title="Enter a map name">
    A map name may contain only letters, numbers, and underscores, and must start with a letter.

    <Columns cols={2}>
      <Frame>
        <img src="/images/developers/v2-localization-pure-localization-guide-1-starting-mapping-2.avif" alt="Map name entry dialog" />
      </Frame>
      <Frame>
        <img src="/images/developers/v2-localization-pure-localization-guide-1-starting-mapping-3.avif" alt="Start mapping page" />
      </Frame>
    </Columns>
  </Step>
  <Step title="Capture the area and stop mapping">
    Drive the AGV through the target area. When capture is complete, click **Stop Mapping** and wait for processing to finish. Processing time depends on the size of the captured area.
  </Step>
  <Step title="Confirm the map">
    After mapping succeeds, open **Map Management** and confirm that the new map is listed.
  </Step>
</Steps>

### Incremental mapping

Use incremental mapping to add an area to an existing map. Before starting, complete relocalization and switch to the map that you want to extend.

<Info>
  Incremental mapping is started and stopped with a manual HTTP request at [http://192.168.100.201:9998/tool/rob/sendCMD](http://192.168.100.201:9998/tool/rob/sendCMD).
</Info>

<Tabs>
  <Tab title="Start incremental mapping">
    ```json
    {
      "cmd": 904,
      "attach": {
        "sub_command": 0,
        "param": ""
      }
    }
    ```
  </Tab>
  <Tab title="Stop incremental mapping">
    ```json
    {
      "cmd": 905,
      "attach": {
        "sub_command": 0,
        "param": ""
      }
    }
    ```
  </Tab>
</Tabs>

After the operation completes, the newly captured area is written to the current map. Restart the localization service for the change to take effect.

## Relocalization

### Manual relocalization

Click **Relocalization**, then drag in the map view to provide an approximate vehicle pose so that the LiDAR point cloud matches the contour map. The **Localization timeout** fault disappears after relocalization succeeds.

<Frame>
  <img src="/images/developers/v2-localization-pure-localization-guide-3-relocation.avif" alt="Manual relocalization in the V2 localization-only interface" />
</Frame>

### AprilTag relocalization

You can place AprilTags at selected locations while building a map. When a complete tag enters the device field of view, the system identifies its map and pose, switches maps if necessary, and relocalizes automatically.

- Generate tags with [the AprilTag generator](https://chaitanyantr.github.io/apriltag).
- Tag IDs must be unique within the visible area of a device.
- Use the fixed tag family shown in the interface example.

## Maintenance

### Update firmware

Import a local firmware package on the update page to update the chassis, actuator, or camera firmware. A restore package records the version before the previous update and can be used to roll back.

<Warning>
  Camera firmware upgraded through the standard host-computer workflow cannot be rolled back. Verify the package and current version before updating.
</Warning>

### Stop or restart the built-in application

Use the built-in application controls to stop or restart the application. Stop the built-in application before connecting LxCameraViewer to the camera; otherwise, the vehicle may retain control of the camera and prevent communication.

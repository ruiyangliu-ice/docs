## Overview

The camera runs as a TCP Socket server and listens on port `5501` by default. Messages are text; ASCII is supported, and JSON encoding is recommended.

<Info>
  Use this protocol when a robot or PLC triggers a 3D vision task and reads the result returned by the camera.
</Info>

## Request message

Send the following message to trigger a 3D vision task:

```json
{"action":"3D"}
```

## Response message

After the camera finishes, it returns a JSON object.

<ResponseField name="result" type="integer" required>
  Detection status. `0` indicates localization failure or no result. Any other integer indicates success and gives the number of returned results.
</ResponseField>

<ResponseField name="pos" type="number[]" required>
  Array of object poses. Each result contains `X`, `Y`, `Z`, `RX`, `RY`, and `RZ` in the robot coordinate system. Position is in mm and angles are in °; `−180° ≤ RZ ≤ 180°`. All values are `0` when detection fails.
</ResponseField>

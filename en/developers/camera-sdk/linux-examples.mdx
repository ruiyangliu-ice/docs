## Overview

The MRDVS Camera SDK for Linux includes C/C++, Java (HTTP service), ROS, and ROS 2 examples. The examples show the basic SDK workflow; add application-specific logic and error handling for your project.

Complete the environment setup, installation script, and sample-directory checks first. Then follow the compilation and execution section for your target language. The appendix lists installation methods for common dependencies and test packages.

### 5.3 ROS1 & ROS2

In the ros description, we will not talk about the driver source code, but about the content corresponding to the configuration in the launch file, how to adjust the configuration, and how to send and subscribe to messages. The provided ros node already contains most of the functions, so it is not recommended to directly modify the driver source code. If you have special needs, you can first understand the code flow in the C++ example and then modify the driver source code.

**When executing the ros driver, you must ensure that the versions of opencv and pcl in the system default path are unique, otherwise an error will occur when linking the ros library, which may cause compilation failure or runtime crash.**

#### 5.3.1 ROS1 launch file description

##### 5.3.1.1 All launch files

1. There are the following files in the src/launch folder:

<Frame>
  <img src="/images/developers/camera-sdk-linux-examples-all-launch-file.avif" alt="All launch file related icons" />
</Frame>

- localization.launch: Run file for visual positioning nodes

- lx\_camera\_ros.launch: running file of ordinary nodes

- mapping.launch: running file for visual positioning mapping

- obstacle.launch: running file of obstacle avoidance algorithm

- obstacleV2.launch: running file of obstacle avoidance algorithm V2

- pallet.launch: pallet algorithm running file

- sensor\_sim.launch: running file for simulating positioning data

2. You can execute different functions of the node by running different launch files. **Except for localization, mapping and sensor, the parameter contents are the same in all nodes, but the configuration is different**. This will be introduced in detail in the following sections. For a detailed introduction to the parameters of the three launch files of localization, mapping and sensor, please see the introduction in the ros-v1pro node.

##### 5.3.1.2 Camera IP, stream configuration, working mode configuration, point cloud unit, etc.

<Frame>
  <img src="/images/developers/camera-sdk-linux-examples-camera-ip-stream-configuration-working-mode-configuration-point-cloud-unit-and-other.avif" alt="Camera IP, stream configuration, working mode configuration, point cloud unit and other related icons" />
</Frame>

- ip: The camera IP that needs to be connected. If it is 0, the first one will be connected by default.

- log\_path: SDK log storage path, the absolute path needs to be filled in

- is\_xyz: whether to push point cloud data

- is\_depth: whether to push depth data

- is\_amp: whether to push intensity map data

- is\_rgb: whether to push RGB image data

- lx\_work\_mode: work mode

- lx\_application: whether to enable the algorithm, 0: off, 1: obstacle avoidance algorithm, 2: pallet algorithm, 3: positioning algorithm, 4: obstacle avoidance algorithm V2

- lx\_tof\_unit: point cloud unit, 1 is meter, 0 is millimeter, needs to be changed to 1 when viewing with rviz

##### 5.3.1.3 Camera pose configuration

<Frame>
  <img src="/images/developers/camera-sdk-linux-examples-camera-pose-configuration.avif" alt="Illustrations related to camera pose configuration" />
</Frame>

The rotation angle and translation vector of the point cloud can be configured.

- X: X translation amount of the camera from the center of the point cloud

- Y: Y translation amount of the camera from the center of the point cloud

- Z: Z translation amount of the camera from the center of the point cloud

- Yaw: the angle of rotation of the point cloud around the Z axis

- Roll: The angle of rotation of the point cloud around the X-axis

- Pitch: The angle of rotation of the point cloud around the Y axis

##### 5.3.1.4 Whether to use launch configuration

<Frame>
  <img src="/images/developers/camera-sdk-linux-examples-whether-to-use-launch-configuration.avif" alt="Whether to use launch configuration related icons" />
</Frame>

You can choose whether to use the camera parameter configuration in the launch file.

- raw\_param: 0: use the configuration in the camera, 1: use the configuration in the launch file

Regardless of whether this parameter is set to 0 or 1, all parameters before this parameter will be set, 0 and 1 are only valid for parameters after this parameter.

**1. Some nouns are explained as follows:** 

-Single stream: refers to turning on any stream of RGB, depth, and intensity

- Dual stream: refers to turning on any two streams of RGB, depth and intensity

-Three streams: refers to RGB, depth, and intensity all three streams are turned on

- Four streams: means RGB, depth, intensity, and point cloud are all turned on

**2. The algorithm has a binding relationship with some streams. The binding relationship is as follows:** 

- To enable the pallet algorithm, the depth must be enabled

- To turn on the obstacle avoidance algorithm, you must turn on depth

- To turn on the positioning algorithm, you must turn on RGB

## Application background

In an AGV warehouse, manual put-away and retrieval can leave WMS location data out of sync with the site. If an occupied location is not detected in time, the dispatch system may issue an unsuitable retrieval or put-away task.

## Recognition-method comparison

| Method | Available information | Typical limitation |
| --- | --- | --- |
| Single-point laser ranging | One distance value | Gaps between cartons or pallets can be missed, making it difficult to represent the full location status. |
| RGB camera | Color and texture | Sensitive to targets outside the training set, lighting, and edge distortion; cannot directly provide load height. |
| RGB-D camera | Color, depth, and point cloud | 3D data can be used to determine occupancy, placement, and load height. |

## Solution overview

The solution captures color and 3D data from storage locations to identify whether a load is present, how it is placed, and whether occupancy is abnormal. Camera-side algorithms can also distinguish load categories when required by the project. Results can be sent to the dispatch system to support warehouse operations.

## Solution characteristics

- **3D status assessment:** Combine depth and color data to identify occupancy and load status.
- **On-camera processing:** Run the algorithm on the camera to reduce external compute requirements.
- **Result reporting:** Support TCP/IP, UDP, and HTTP communication, with JSON output to the control system.
- **Dispatch coordination:** Provide real-time location data to support task assignment.

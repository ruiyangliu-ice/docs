Manual forklift operations often depend on driver experience, while storage-location records can lag behind the actual site. Path efficiency, location status, and safety in narrow aisles are common warehouse-management concerns.

## Solution architecture

The solution combines a manual-forklift localization system with a storage-location status system. The first provides forklift position and pose; the second provides occupancy and load status. Both data streams can be sent to a higher-level system for task dispatch and load tracking.

## Manual forklift localization

Mount a 3D vision camera on top of the forklift. The system localizes using natural features on the ceiling and walls, with IMU assistance. Project records report **1–5 cm** positioning accuracy; actual accuracy depends on site features, installation, and calibration.

## Storage-location status recognition

An RGB-D camera captures 3D and color information to identify whether a location contains a load, how it is placed, and whether occupancy is abnormal. The algorithm can distinguish load categories when required, and camera-side results can be sent to the dispatch system.

| Feature | Description |
| --- | --- |
| 3D monitoring | Combine depth and color data to identify location status. |
| On-camera processing | Reduce the need for an external industrial computer. |
| Communication integration | Support TCP/IP, UDP, and HTTP, with JSON output. |
| Dispatch coordination | Provide real-time location information for task assignment. |

## Coordination workflow

1. The localization system continuously outputs forklift position and pose.
2. The location camera reports occupancy and load status for the target location.
3. The higher-level system associates vehicle, load, and location data to confirm pickup and placement tasks and record load positions.
4. When location or vehicle status changes, the higher-level system can update task routes and resource allocation.

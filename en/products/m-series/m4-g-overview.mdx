M4-G is an iToF RGB-D camera for close-range perception. It uses a GMSL2 interface and can output depth, RGB, and point-cloud data at a typical 30 fps within a 0.05–1 m range. Typical applications include robot wrists, tabletop grasping, and precision manipulation.

## Key features

- **Close-range iToF depth sensing:** Outputs depth data within a 0.05–1 m range.
- **Synchronized RGB-D output:** Provides depth maps, RGB images, and point clouds with RGB alignment.
- **Low-light performance:** Active illumination supports targets with low illumination or limited texture.
- **GMSL2 transmission:** Provides a long-distance, interference-resistant data link for system integration.

## GMSL2 interface

### Serializer configuration

| Item | Parameter |
| --- | --- |
| Serializer | MAX96717 |
| I²C address | `0x40` (7-bit) |
| GMSL rate | GMSL2, 6 Gbps |
| Operating mode | Pixel mode |
| Frame synchronization | Controlled by MAX96717 MFP0 |

### Video stream parameters

| Stream | Resolution and format | Data type | Control and virtual channel |
| --- | --- | --- | --- |
| ToF | 2560 × 960, RAW8 | `0x2A` | I²C forwarded by FPGA; VC0 |
| RGB | 1280 × 1080, YUV422, 8-bit | `0x1E` | I²C forwarded by FPGA; VC1 |

### FPGA configuration

| Item | Parameter |
| --- | --- |
| I²C address | `0x11` (7-bit) |
| MIPI output | 4 lanes, 1.2 Gbps |

## Start streaming

Image capture requires both hardware frame synchronization and stream control.

<Steps>
  <Step title="Provide a frame synchronization signal">
    Configure frame synchronization between MAX96717 and the selected deserializer according to the hardware manuals. MAX96717 MFP0 can be configured as a GPIO trigger output.

    <Frame>
      <img src="/images/products/m-series-m4-g-overview-2-max96724-f-r-frame-sync-signal-transmission.avif" alt="MAX96724 series frame synchronization signal transmission" />
    </Frame>
  </Step>
  <Step title="Send the stream-start command">
    Write `1` to FPGA register `0x2000` to start streaming, or write `0` to stop streaming.

    ```bash
    i2ctransfer -f -y 4 w3@0x11 0x20 0x00 1
    ```

    In this example, `0x11` is the FPGA I²C address.
  </Step>
</Steps>

## 1. Product overview

This guide describes the M4 Mega camera developed by Zhejiang MRDVS Technology Co., Ltd., including its specifications, features, and operating instructions.

The M4 Mega product family includes the M4 Mega camera. This ToF product uses a Gigabit Ethernet communication interface and a 24VDC power supply.

## 2. Product specifications

### 2.1 Distance measurement principle

The M4 Mega camera uses time-of-flight (ToF) ranging. It measures the time between emitting light and receiving the reflection from an object to calculate the distance between the object and camera.

The M4 Mega camera captures depth directly, providing high-quality depth information with minimal compute resources and building depth scenes at a high frame rate. Like LiDAR, ToF cameras can use pulsed ToF (P\_ToF) or continuous-wave modulated ToF (CW\_ToF). P\_ToF emits light pulses, usually invisible, and calculates distance from their round-trip time. CW\_ToF emits modulated continuous light and calculates distance from the phase difference between the emitted and returned light.

This compact industrial RGB-D camera integrates a Sony CW iToF camera with three-channel true-color RGB output. It provides depth data from 0.2m~5m and aligns RGB and depth images spatially and temporally. The camera can also output structured pallet-recognition results over CAN.

### 3.3 Detection range

Figure 3.4 shows the M4 Mega detection area. The camera's horizontal field of view is 81°, and its vertical field of view is 61°. Detection distance depends on optical power. Higher optical power and target reflectivity increase the near blind zone but reduce the far blind zone. Lower optical power and target reflectivity reduce the near blind zone but increase the far blind zone. For objects with 10%~90% reflectivity, the M4 Mega measurement range is 0.2~5m. Beyond 5m, weak reflected signals are more susceptible to ambient light and produce larger errors.

### 3.8 Hardware trigger

1.  Hardware triggering works only when the product is in hardware-trigger mode. Hardware-trigger and obstacle-avoidance modes are mutually exclusive. In hardware-trigger mode, the camera waits for a valid pulse on I/O input pin 1, exposes one frame, and returns to the waiting state.

2.  The hardware trigger signal must be 3.3V~24V and provide at least 10mA drive current. Add a decoupling circuit to the trigger signal pin to prevent false triggers from pulse noise.

### 3.8.1 Configure hardware triggering in the host application

Open the camera → verify that the obstacle-avoidance algorithm is off → stop the stream → **Basic tools** → **Acquisition settings** → switch **Trigger mode** to **Hardware trigger** → configure the parameters → start the stream and wait for an external trigger signal.

**1. Hardware trigger parameters**

**Filter time:** Time to wait before checking the trigger signal again after the hardware trigger input receives it. This filters noise. Use a value below 5ms. A larger value may filter valid trigger signals.

**Minimum interval:** Interval between two consecutive automatic trigger signals. The range is 70000~100000000μs, and the value must exceed the integration time.

**Trigger time:** Delay after checking the trigger signal again. Set to 0 for immediate triggering. The range is 0~100000000μs.

**Frames per trigger:** Number of consecutive frames triggered by one hardware trigger. The range is 1~200 frames.

**2. Hardware trigger examples**

**Example 1:** Filter time: 1000; minimum interval: 70000 (minimum); trigger time: 2000; frames per trigger: 10.

After receiving the signal, the camera waits 1ms and checks it again. It waits another 2ms before triggering the first frame, then waits 70ms between subsequent frames until frame 10. Trigger signals received during this sequence are ignored.

**Example 2:** Filter time: 1000; minimum interval: disabled; trigger time: 0; frames per trigger: 1.

After receiving the signal, the camera waits 1ms, checks it again, and triggers immediately. The minimum interval does not apply when frames per trigger is 1.

**Example 3:** Filter time: 1000; minimum interval: 200000; trigger time: 0; frames per trigger: 200.

## 4. Accuracy and precision

### 4.1 Accuracy

Calibration removes fixed offsets between sensor measurements and actual values by mapping one to the other. To calibrate an M4 Mega camera, mount it on a rail and point it at a white wall. Map each pixel's measured value to the distance between the camera and the corresponding wall area, then correct measurements with this mapping, as shown in Figure 4.1.

<Frame>
  <img src="/images/legacy/379b03b6-5b43-4db0-bc44-b715c859694e/cd16d195-e215-49ca-a673-1c00034af49f.avif" alt="M4 Mega calibration setup" />
</Frame>

Figure 4.1. Calibration

The M4 Mega uses active ranging, so many factors affect its accuracy. Improper installation can introduce multipath effects and distort measurements.

This is especially noticeable when measuring a low-reflectivity target beside a highly reflective object. Ideally, light follows the blue path. Light reflected from a wall may also reach the target and return to the camera along the green path. Illumination reflected directly from the adjacent high-reflectivity object may enter the camera along the red path. Optical imperfections turn this light into glare that interferes with ranging, as shown in Figure 4.2.

### 4.2 Precision

Precision typically refers to single-point variation (1-σ). As an optoelectronic measurement device, the M4 Mega is affected by ADC conversion, signal-to-noise ratio, and system noise. Figure 4.3 shows the relationship between measurement precision and signal intensity.

<Frame>
  <img src="/images/legacy/ee73d646-4d62-4aaf-8833-5f2c4b5cf3d9/a3a07cf5-cd92-42b4-a817-e1ef02896e6d.avif" alt="Measurement precision and signal intensity" />
</Frame>

Figure 4.3. Measurement precision and signal intensity

The M4 Mega converts optoelectronic analog signals into digital data. During ADC conversion, the analog value is rounded to the smallest resolution step, introducing noise. A 1.1V signal may be represented as 1V, producing 10% error. A 10.1V signal may become 10V, and a 100.1V signal may become 100V, each producing about 1% error. ADC conversion therefore introduces more noise and error at low signal levels. A ToF camera's signal-to-noise ratio generally follows SNR∝1/√t, where `t` is integration time. Increasing integration time raises signal intensity.

## 5. Multi-camera interference

### 5.1 Parallel interference

When two cameras are installed in parallel, each can receive both its own reflected illumination and illumination from the other camera, interfering with depth data. Parallel interference increases measurement error when two M4 Mega cameras face the same direction.

<Frame>
  <img src="/images/legacy/3b09c42a-62ef-4a7f-96ad-6366d662588e/172c3d00-c862-4b8b-b294-8cfbcccfe558.avif" alt="Parallel camera interference" />
</Frame>

To avoid parallel interference, install the cameras at an angle to the horizontal. If they must be installed at 0° to the horizontal plane, contact MRDVS for a custom configuration.

### 6.4 Demo application

### 6.4.1 IP configuration

Connect the power cable, then connect the camera to the computer with an Ethernet cable. Because the camera's factory-default IP address is 192.168.100.82, set the computer to the 192.168.100.x subnet, excluding 100.82, with subnet mask 255.255.255.0. Open **Settings** → **Ethernet** → **Change adapter options**. Right-click **Ethernet**, select **Properties**, then select **Internet Protocol Version 4 (TCP/IPv4)**. For example, set the IP address to 192.168.100.20 and the subnet mask to 255.255.255.0.

Download and install the software, then double-click the LxCameraViwer demo icon. The host application opens, searches for a camera, opens it, and starts acquisition automatically when one camera is connected.

The application includes basic controls, image selection, camera configuration, and camera information. Camera information includes the IP address, ID, MAC address, serial number, and software and algorithm versions. Select the depth, amplitude, RGB, or point-cloud checkbox to display that view in the center panel.

### 8.2 Correct operation

The M4 Mega camera uses a 24V DC power supply. Connect the power cable before applying power. Do not reverse the positive and negative terminals, as this may damage the product.

| No. | Issue | Resolution |
| --- | --- | --- |
| 1 | The camera does not respond when opened | Confirm that the firewall is disabled. |
| 2 | The host application detects multiple IP addresses | A duplicate IP address exists on the LAN. Select the required IP address. |
| 3 | Camera data is unstable after opening | Use a Gigabit Ethernet cable. Data may initially be unstable over a 100Mbps cable. |
| 4 | Software installation location | Installing to the default location on the C drive may require permission changes. |

### 8.3 Operating temperature

Keep the ambient temperature within the range specified in Table 2-1. Mount the product on a thermally conductive component, such as a metal bracket, to improve heat dissipation and achieve optimal performance.

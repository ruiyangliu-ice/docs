## 1. Product overview

### 1.1 Introduction

The MRDVS V2 Pro is a navigation controller that fuses data from an upward-facing visual SLAM camera, 2D LiDAR, and an IMU. Its integrated compute platform requires no external industrial computer. As the robot moves through the navigation area, the camera captures environmental data and generates a map for mobile-robot localization and navigation.

### 1.2 Features

#### 1.2.1 Mobile robot navigation

- Fuses data from a 108°×57° upward-facing camera, a 2D LiDAR with a 270° scan range, and an IMU for ±1cm repeat positioning accuracy.

- Uses a 6 TOPS processor and a visual interface for mapping.

#### 1.2.2 Manual forklift enhancement

- Includes Wi-Fi 6 for manual-forklift localization data transmission.

- Provides a 3-pin audio output for voice prompts and alerts.

### 1.3 Applications

1.3.1 Factory logistics: Supports seamless indoor and outdoor operation by AGVs and autonomous forklifts between warehouses, loading docks, and workshops.

1.3.2 Manual forklift upgrades: Replaces UWB with higher accuracy (under 3cm), lower deployment cost, and shorter deployment time.

#### 3.3.2 Wi-Fi communication module

The V2 Pro navigation controller includes a wireless network adapter for stable, high-speed Wi-Fi. Multi-user MIMO (MU-MIMO) increases channel capacity when multiple devices share the same band, and dual-stream operation supports rates up to 1200Mbps. The adapter supports IEEE 802.11ax/ac/a/b/g/n in the 2.400GHz~2.4835GHz ISM band.

### 5. Quickstart

### 5.1 Package contents

| No. | Description |
| --- | --- |
| 1 | V2 Pro controller |
| 2 | 2-pin power cable |

#### 5.2 Connection test

<Frame>
  <img src="/images/legacy/dfd4c3be-816c-4ffc-81ac-13141e3c2644/9d8858e6-85a0-492f-a353-4e5adf1f81a2.avif" alt="Connection test" />
</Frame>

#### 5.2.1 Power the camera

- Connect one end of the DC cable to the camera and the other to a 24V/3A DC adapter.

- A slowly flashing blue power indicator confirms normal power-on.

- If the indicator is off or abnormal, such as solid red, check the power connector and adapter.

#### 5.2.2 Configure the network

- Connect the camera directly to the computer with a Category 6 Ethernet cable.

- The factory-default camera IP address is 192.168.100.201.

- Set the computer IP to the same subnet and disable the firewall.

- On a LAN computer, enter 192.168.100.201:9998 in a web browser to sign in without a password.

#### 5.2.3 Run the software

- Run the host software, acquire camera images, and verify that they are clear and stable.

- See section 5.3 or the LxCameraViewer user guide.

Contact MRDVS sales or technical support for further help.

#### 5.3 Interface overview

<Frame>
  <img src="/images/legacy/4e6942a8-55ab-4030-9609-4c1392a560fa/69362392-66e2-4bd2-8fc3-72e4a3e1efbe.avif" alt="V2 Pro interface" />
</Frame>

1.  Red menu bar: Open functions such as mapping, relocalization, and map management.

2.  Blue box: Displays the loaded 2D outline map and vehicle-localization icon.

3.  Green box: Displays the live RGB image from the upward-facing camera.

4.  Yellow box: Displays relocalization signal strength in real time (green: excellent, blue: good, red: poor), along with the vehicle coordinates, vehicle speed, and cursor coordinates.

#### 5.4 Mapping

1. Start mapping

<Frame>
  <img src="/images/legacy/edcaac72-adf1-4432-ba27-9b7323cde695/8cdd9468-9cb7-437d-a823-4a0cbffae053.avif" alt="Start mapping" />
</Frame>

2. Enter a map name

- A map name can contain letters, numbers, and underscores but cannot begin with a number.

3. Mapping interface

<Frame>
  <img src="/images/legacy/2568abac-69b0-4e56-af86-53575c614918/12edb330-0198-49f8-a1de-cbcc05192b4a.avif" alt="Mapping interface" />
</Frame>

- During mapping, keep the vehicle's linear speed at or below 0.5m/s and angular speed at or below 20 degrees/s.

- The interface displays the laser outline and upward-facing camera image.

4. Plan an approximate mapping path in advance. Move steadily at low speed (&lt;0.2m/s), use straight lines and right-angle turns, and avoid repeated paths and random vibration. Maintain approximately one-third visual overlap along paths in the same direction. Parallel paths should usually be approximately 2m apart. Cover the mapping area or robot route as completely as possible, as shown below.

<Frame>
  <img src="/images/legacy/7f2923e9-c17c-47a0-8720-9de448d0a851/045b2eb3-7aca-4ae2-8693-660381d3301e.avif" alt="Recommended mapping path" />
</Frame>

5. Finish mapping

- After the AGV scans the operating area, click **Finish mapping** in the menu and wait. Processing time depends on the scanned area.

#### 5.5 Map management

1. Map management interface

2. Background maps

<Frame>
  <img src="/images/legacy/60beebd0-634a-4145-907c-e553c16a5db1/82af1a8d-d514-4304-89d0-439e413314b3.avif" alt="Background map management" />

After mapping, click **Map management**. The new map appears in the background-map list. Click **Deploy** to deploy it to the host. Deploy every map when multiple maps exist. You can also edit, export, or delete maps.

3. Host maps

<Frame>
  <img src="/images/legacy/22d53108-6c4c-4352-afde-cea020bcf3dc/60add4a5-7fb8-403b-89fa-52c52a02ac2f.avif" alt="Host map management" />
</Frame>

Click **Apply** on a host map to activate it. You can export or delete host maps. Restart the controller after clicking **Apply**; otherwise, the map does not become active and relocalization can fail.

#### 5.6 Relocalization

<Frame>
  <img src="/images/legacy/a2c8e5ae-7696-4335-aac3-d104f19e5661/deb0504b-1722-4266-9085-373b38eaf7b1.avif" alt="Relocalization" />
</Frame>

- Click **Relocalize**, then drag with the mouse to provide an approximate pose by matching the laser point cloud to the laser outline.

- After relocalization succeeds, the localization timeout error disappears.

## 6 Troubleshooting

| No. | Problem | Resolution |
| --- | --- | --- |
| 1 | Camera does not respond when opened | Disable the firewall so it does not block camera communication |
| 2 | Host software detects multiple IP addresses | Resolve the LAN address conflict and select the correct IP address |
| 3 | Camera stream is unstable | Use a Gigabit Ethernet cable; Fast Ethernet can cause initial instability |
| 4 | Software installation location | Use a directory with suitable read and write permissions instead of the default drive C location |

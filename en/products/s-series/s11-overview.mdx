## 1 Product overview

### 1.1 Introduction

The MRDVS S11 is a solid-state RGB-D camera based on dToF technology for mobile robots, with Ethernet and USB versions. Its typical outdoor range is 0.1-5m\[1\], with ranging accuracy up to 1cm. It synchronously outputs depth data, RGB images, and IR images for obstacle avoidance, spatial perception, and modeling. A custom MIPI module has the same performance specifications but different communication, dimensions, and power requirements.

Note \[1\]: The maximum indoor range is 10m.

### 1.2 Features

- dToF depth sensing: Calculates distance from the round-trip time of a laser signal. Compared with iToF, it provides higher accuracy (≤1cm), a wider effective range (0.1-10m), and stronger outdoor ambient-light resistance.

- Synchronized data output: Outputs depth, RGB, and IR amplitude images without acquisition delay.

- Solid-state design: Contains no moving parts. The 90×25×25mm camera weighs approximately 130g with Ethernet or 90g with USB and resists vibration. A custom 33×18×13mm MIPI module is also available.

### 1.3 Applications

1.  Cleaning robots for floor-obstacle detection

2.  Industrial mobile robots (AGV/AMR)

3.  Service robots for retail guidance and delivery

4.  Home companion robots for environment perception

5.  Special-purpose robots for cold or humid environments

### 4.2 Important installation precautions

#### 4.2.1 Recommended installation

Keep the illustrated clearance area completely free of objects when installing the 3D vision camera. An object in this area interferes with point cloud data and can cause image noise, false stops, and data errors.

</Frame>

#### 4.2.2 Causes of camera errors

An unsuitable installation environment can interfere with time-of-flight and triangulation measurements and cause operational errors:

1. Improper hardware installation: physical obstruction and FOV clipping

Enclosure edges or ribs can enter the specified FOV. Off-center installation or an opening that is too small can also clip the view and cause the sensor to detect the enclosure as a target. Symptoms include fixed blind spots, false short-range measurements, and a compressed FOV, which can cause missed obstacles and incorrect paths.

2. Optical interference: near-field infrared crosstalk and reflections

This common failure occurs when a confined installation space causes abnormal infrared reflections and crosstalk:

First, reflections from inner walls cause multipath interference. Infrared light reflects from enclosure walls multiple times before reaching the receiver, changing the measured time and distorting the calculated distance.

Second, highly reflective surfaces cause local failures. Smooth, bright materials such as polished metal and mirrors create specular reflections, causing missing signals or saturated overexposure.

Third, strong reflections from nearby structures cause glare contamination. When a bright, smooth structure is less than 10cm from the sensor, strong reflections saturate the receiver. Glare disrupts an iToF camera's phase and prevents flight-time calculation. This interference distorts data, accelerates receiver aging, and affects core functions.

3. Optical interference: optical windows

The material and cleanliness of protective glass or other optical windows in front of the sensor can cause three types of interference:

First, multiple reflections cause image artifacts. Reflections from both window surfaces mix with the valid signal, causing ghosting and flying pixels in the depth image and duplicate LiDAR point clouds.

Second, contamination and fog reduce the signal-to-noise ratio. Sealed spaces can allow the window to fog or collect dust, preventing the sensor from identifying valid signals.

Third, incompatible materials attenuate the signal. Signals may not pass through ordinary glass or acrylic, causing complete data loss. Window interference increases measurement error, can damage the lens, and raises maintenance costs. High-transmission glass can improve performance.

4. Other indirect factors

First, installation stress: Over-tightened clips can deform the lens or shift the optical axis, causing ToF distance errors.

Second, thermal interference: Poor heat dissipation in a sealed space raises sensor temperature and deforms the window, causing range drift and possible long-term failure.

## 5 Quickstart

### 5.1 Connection test

  <img src="/images/legacy/45ef596f-1507-4062-bb8f-480af2742ac6/b2df4d53-b802-4387-b1ca-b8e45f0a5ac5.avif" alt="Connection test" />

#### 5.1.1 Power the camera

1.  Connect one end of the DC cable to the camera and the other to a 24V/2A DC adapter.

2.  A slowly flashing blue power indicator confirms normal power-on.

3.  If the indicator is off or abnormal, such as solid red, check the power connector and adapter.

#### 5.1.2 Configure the network

1.  Connect the camera directly to the computer with a Category 6 Ethernet cable.

2.  The factory-default camera IP address is 192.168.100.82.

3.  Set the computer IP to the same subnet and disable the firewall.

#### 5.1.3 Run the software

1.  Run the host software, acquire camera images, and verify that they are clear and stable.

2.  See section 5.3 or the LxCameraViewer user guide.

3.  Contact MRDVS sales or technical support for further help.

### 5.2 Interface overview

LxCameraViewer opens and displays multiple devices. It displays depth, intensity, point cloud, and RGB images and configures basic camera and algorithm parameters. You can view and save image data, camera configuration, and algorithm details.

The initial interface has a device list (blue area 1), menu bar (green area 2), and image display (red area 4). Double-click the menu bar to maximize or minimize it. Drag the interface with the left mouse button.

After opening a camera, a function panel appears on the right (area 3). Use **Basic tools**, **Application algorithms**, and **Other** at the top. Basic tools include image display, 2D settings, 3D settings, and filtering.

The device list is in the upper left. Click **Open** or **Close** to control the camera. The center-left area shows camera information, software operations, results, and stream results. The lower-left area shows information for the selected device.

Click **Capture** to start or stop image acquisition. The button remains disabled until the operation finishes. Click **Save frame** to save images from active streams. Select **Depth**, **Intensity**, **RGB**, or **Point cloud** to open the required views.

For details, see the LxCameraViewer user guide.

## 6 Troubleshooting

| No. | Problem | Resolution |
| --- | --- | --- |
| 1 | Camera does not respond when opened | Disable the firewall so it does not block camera communication |
| 2 | Host software detects multiple IP addresses | Resolve the LAN address conflict and select the correct IP address |
| 3 | Camera stream is unstable | Use a Gigabit Ethernet cable; Fast Ethernet can cause initial instability |
| 4 | Software installation location | Use a directory with suitable read and write permissions instead of the default drive C location |

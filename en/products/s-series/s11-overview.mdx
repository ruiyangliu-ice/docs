## Product overview

The S11 is a solid-state dToF RGB-D camera for mobile robots. Ethernet and USB versions are available, and a MIPI module can be supplied on request. The camera synchronously outputs depth, RGB, and IR amplitude data for obstacle avoidance, spatial perception, and mapping.

The typical outdoor range is 0.1–5 m at 10%–90% reflectivity; the maximum indoor range is 10 m. The MIPI module uses the same depth technology as the standard versions but has different interfaces, dimensions, and power requirements.

## Features and applications

- dToF depth sensing for 3D data capture indoors and outdoors.
- Synchronized depth, RGB, and IR amplitude output; Ethernet and USB versions support RGB-D alignment.
- Compact solid-state design: approximately 130 g for Ethernet and 90 g for USB.
- Suitable for cleaning robots, AGVs/AMRs, service robots, companion robots, and special-purpose robots.

### Installation requirements

<Warning>
Keep the illustrated clearance area free of structures and obstructions. Components that enter this area can interfere with point clouds and cause noise, false detections, or incorrect stops.
</Warning>

<Frame>
  <img src="/images/products/s-series-s11-overview-recommended-installation.avif" alt="S11 recommended installation and clearance area" />
</Frame>

- Avoid frames, ribs, or openings that are too small and clip the field of view.
- Avoid installation near mirrors, bright metal, or confined reflective cavities to reduce near-field reflections and multipath effects.
- If a protective window is required, validate its transmission, cleanliness, and ranging performance after installation.
- Avoid over-tightening the mount and provide a heat path in enclosed spaces.

## First connection

<Steps>
  <Step title="Connect the device">
    Power the Ethernet version with a 24 VDC / 2 A supply and connect it to the computer with a Cat 6 or better Ethernet cable. A slowly flashing blue indicator indicates normal power.
  </Step>
  <Step title="Configure the network">
    The Ethernet version's factory-default IP address is `192.168.100.82`. Set the computer to the same subnet and make sure the firewall does not block device communication.
  </Step>
  <Step title="Verify the stream">
    Open the camera in the host application and check that the image is clear and free of artifacts or stutter. Configure image and algorithm parameters as needed.
  </Step>
</Steps>

## LxCameraViewer interface

The host application displays depth, IR amplitude, point-cloud, and RGB images and configures basic camera and algorithm parameters.

- Use **Capture** to start or stop streaming and **Save frame** to save active streams.

## Troubleshooting and safety

| Problem | Suggested checks |
| --- | --- |
| The camera does not respond when opened | Check the firewall and confirm that the computer and camera are on the same subnet. |
| The host application detects multiple IP addresses | Resolve LAN IP conflicts and select the correct device address. |
| The stream is unstable | Use a Gigabit Ethernet cable; Fast Ethernet may make initial data transfer unstable. |
| The software does not start correctly | Check permissions for the installation directory. |

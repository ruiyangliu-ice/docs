## Product overview

### Purpose and scope

This document describes the S10 camera specifications and selected design details for developers who integrate and use the product.

### Terminology

| RGB-D camera | A camera that outputs synchronized and spatially aligned RGB and depth images |
| --- | --- |
| Depth | A video stream similar to a color or monochrome stream, where each pixel represents its distance from the camera |
| FOV | Field of view: the angular range of a scene captured by a camera, including horizontal (HFOV), vertical (VFOV), and diagonal (DFOV) fields of view |
| IR camera | An infrared camera; in this product, it primarily captures the near-infrared band |
| RGB camera | A color camera |
| Depth camera | A camera whose depth module typically includes a dot projector, an IR camera, and a depth processor; its color module is typically an RGB camera |
| SoC | System on a chip used for image processing, depth calculation, and application algorithms |
| dToF | Direct time-of-flight depth-sensing technology |

### Operating principle

This product is a dToF 3D depth camera with long range, good accuracy, strong interference resistance, and stable data. An integrated RGB module captures color images. The camera aligns RGB and depth images internally.

### Product details

This compact industrial RGB-D camera provides depth data from 0.3m to 8m. It spatially and temporally aligns RGB and depth images. Its integrated SoC outputs depth, intensity, RGB, point cloud, and obstacle-detection results. In addition to Ethernet, two IO inputs and two IO outputs switch among four obstacle-avoidance zones and output emergency-stop, slowdown, and safe-state signals.

## First use

### Connection test

1.  Connect the camera to a 24V/2A DC supply. A slowly flashing blue power indicator confirms normal power-on.

2.  Connect the camera to the computer with a Category 6 Ethernet cable.

3.  The factory-default camera IP address is 192.168.100.82.

4.  Set the computer IP to the camera's subnet and disable the computer firewall.

5.  Run the host software to view the camera images.

6.  Contact MRDVS sales or technical support to obtain the host software.

### I/O wiring

| Interface | Type | Notes |
| --- | --- | --- |
| IN1 | Input | Active-high discrete input; supports 24V/open input |

| OUT2 | Output | Ground/open discrete output; supports 24V pull-up |
| COM | GND | Common ground for discrete outputs; use IN1 with IN_COM1 |

### Troubleshooting

| No. | Problem | Resolution |
| --- | --- | --- |
| 1 | Camera does not respond when opened | Verify that the firewall is disabled |
| 2 | Host software detects multiple IP addresses | Another device on the LAN uses the same IP; select the required IP address |
| 3 | Camera data is unstable after opening | Use a Gigabit Ethernet cable; data can initially be unstable over Fast Ethernet |
| 4 | Software installation location | A default installation on drive C may be affected by permission settings |

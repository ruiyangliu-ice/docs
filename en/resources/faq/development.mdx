---
title: "Development FAQs"
description: "Troubleshoot Camera SDK error codes, device connections, streaming, and development environments."
---

Error codes help locate SDK, connection, and data-stream problems. Record the error code, SDK version, camera model, and device log before troubleshooting.

<Warning>
  Do not troubleshoot by changing only the SDK version. Also confirm compatibility between the camera firmware, data-processing library, and development environment.
</Warning>

## SDK error codes and development environment

<AccordionGroup>
<Accordion title="LX_ERROR = -1">
**Possible cause**
Unknown error.
**Resolution**
Export the device log and send it to MRDVS technical support.
</Accordion>

<Accordion title="LX_E_NOT_SUPPORT = -2">
**Possible cause**
The function is not supported.
**Resolution**
The current device does not support this feature.
</Accordion>

<Accordion title="LX_E_NETWORK_ERROR = -3">
**Possible cause**
Network communication error.
**Resolution**
Check the IP configuration, network cables, switch, and packet loss.
</Accordion>

<Accordion title="LX_E_INPUT_ILLEGAL = -4">
**Possible cause**
An input parameter is invalid.
**Resolution**
Check the parameters passed to the interface.
</Accordion>

<Accordion title="LX_E_RECONNECTING = -5">
**Possible cause**
The device is reconnecting.
**Resolution**
1. Wait briefly and retry until the connection is restored.<br />2. Check that the camera is powered on and the network connection is normal.
</Accordion>

<Accordion title="LX_E_DEVICE_ERROR = -6">
**Possible cause**
The device failed or did not respond.
**Resolution**
Restart the application or camera and try again.
</Accordion>

<Accordion title="LX_E_DEVICE_NEED_UPDATE = -7">
**Possible cause**
The device version is too old.
**Resolution**
Upgrade the firmware.
</Accordion>

<Accordion title="LX_E_API_NEED_UPDATE = -8">
**Possible cause**
The API version is too old.
**Resolution**
Update the SDK.
</Accordion>

<Accordion title="LX_E_CTRL_PERMISS_ERROR = -9">
**Possible cause**
Exclusive-control permission could not be obtained.
**Resolution**
Only one primary client can control a camera. If the camera was not shut down normally, wait `3–5 s` for the permission to be released.
</Accordion>

<Accordion title="LX_E_GET_DEVICEINFO_ERROR = -10">
**Possible cause**
Device information could not be read.
**Resolution**
Check the device firmware version or contact MRDVS support.
</Accordion>

<Accordion title="LX_E_IMAGE_SIZE_ERROR = -11">
**Possible cause**
The requested image size does not match the stream.
**Resolution**
Close the device and open it again.
</Accordion>

<Accordion title="LX_E_IMAGE_PARTITION_ERROR = -12">
**Possible cause**
Image parsing failed, often because the pixel format is incorrect.
**Resolution**
Open the device again.
</Accordion>

<Accordion title="LX_E_DEVICE_NOT_CONNECTED = -13">
**Possible cause**
The camera is not connected.
**Resolution**
The application attempted to use a camera that was not open. Check the call order and confirm that the camera opened successfully.
</Accordion>

<Accordion title="LX_E_DEVICE_INIT_FAILED = -14">
**Possible cause**
Camera initialization failed.
**Resolution**
Open the camera again.
</Accordion>

<Accordion title="LX_E_DEVICE_NOT_FOUND = -15">
**Possible cause**
No camera, or no matching camera, was found.
**Resolution**
Check that the camera handle passed to the interface is valid.
</Accordion>

<Accordion title="LX_E_FILE_INVALID = -16">
**Possible cause**
The file name, type, format, or path is invalid, or the file could not be opened.
**Resolution**
Check that the path is correct and that the file can be opened manually.
</Accordion>

<Accordion title="LX_E_CRC_CHECK_FAILED = -17">
**Possible cause**
CRC or MD5 verification failed during upload or download.
**Resolution**
Retry the transfer.
</Accordion>

<Accordion title="LX_E_TIME_OUT = -18">
**Possible cause**
The request timed out.
**Resolution**
Retry. Packet loss or a low camera frame rate may be responsible.
</Accordion>

<Accordion title="LX_E_FRAME_LOSS = -19">
**Possible cause**
Frames were lost, often in callback mode.
**Resolution**
Occasional loss may not require action. For frequent loss, check network bandwidth and buffering.
</Accordion>

<Accordion title="LX_E_ENABLE_ANYSTREAM_FAILED = -20">
**Possible cause**
The requested stream configuration could not be enabled.
**Resolution**
Versions before `2.4.9` did not allow all streams to be disabled. This error code is obsolete in later versions.
</Accordion>

<Accordion title="LX_E_NOT_RECEIVE_STREAM = -21">
**Possible cause**
No stream data was received.
**Resolution**
Use the camera-open troubleshooting steps for a missing data stream.
</Accordion>

<Accordion title="LX_E_PARSE_STREAM_FAILED = -22">
**Possible cause**
The stream started but could not be parsed.
**Resolution**
Check the firmware version or contact MRDVS support.
</Accordion>

<Accordion title="LX_E_PROCESS_IMAGE_FAILED = -23">
**Possible cause**
Image processing failed.
**Resolution**
Confirm that the `Dataprocess` image-processing library version is compatible.
</Accordion>

<Accordion title="LX_E_SETTING_NOT_ALLOWED = -24">
**Possible cause**
The requested setting is not allowed in continuous mode.
**Resolution**
In continuous mode, stop the stream before changing the ROI, multi-camera mode, alignment, or other stream-related settings. Set the camera mode first, then apply the setting.
</Accordion>

<Accordion title="LX_E_LOAD_DATAPROCESSLIB_ERROR = -25">
**Possible cause**
The image-processing library could not be loaded.
**Resolution**
Ensure that the `Dataprocess` dynamic library/shared object exists and is included in the library path. Run `source install.sh`.
</Accordion>

<Accordion title="LX_E_FUNCTION_CALL_LOGIC_ERROR = -26">
**Possible cause**
The function call order or prerequisite logic is invalid.
**Resolution**
Check the SDK log and adjust the call sequence. Common cases include:<br />1. Stop the stream before changing a setting that changes the image size.<br />2. Do not change image size in continuous mode.<br />3. Algorithm upload and built-in application algorithms are mutually exclusive.<br />4. Do not enable auto exposure when HDR is enabled.<br />5. Auto-exposure parameters depend on the auto-exposure state.<br />6. Trigger parameters depend on the trigger mode.
</Accordion>

<Accordion title="LX_E_IPAPPDR_UNREACHABLE_ERROR = -27">
**Possible cause**
The IP address is unreachable or the network is misconfigured.
**Resolution**
Confirm that the SDK host can ping the camera.
</Accordion>

<Accordion title="LX_E_FRAME_ID_NOT_MATCH = -28">
**Possible cause**
RGB and ToF frames were not synchronized within the timeout window.
**Resolution**
When RGB-D synchronization is enabled, choose whether to discard the frame or ignore the warning based on whether synchronization or real-time data is more important for your application.
</Accordion>

<Accordion title="LX_E_FRAME_MULTI_MACHINE = -29">
**Possible cause**
The frame contains an interference signal from another camera.
**Resolution**
Based on the application, discard the frame or ignore the warning.
</Accordion>

<Accordion title="Error `-25` or `LxDataProcess` fails to load">
**Possible cause**
On Linux, the library is not included in the environment path.
**Resolution**
Run `source install.sh`.
</Accordion>

<Accordion title="The captured image is divided into three sections">
**Possible cause**
The displayed image aspect ratio is incorrect.
</Accordion>

<Accordion title="ROS 2 conflicts with the SDK library">
**Resolution**
Use the wrapper in the `util` directory. It links the camera SDK indirectly and avoids a direct library conflict.
</Accordion>

<Accordion title="How can I identify a Lite camera versus a standard camera with RGB?">
**Resolution**
Call `DcGetBoolValue(handle, LX_BOOL_ENABLE_2D_STREAM, &test_rgb)`. A Lite camera returns that the feature is not supported.
</Accordion>

<Accordion title="How is time synchronization handled?">
**Resolution**
The SDK synchronizes with PTP automatically on each interface call. No additional trigger is required.
</Accordion>
</AccordionGroup>

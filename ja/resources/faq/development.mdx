＃＃ よくある質問

<AccordionGroup>
<Accordion title="LX_ERROR = -1">

**考えられる理由**

不明なエラー

**解決**

結果は異常であるため、デバイス ログを取得して MRDVS に送信して処理する必要があります。

</Accordion>

<Accordion title="LX_E_NOT_SUPPORT = -2">

**考えられる理由**

サポートされていない機能

**解決**

現在のデバイスはこの機能をサポートしていません

</Accordion>

<Accordion title="LX_E_NETWORK_ERROR = -3">

**考えられる理由**

ネットワーク通信エラー

**解決**

ネットワーク通信の障害、信号伝送のパケット損失などの問題、IP 構成の確認、ネットワーク ケーブルとスイッチが正常かどうかの確認、パケット損失の問題のトラブルシューティング

</Accordion>

<Accordion title="LX_E_INPUT_ILLEGAL = -4">

**考えられる理由**

高麗人参の摂取は違法

**解決**

渡されたパラメータが間違っています。呼び出し側インターフェイスのパラメータを確認する必要があります。

</Accordion>

<Accordion title="LX_E_RECONNECTING = -5">

**考えられる理由**

デバイスの再接続中

**解決**

1. デバイスは再接続中です。少し待って、再接続が成功するまで再試行してください<br />2。カメラの電源が正常にオンになっているか、ネットワーク接続が正常であるかを確認してください。

</Accordion>

<Accordion title="LX_E_DEVICE_ERROR = -6">

**考えられる理由**

デバイス障害またはデバイス応答障害

**解決**

デバイスに異常があります。プログラムまたはカメラを再起動して、もう一度試してください。

</Accordion>

<Accordion title="LX_E_DEVICE_NEED_UPDATE = -7">

**考えられる理由**

デバイスのバージョンが低すぎるため、アップグレードする必要があります。

**解決**

ファームウェアのバージョンをアップグレードする

</Accordion>

<Accordion title="LX_E_API_NEED_UPDATE = -8">

**考えられる理由**

APIのバージョンが低すぎる

**解決**

SDKのアップデート

</Accordion>

<Accordion title="LX_E_CTRL_PERMISS_ERROR = -9">

**考えられる理由**

排他制御許可に失敗しました

**解決**

カメラは 1 つの主認証局クライアント接続のみをサポートします。カメラが異常終了した場合、許可が解除されるまで 3 ～ 5 秒待つ必要があります。

</Accordion>

<Accordion title="LX_E_GET_DEVICEINFO_ERROR = -10">

**考えられる理由**

機器情報の取得に失敗しました

**解決**

デバイスのファームウェアのバージョンを確認するか、MRDVS サポートに連絡してください

</Accordion>

<Accordion title="LX_E_IMAGE_SIZE_ERROR = -11">

**考えられる理由**

画像のサイズが一致しないため、再度開く必要があります

**解決**

取得した画像のサイズが実際のコードストリームのサイズと一致しません。デバイスを閉じて、再度開く必要があります。

</Accordion>

<Accordion title="LX_E_IMAGE_PARTITION_ERROR = -12">

**考えられる理由**

画像の解析に失敗しました (pixformat が正しくありません)。もう一度開いてみる必要があります

**解決**

デバイスの電源を再度オンにします

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_CONNECTED = -13">

**考えられる理由**

カメラが接続されていません

**解決**

不正なタイプのパラメータ入力です。カメラの電源が入っていない状態でカメラを操作すると、このエラーが返されます。呼び出しコードのロジックをチェックして、カメラが正常にオンになっていることを確認する必要があります。

</Accordion>

<Accordion title="LX_E_DEVICE_INIT_FAILED = -14">

**考えられる理由**

カメラの初期化に失敗しました

**解決**

カメラを再びオンにします

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_FOUND = -15">

**考えられる理由**

カメラが見つかりません (または一致するカメラが見つかりません)

**解決**

不正な入力パラメータの一種。このエラー コードは通常、インターフェイスに渡されたカメラのハンドルが不正な場合に返されます。呼び出しコードのロジックを確認する必要があります。

</Accordion>

<Accordion title="LX_E_FILE_INVALID = -16">

**考えられる理由**

ファイルエラー（ファイル名、タイプ、形式が間違っているか、ファイルを開けません）

**解決**

ファイルのパス/タイプが間違っている場合、またはファイルが存在しないか開けない場合は、受信パラメータ ファイルのパスを手動で開けるかどうかを確認する必要があります。

</Accordion>

<Accordion title="LX_E_CRC_CHECK_FAILED = -17">

**考えられる理由**

ファイルの crc または md5 チェックに失敗しました

**解決**

ファイルのアップロードまたはダウンロード時に crc または md5 検証に失敗しました。再試行する必要があります

</Accordion>

<Accordion title="LX_E_TIME_OUT = -18">

**考えられる理由**

タイムアウト

**解決**

もう一度やり直してください。パケット損失が多いか、カメラのフレーム レートが低すぎるなどのネットワーク上の理由が考えられます。

</Accordion>

<Accordion title="LX_E_FRAME_LOSS = -19">

**考えられる理由**

フレームがありません

**解決**

コールバック モードでは、パケット損失とフレーム損失がよく発生します。少数のケースでは、特別な処理は必要ありません。多くの場合、ネットワーク帯域幅、キャッシュなどをチェックする必要があります。

</Accordion>

<Accordion title="LX_E_ENABLE_ANYSTREAM_FAILED = -20">

**考えられる理由**

ストリームを開けませんでした

**解決**

バージョン 2.4.9 より前では、すべてのストリーム データを閉じることはできません。このエラーコードは、以降のバージョンでは廃止されました。

</Accordion>

<Accordion title="LX_E_NOT_RECEIVE_STREAM = -21">

**考えられる理由**

ストリーミングデータを受信できませんでした

**解決**

コード ストリーム データを受信しません。参照カメラが開いていますが、データがありません。

</Accordion>

<Accordion title="LX_E_PARSE_STREAM_FAILED = -22">

**考えられる理由**

ストリームは正常に開始されましたが、ストリーム データの解析に失敗しました。

**解決**

コードストリームの解析に失敗しました。ファームウェアのバージョンを確認するか、MRDVS サポートにお問い合わせください。

</Accordion>

<Accordion title="LX_E_PROCESS_IMAGE_FAILED = -23">

**考えられる理由**

画像計算処理に失敗しました

**解決**

画像アルゴリズムライブラリ「Dataprocess」のバージョン確認

</Accordion>

<Accordion title="LX_E_SETTING_NOT_ALLOWED = -24">

**考えられる理由**

ノーマルオープンモードでは設定不可

**解決**

作業モードであるノーマリーオープンモードでは、ROI の設定や複数のカメラ/位置合わせの設定などの操作は許可されません。呼び出しロジックを調整する必要があります。カメラ動作モードを設定してから設定操作を行ってください。

</Accordion>

<Accordion title="LX_E_LOAD_DATAPROCESSLIB_ERROR = -25">

**考えられる理由**

画像処理アルゴリズム ライブラリのロード中にエラーが発生しました

**解決**

イメージ アルゴリズム ライブラリ Dataprocess (ダイナミック リンク ライブラリ/共有オブジェクト ファイル) が存在し、ライブラリによってロードされる環境変数に含まれていることを確認してください。実行：source install.sh

</Accordion>

<Accordion title="LX_E_FUNCTION_CALL_LOGIC_ERROR = -26">

**考えられる理由**

関数呼び出しロジックエラー

**解決**

関数の呼び出し順序やプリロジックが間違っている場合は、SDKのログに従ってエラーを報告し、呼び出しロジックを調整する必要があります。一般的なロジックの問題は次のとおりです:<br />1。パラメータを設定すると画像サイズが変更されるため、最初にフローを停止する必要があります;<br />2。ノーマリーオンモードでは、画像サイズの変更は許可されません;<br />3。アルゴリズムを上に移動することと、組み込みアプリケーション アルゴリズム <br />4 をオンにすることは相互に排他的です。 HDR が有効になっている場合、自動露出をオンにすることはできません;<br />5。自動露出関連パラメータは、自動光爆発状態に関連します;<br /> />6.トリガー関連パラメータはトリガーモードに関連します

</Accordion>

<Accordion title="LX_E_IPAPPDR_UNREACHABLE_ERROR = -27">

**考えられる理由**

IP に到達できない、またはネットワーク構成エラー

**解決**

ネットワークが利用できません。 SDK ホストがビジュアル デバイスに ping できることを確認する必要があります。

</Accordion>

<Accordion title="LX_E_FRAME_ID_NOT_MATCH = -28">

**考えられる理由**

タイムアウト範囲内のフレーム同期エラー

**解決**

RGBD フレーム同期機能がオンになっている場合、実際のコード ストリームで tof データ フレームと RGB データ フレームが同期していない場合にこのエラーが報告されます。実際の使用シナリオ (同期の優先度が高いか、データのリアルタイムの優先度が高い) に応じて、フレーム データを破棄するかエラーを無視するかを選択できます。

</Accordion>

<Accordion title="LX_E_FRAME_MULTI_MACHINE = -29">

**考えられる理由**

フレーム内で複数のマシンの干渉信号が検出されました

**解決**

実際の使用シナリオに応じて、フレーム データを破棄するか、エラー レポートを無視するかを選択します。

</Accordion>

<Accordion title="エラー コード: -25 または LxDataProcess の読み込み失敗">

**考えられる理由**

(Linux オペレーティング システム環境にのみ適用) アルゴリズム ライブラリが環境変数に設定されていない

**解決**

ソースのinstall.shを実行する

</Accordion>

<Accordion title="ストリーミング画像の 3 分の 3 を取得します">

**考えられる理由**

画像取得後、表示されるアスペクト比が異常です。

</Accordion>

<Accordion title="ros2 は SDK ライブラリと競合します">

**解決**

カメラ SDK は util フォルダーにカプセル化されており、カメラ SDK ライブラリに間接的にリンクされます。

</Accordion>

<Accordion title="カメラが Lite バージョンであるか、ソフトウェア レベルで RGB を備えた標準バージョンであるかを判断する方法">

**解決**

DcGetBoolValue(handle, LX_BOOL_ENABLE_2D_STREAM, &test_rgb) カメラが Lite バージョンの場合、サポートされていないことを返します。

</Accordion>

<Accordion title="時刻同期方法">

**解決**

SDKとPTPは同期されています。各インターフェイス呼び出しは、追加のトリガーなしで自動的に同期されます。

</Accordion>

</AccordionGroup>

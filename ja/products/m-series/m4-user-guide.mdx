## 1. 製品概要

このガイドでは、Zhejiang MRDVS Technology Co., Ltd. が開発した M4 カメラの仕様、特徴、操作手順について説明します。

M4 製品ファミリーには、M4 Lite および M4 カメラが含まれます。これらの ToF 製品は、ギガビット イーサネット通信インターフェイスと 24VDC 電源を使用します。

![M4カメラの概要](/images/legacy/e882097c-c641-4cc8-8d4a-1e3856985f4c/1752f961-5a67-4a8a-b7d1-3bd19dae0f11.avif) 

図1.1。 M4カメラ

### 1.1 製品の特徴

- 内蔵 1Tops コンピューティング能力により、外部 CPU なしで動作可能

- RGB-Dのサポート

- スタンドアロンの障害物回避、ドッキング、位置特定、ソフトパッケージの把握、保管場所の検出、体積測定

- ギガビットイーサネットポート

### 1.2 利用規約

**使用中は次の要件に従ってください:**

・製品のラベルは剥がさないでください。

・製品を分解しないでください。分解すると破損する恐れがあります。

- 強い磁界の中で製品を使用しないでください。

- 100KLUXを超える強い太陽光の下で製品を使用しないでください。

- 筐体を清掃するには、精製水またはアルコールで軽く湿らせた、糸くずの出ない柔らかい布で拭きます。

- 元の梱包材を保管し、輸送中は可能な限り使用してください。

## 2. 製品仕様

### 2.1 距離測定原理

M4 カメラはタイムオブフライト (ToF) 測距を使用します。光を発してから物体からの反射を受け取るまでの時間を測定し、物体とカメラの間の距離を計算します。

M4 カメラは深度を直接キャプチャし、最小限のコンピューティング リソースで高品質の深度情報を提供し、高フレーム レートで深度シーンを構築します。 LiDAR と同様に、ToF カメラはパルス ToF (P\_ToF) または連続波変調 ToF (CW\_ToF) を使用できます。 P\_ToF は、通常は目に見えない光パルスを放射し、その往復時間から距離を計算します。 CW\_ToF は、変調された連続光を放射し、放射された光と戻ってきた光の位相差から距離を計算します。

## 4. 精度と精度

### 4.1 精度

キャリブレーションでは、センサーの測定値と実際の値を相互にマッピングすることで、それらの間の固定オフセットを除去します。 M4 カメラをキャリブレーションするには、レールに取り付けて白い壁に向けます。図 4.1 に示すように、各ピクセルの測定値をカメラと対応する壁領域の間の距離にマッピングし、このマッピングを使用して測定値を補正します。

<Frame>
  <img src="/images/legacy/f56d3e54-eb77-4bdb-9576-65f9aeb7a628/67a4d596-b169-43c0-93b0-dfa2ef85c421.avif" alt="M4 キャリブレーションのセットアップ" />
</Frame>

図4.1。較正

M4 はアクティブ レンジングを使用するため、多くの要因がその精度に影響します。不適切に設置すると、マルチパス効果が生じ、測定が歪む可能性があります。

これは、反射率の高い対象物の横にある低反射率の対象物を測定する場合に特に顕著です。理想的には、光は青い経路をたどります。壁から反射した光もターゲットに到達し、緑のパスに沿ってカメラに戻る場合があります。隣接する高反射率の物体から直接反射された照明が、赤い経路に沿ってカメラに入射する可能性があります。図 4.2 に示すように、光学的欠陥によりこの光がグレアに変わり、測距の妨げになります。

  <img src="/images/legacy/be0eb189-bf98-4c78-b2dc-3a4e8a356124/04671b63-d54f-4cac-8774-4afb08cccbc2.avif" alt="M4の設置と反射経路" />
</Frame>

図4.2。カメラの設置

### 4.2 精度

精度は通常、単一点の変動 (1-σ) を指します。光電子測定デバイスとして、M4 は ADC 変換、信号対雑音比、およびシステム ノイズの影響を受けます。図 4.3 に測定精度と信号強度の関係を示します。

<Frame>
  <img src="/images/legacy/16312574-c0be-48c1-81b8-0cfc40ff06ae/74264eea-5a19-48a4-8520-84410a78262c.avif" alt="測定精度と信号強度" />
</Frame>

図4.3。測定精度と信号強度

M4 は、光電子アナログ信号をデジタル データに変換します。 ADC 変換中、アナログ値は最小の分解能ステップに丸められるため、ノイズが発生します。 1.1V 信号は 1V として表される場合があり、10% の誤差が生じます。 10.1V の信号が 10V になる場合や、100.1V の信号が 100V になる場合があり、それぞれ約 1% の誤差が生じます。したがって、ADC 変換では、信号レベルが低い場合、より多くのノイズとエラーが発生します。 ToF カメラの S/N 比は通常、SNR∝1/√t に従います。ここで、`t` は積分時間です。積分時間を長くすると、信号強度が増加します。

## 5. マルチカメラの干渉

### 5.1 平行干渉

2 台のカメラが並列に設置されている場合、それぞれのカメラが自身の反射照明と他のカメラからの照明の両方を受け取り、深度データに干渉する可能性があります。 2 台の M4 カメラが同じ方向を向いている場合、平行干渉により測定誤差が増加します。

<Frame>
  <img src="/images/legacy/af686e8c-9957-4707-a2e4-1454fccc29b5/36fa2e7f-5002-4f7d-941c-841fa869060d.avif" alt="平行カメラ干渉" />
</Frame>

平行干渉を避けるため、カメラを水平に対して斜めに設置してください。水平面に対して 0° で設置する必要がある場合は、カスタム構成について MRDVS にお問い合わせください。

## 6. ソフトウェアの概要

### 6.1 SDK

SDK を使用してカメラを初期化して起動し、データを取得し、カメラを閉じます。 Linux または Windows 以外のプラットフォームの場合は、SDK のソース コードを参照してアプリケーションを移植します。

表6-1。 SDKの詳細

|オペレーティング システム | Linux&Windows |
| --- | --- |
|プログラミング言語 | C/C++ |
|データ形式 | 16 ビットの符号なし short |
|統合 |共有ライブラリ (.so)、ダイナミックリンク ライブラリ (.dll)、または統合 SDK ソース コード |

### 6.2 SDKの出力順序

![SDKの出力順序](/images/legacy/c475e8c8-74c3-4950-bec1-a5ba15f8d9cf/4bcb3dbe-c3e6-47a5-a08f-00691b16a489.avif) 

図6.1。 SDKの出力順序

SDK は、設定された解像度で深度と振幅データをそれぞれ 1 フレームずつ出力します。各フレームは、左上のピクセルから始まり、左から右、上から下の順に進む 1 次元配列として保存されます。図 6.1 は、x 軸に沿って出力を示し、その後に y 軸が続き、次の順序で生成されます。

(x,y):(0,0)(0,1)(0,2)(0,…)(1,0)(1,1)(1,2)(1,…)(2,…)(3,…)(…,…).

深度画像の各ピクセルは、測定対象物からカメラ平面までの距離を表す 16 ビットの符号なし short です。

各振幅画像ピクセルは、返された信号強度を表す 16 ビットの符号なし short です。

### 6.3 IPアドレス

M4 カメラのデフォルトの IP アドレスは 192.168.100.82 です。 IP アドレスを変更するか、DHCP を有効にするには、MRDVS が提供するデモ アプリケーションを使用します。

### 6.4 デモアプリケーション

### 6.4.1 IP 構成

電源ケーブルを接続し、カメラとパソコンをイーサネットケーブルで接続します。カメラの工場出荷時のデフォルト IP アドレスは 192.168.100.82 であるため、コンピュータを 100.82 を除く 192.168.100.x サブネット、サブネット マスク 255.255.255.0 に設定します。 **設定** → **イーサネット** → **アダプター オプションの変更** を開きます。 **イーサネット**を右クリックし、**プロパティ**を選択し、次に**インターネット プロトコル バージョン 4 (TCP/IPv4)**を選択します。たとえば、IP アドレスを 192.168.100.20 に、サブネット マスクを 255.255.255.0 に設定します。

### 6.4.2 アプリケーションを開く

ソフトウェアをダウンロードしてインストールし、LxCameraViwer デモ アイコンをダブルクリックします。ホスト アプリケーションが開き、カメラを検索して開き、1 台のカメラが接続されると自動的に取得を開始します。

**注: 一部のシステムでは、最初の起動時に、アプリケーションにファイアウォールの通過を許可する必要があります。**

アプリケーションには、基本的なコントロール、画像の選択、カメラの設定、およびカメラ情報が含まれています。カメラ情報には、IP アドレス、ID、MAC アドレス、シリアル番号、ソフトウェアとアルゴリズムのバージョンが含まれます。深度、振幅、RGB、または点群チェックボックスを選択して、そのビューを中央パネルに表示します。

### 6.4.3 画像の保存

[**保存**] を選択して、選択した画像をインストール ディレクトリの `./data` フォルダーに保存します。端末はパスも出力します (例: `save path= ./data/<camera-ID>`)。

保存後、`_a` は振幅画像、`_d` は深さ画像、`_r` は RGB (MONO) 画像を識別します。 `.pcd` は点群を識別します。深度と振幅の画像は PGM 形式を使用し、RGB 画像は PNG を使用し、点群は PCD を使用します。

### 8.2 正しい操作

M4 カメラは 24V DC 電源を使用します。電源を入れる前に電源ケーブルを接続してください。製品を損傷する可能性があるため、プラスとマイナスの端子を逆にしないでください。

### 8.3 動作温度

周囲温度は表 2-1 に指定された範囲内に保ってください。熱放散を改善し、最適なパフォーマンスを実現するには、金属ブラケットなどの熱伝導性コンポーネントに製品を取り付けます。

## 1. 製品概要

このガイドでは、Zhejiang MRDVS Technology Co., Ltd. が開発した M4 メガ カメラの仕様、特徴、操作方法について説明します。

M4 Mega 製品ファミリーには、M4 Mega カメラが含まれます。この ToF 製品は、ギガビット イーサネット通信インターフェイスと 24VDC 電源を使用します。

- 内蔵 1Tops コンピューティング能力により、外部 CPU なしで動作可能

- RGB-Dのサポート

- スタンドアロンの障害物回避、ドッキング、位置特定、ソフトパッケージの把握、保管場所の検出、体積測定

- ギガビットイーサネットポート

- IP67の侵入保護

### 1.2 利用規約

**使用中は次の要件に従ってください:**

・製品のラベルは剥がさないでください。

・製品を分解しないでください。分解すると破損する恐れがあります。

- 強い磁界の中で製品を使用しないでください。

- 100KLUXを超える強い太陽光の下で製品を使用しないでください。

- 筐体を清掃するには、精製水またはアルコールで軽く湿らせた、糸くずの出ない柔らかい布で拭きます。

- 元の梱包材を保管し、輸送中は可能な限り使用してください。

## 2. 製品仕様

### 2.1 距離測定原理

M4 Mega カメラはタイムオブフライト (ToF) 測距を使用します。光を発してから物体からの反射を受け取るまでの時間を測定し、物体とカメラの間の距離を計算します。

M4 Mega カメラは深度を直接キャプチャし、最小限のコンピューティング リソースで高品質の深度情報を提供し、高フレーム レートで深度シーンを構築します。 LiDAR と同様に、ToF カメラはパルス ToF (P\_ToF) または連続波変調 ToF (CW\_ToF) を使用できます。 P\_ToF は、通常は目に見えない光パルスを放射し、その往復時間から距離を計算します。 CW\_ToF は、変調された連続光を放射し、放射された光と戻ってきた光の位相差から距離を計算します。

### 2.2 製品概要

このコンパクトな産業用 RGB-D カメラは、3 チャンネルのトゥルーカラー RGB 出力を備えた Sony CW iToF カメラを統合しています。 0.2m〜5mの深度データを提供し、RGB画像と深度画像を空間的および時間的に位置合わせします。カメラは、構造化されたパレット認識結果を CAN 経由で出力することもできます。

### 3.8 ハードウェアトリガー

1. ハードウェア トリガーは、製品がハードウェア トリガー モードの場合にのみ機能します。ハードウェア トリガー モードと障害物回避モードは相互に排他的です。ハードウェア トリガー モードでは、カメラは I/O 入力ピン 1 上の有効なパルスを待機し、1 フレームを露光して待機状態に戻ります。

2. ハードウェア トリガー信号は 3.3V ～ 24V で、少なくとも 10mA の駆動電流を供給する必要があります。パルスノイズによる誤トリガーを防止するため、トリガー信号端子にデカップリング回路を追加してください。

### 3.8.1 ホスト アプリケーションでのハードウェア トリガーの構成

カメラを開く → 障害物回避アルゴリズムがオフであることを確認する → ストリームを停止する → **基本ツール** → **取得設定** → **トリガー モード**を**ハードウェア トリガー**に切り替える → パラメータを設定する → ストリームを開始し、外部トリガー信号を待ちます。

**1.ハードウェアトリガーパラメータ**

**フィルター時間:** ハードウェア トリガー入力がトリガー信号を受信した後、トリガー信号を再度チェックするまでの待機時間。これによりノイズが除去されます。 5ms 未満の値を使用してください。値を大きくすると、有効なトリガー信号がフィルターされる可能性があります。

**最小間隔:** 2 つの連続する自動トリガー信号間の間隔。範囲は 70000 ～ 100000000μs で、値は積分時間を超える必要があります。

**トリガー時間:** トリガー信号を再度確認した後の遅延。即時トリガーするには 0 に設定します。範囲は0～100000000μsです。

**トリガーあたりのフレーム数:** 1 つのハードウェア トリガーによってトリガーされる連続フレームの数。範囲は 1 ～ 200 フレームです。

**2.ハードウェアトリガーの例**

**例 1:** フィルタ時間: 1000;最小間隔: 70000 (最小);トリガー時間: 2000;トリガーあたりのフレーム数: 10。

信号を受信した後、カメラは 1ms 待ってから再度確認します。最初のフレームをトリガーする前にさらに 2ms 待機し、フレーム 10 まで後続のフレーム間で 70ms 待機します。このシーケンス中に受信したトリガー信号は無視されます。

**例 2:** フィルタ時間: 1000;最小間隔: 無効;トリガー時間: 0;トリガーあたりのフレーム数: 1。

信号を受信した後、カメラは 1ms 待機し、再度確認し、すぐにトリガーします。トリガーあたりのフレーム数が 1 の場合、最小間隔は適用されません。

**例 3:** フィルター時間: 1000;最小間隔: 200000;トリガー時間: 0;トリガーあたりのフレーム数: 200。

### 3.9 ソフトウェアトリガー

ソフトウェア トリガーは、製品がソフトウェア トリガー モードの場合にのみ機能します。ソフトウェアトリガーモードと障害物回避モードは相互に排他的です。ソフトウェア トリガー モードでは、カメラは有効なソフトウェア トリガー信号を待ち、1 フレームを露光して待機状態に戻ります。

### 3.9.1 ホスト アプリケーションでのソフトウェア トリガーの構成

1. **ソフトウェアトリガーパラメータ**

**フィルタ時間:** ソフトウェア トリガ入力がトリガ信号を受信した後、トリガ信号を再度チェックするまでの待ち時間。これによりノイズが除去されます。 5ms 未満の値を使用してください。値を大きくすると、有効なトリガー信号がフィルターされる可能性があります。

**トリガーあたりのフレーム数:** 1 つのソフトウェア トリガーによってトリガーされる連続フレームの数。範囲は 1 ～ 200 フレームです。

1. **ソフトウェアトリガーの例**

## 4. 精度と精度

### 4.1 精度

キャリブレーションでは、センサーの測定値と実際の値を相互にマッピングすることで、それらの間の固定オフセットを除去します。 M4 Mega カメラを調整するには、レールに取り付けて白い壁に向けます。図 4.1 に示すように、各ピクセルの測定値をカメラと対応する壁領域の間の距離にマッピングし、このマッピングを使用して測定値を補正します。

<Frame>
  <img src="/images/legacy/379b03b6-5b43-4db0-bc44-b715c859694e/cd16d195-e215-49ca-a673-1c00034af49f.avif" alt="M4 Mega キャリブレーションのセットアップ" />
</Frame>

図4.1。較正

M4 Mega はアクティブ レンジングを使用するため、多くの要因がその精度に影響します。不適切に設置すると、マルチパス効果が生じ、測定が歪む可能性があります。

これは、反射率の高い対象物の横にある低反射率の対象物を測定する場合に特に顕著です。理想的には、光は青い経路をたどります。壁から反射した光もターゲットに到達し、緑のパスに沿ってカメラに戻る場合があります。隣接する高反射率の物体から直接反射された照明が、赤い経路に沿ってカメラに入射する可能性があります。図 4.2 に示すように、光学的欠陥によりこの光がグレアに変わり、測距の妨げになります。

<Frame>
  <img src="/images/legacy/e4159efc-5b5f-458d-b57f-e01e572c892f/b523267e-64ca-4b4c-9c2d-6d3c5afd55b6.avif" alt="M4 Mega のインストールと反映パス" />
</Frame>

図4.2。カメラの設置

### 4.2 精度

精度は通常、単一点の変動 (1-σ) を指します。光電子測定デバイスとして、M4 Mega は ADC 変換、信号対雑音比、およびシステム ノイズの影響を受けます。図 4.3 に測定精度と信号強度の関係を示します。

<Frame>
  <img src="/images/legacy/ee73d646-4d62-4aaf-8833-5f2c4b5cf3d9/a3a07cf5-cd92-42b4-a817-e1ef02896e6d.avif" alt="測定精度と信号強度" />
</Frame>

図4.3。測定精度と信号強度

M4 Mega は、光電子アナログ信号をデジタル データに変換します。 ADC 変換中、アナログ値は最小の分解能ステップに丸められるため、ノイズが発生します。 1.1V 信号は 1V として表される場合があり、10% の誤差が生じます。 10.1V の信号が 10V になる場合や、100.1V の信号が 100V になる場合があり、それぞれ約 1% の誤差が生じます。したがって、ADC 変換では、信号レベルが低い場合、より多くのノイズとエラーが発生します。 ToF カメラの S/N 比は通常、SNR∝1/√t に従います。ここで、`t` は積分時間です。積分時間を長くすると、信号強度が増加します。

## 5. マルチカメラの干渉

### 5.1 平行干渉

2 台のカメラが並列に設置されている場合、それぞれのカメラが自身の反射照明と他のカメラからの照明の両方を受け取り、深度データに干渉する可能性があります。 2 台の M4 Mega カメラが同じ方向を向いている場合、平行干渉により測定誤差が増加します。

<Frame>
  <img src="/images/legacy/3b09c42a-62ef-4a7f-96ad-6366d662588e/172c3d00-c862-4b8b-b294-8cfbcccfe558.avif" alt="平行カメラ干渉" />
</Frame>

平行干渉を避けるため、カメラを水平に対して斜めに設置してください。水平面に対して 0° で設置する必要がある場合は、カスタム構成について MRDVS にお問い合わせください。

### 6.2 SDKの出力順序

<Frame>
  <img src="/images/legacy/84ac41e4-cda9-4a16-8090-118e488a3bb8/cd2fd356-196f-47a6-a7e3-2dadb89d7acd.avif" alt="SDKの出力順序" />
</Frame>

図6.1。 SDKの出力順序

SDK は、設定された解像度で深度と振幅データをそれぞれ 1 フレームずつ出力します。各フレームは、左上のピクセルから始まり、左から右、上から下の順に進む 1 次元配列として保存されます。図 6.1 は、x 軸に沿って出力を示し、その後に y 軸が続き、次の順序で生成されます。

(x,y):(0,0)(0,1)(0,2)(0,…)(1,0)(1,1)(1,2)(1,…)(2,…)(3,…)(…,…).

深度画像の各ピクセルは、測定対象物からカメラ平面までの距離を表す 16 ビットの符号なし short です。

各振幅画像ピクセルは、返された信号強度を表す 16 ビットの符号なし short です。

### 6.3 IPアドレス

M4 Mega カメラのデフォルトの IP アドレスは 192.168.100.82 です。 IP アドレスを変更するか、DHCP を有効にするには、MRDVS が提供するデモ アプリケーションを使用します。

### 6.4 デモアプリケーション

### 6.4.1 IP 構成

電源ケーブルを接続し、カメラとパソコンをイーサネットケーブルで接続します。カメラの工場出荷時のデフォルト IP アドレスは 192.168.100.82 であるため、コンピュータを 100.82 を除く 192.168.100.x サブネット、サブネット マスク 255.255.255.0 に設定します。 **設定** → **イーサネット** → **アダプター オプションの変更** を開きます。 **イーサネット**を右クリックし、**プロパティ**を選択し、次に**インターネット プロトコル バージョン 4 (TCP/IPv4)**を選択します。たとえば、IP アドレスを 192.168.100.20 に、サブネット マスクを 255.255.255.0 に設定します。

### 6.4.2 アプリケーションを開く

ソフトウェアをダウンロードしてインストールし、LxCameraViwer デモ アイコンをダブルクリックします。ホスト アプリケーションが開き、カメラを検索して開き、1 台のカメラが接続されると自動的に取得を開始します。

**注: 一部のシステムでは、最初の起動時に、アプリケーションにファイアウォールの通過を許可する必要があります。**

アプリケーションには、基本的なコントロール、画像の選択、カメラの設定、およびカメラ情報が含まれています。カメラ情報には、IP アドレス、ID、MAC アドレス、シリアル番号、ソフトウェアとアルゴリズムのバージョンが含まれます。深度、振幅、RGB、または点群チェックボックスを選択して、そのビューを中央パネルに表示します。

### 8.2 正しい操作

M4 Mega カメラは 24V DC 電源を使用します。電源を入れる前に電源ケーブルを接続してください。製品を損傷する可能性があるため、プラスとマイナスの端子を逆にしないでください。

|番号 |問題 |解像度 |
| --- | --- | --- |
| 1 |カメラを開いたときに反応しません |ファイアウォールが無効になっていることを確認します。 |
| 2 |ホスト アプリケーションが複数の IP アドレスを検出します。 LAN上に重複したIPアドレスが存在します。必要な IP アドレスを選択します。 |
| 3 |開いた後のカメラデータが不安定です |ギガビットイーサネットケーブルを使用してください。 100Mbps ケーブルでは、最初はデータが不安定になる可能性があります。 |

## 1. 概要

LxCameraViewer は、Blue Core の自社開発カメラ製品およびカメラ SDK C または C++ 開発キットに基づいて開発された Windows ホスト コンピュータ ソフトウェア (以下、本ソフトウェアと呼びます) です。

このソフトウェアは、ビュー、カメラ設定、アルゴリズム設定、IP 構成、点群表示、パケット キャプチャ ツール、通信ツール、ファイアウォール構成およびその他の機能などのシンプルかつ高速な機能をカメラ ユーザーに提供します。現在、このソフトウェアは複数のデバイスを開いて表示することをサポートしており、深度画像、強度画像、点群画像、および RGB 画像の出力の表示をサポートしています。また、カメラの基本機能やアルゴリズムの基本構成などのパラメータの読み込みと設定もサポートします。

このアプリケーションを通じて、カメラから取得した高精度な画像データやカメラの構成、アルゴリズムの詳細データなどを閲覧・保存することができます。

### 2.2 IP アドレスとゲートウェイの構成

・接続方法は固定IPアドレス接続です。 IPを変更する必要がある場合は、このソフトウェアを通じて変更できます。

- カメラのデフォルトの IP アドレスは 192.168.100.82 です。

- デフォルト ゲートウェイは空でもかまいません。

- サブネット マスクは 255.255.0.0、Linux プラットフォームでは 255.255.255.0 です。

- ホストコンピューター上のカメラに接続されているイーサネットの IP アドレスを 192.168.100.x に設定します。 x は 3 ～ 254 の任意の数値を表します。ホスト コンピュータの IP とカメラの IP を同じにすることはできません。

- たとえば、次の図に示すように、カメラの IP アドレスが 192.168.100.12 の場合、ホスト コンピュータの IP アドレスは 192.168.100.89、サブネット マスクは 255.255.0.0 に設定されます。

### 3.2 デバイスの電源のオンとオフ

#### 3.2.1 デバイスリスト

- デバイス リストはインターフェイスの左上隅 (図 1 の赤いボックス内) にあります。

### 3.4 デバイス情報

- デバイス情報は、インターフェイスの左下隅 (図 1 の赤いボックス内) にあります。

- デバイス情報には、デバイスリストで現在選択されているデバイスに関する関連情報が表示されます。情報は次のとおりです。

- シリアル番号: デバイスの一意の識別子

- カメラ ID: デバイスの一意の識別子。 SKD を使用して、カメラ ID に従ってデバイスを開くことができます。

- IP アドレス: デバイスの IP アドレス

- 物理アドレス: デバイスの MAC アドレス

- ソフトウェアバージョン：デバイス内蔵ソフトウェアバージョン

- アルゴリズムバージョン：デバイス内蔵アルゴリズムバージョン（カメラを開いた後に表示されます）

- ネットワークカードのネットワーク速度: デバイスの電源が入っていない場合、デバイスは切断されます。ネットワーク カードのネットワーク ケーブルをオンにすると、100 メガビットかギガビットかが決まります。

- カメラの種類: デバイスモデル

- ミラーバージョン：デバイス内蔵ミラーバージョン（カメラを開いた後に表示されます）

- 折りたたみボタンをクリックしてデバイス情報を折りたたみます (図 2 の黄色のボックス内)

- デバイス情報の項目を右クリックして情報をコピーします (図 3 に示すように、ネットワーク カードの速度情報をコピーします)。

#### 3.5.2 画像（フロー）制御

- [フローの開始/停止] ボタン: クリックしてデバイスのフローを開始または停止します。

・[シングル保存]ボタン：ONにしたストリームデータの画像を保存します。

- たとえば、図 2 では、深度画像と点群画像のみが保存されます。 RGB 画像と輝度画像はオンになっていないため保存されません。

- 深度マップおよび強度マップ ファイルの接尾辞は .pgm、RGB イメージ ファイルの接尾辞は .png、点群イメージ ファイルの接尾辞は .pcd です。

- 保存された深度画像、強度画像、点群画像は、偽色を付加していないオリジナルの画像です。

・複数回クリックして連続保存できますが、連続で保存できるのは3回までです。

- [Depth Map]チェックボックス: デプスマップを表示(オン)するかどうか

・[強度グラフ]チェックボックス：強度グラフを表示(オン)するかどうか

##### 3.7.2.3 3D 設定

・ [3D幅] 点群画像の幅は機種によって決まり、設定できません。

- [RGB 位置合わせ] 深度マップ、強度マップ、点群、RGB 画像を位置合わせします。位置合わせ後、深度マップと強度マップは RGB 画像と同じ幅と高さになります。点群データは増加し、フローを停止した後に変更できます。

- [マルチマシンアルゴリズム感度] デバイスの内蔵パラメータはデバイスモデルによって決定されるため、設定できません。

- [アンチディストーション係数] ディストーション係数を調整して、遮蔽された画像の歪みを強めたり弱めたりします。

- [周波数モード] デバイスの内蔵パラメータはデバイスのモデルによって決まります (一部のカメラの周波数は固定されており、設定できません)

- [HDR 有効] HDR 有効をオンにすると、明暗の変化が激しいシーンの画質を向上させることができます。

- [自動露出] オンにすると、デバイスは周囲の光の強さに応じて露出パラメータを自動的に調整します。

- [自動露出下限値] 自動露出の下限値を設定します(設定不可)

- [ゲイン]は強度値を倍数で増加させることができます(設定できません)

- [x] ROI 領域セットの左上隅の x 座標。フローが停止した後に変更できます。

- [y] ROI 領域の左上隅の y 座標を設定します。フローの停止後に変更できます。

- [高統合] 高い統合時間 (長くすると、カメラはさらに距離を測定できるようになります)

- [低積分] 低積分時間 (主に近距離での距離測定を制御します。通常は変更する必要はありません)

- [x offset] 幅方向のオフセット値。フローを停止した後に変更できます。

- [y offset] 高方向のオフセット値。フローが停止した後に変更できます。

##### 3.7.2.4 フィルター設定

- [最小深度] 深度マップの偽色または距離の下限。この値を下回ると、偽カラーは黒として表示され、強制的にゼロに設定されます。

- [最大深度] 深度マップの偽色または距離の上限。この値を超えると、擬似カラーは黒として表示され、強制的にゼロに設定されます。

- [低信号閾値] 強度値の下限値。この値を下回ると、深度マップ内の対応するピクセルが強制的にゼロに設定されます (強度マップには効果がありません)。

- [高信号閾値] 強度値の上限。この値より大きい場合、深度マップ内の対応するピクセルは強制的にゼロに設定されます (強度マップには影響しません)。

- [小信号測定] 小信号の検出レベルを設定します。レベルが高いほど、検出されやすくなります。

- [フィルターモード] フィルターのシールドレベルを設定します

- [スムージング レベル] 空間フィルタリングのスムージング強度を制御し、ノイズをスムージングしながらオブジェクトのエッジの詳細を保持します。

- [ノイズレベル] ノイズフィルターの閾値を設定します(値が大きいほどノイズが多くなります)

- [時間レベル] 時間フィルタリングの強度を制御し、過去のマルチフレーム データを使用して深度データの連続性を向上させます。

- 注: スムージング レベル、ノイズ レベル、およびタイム ドメイン レベルを 0 に設定すると、フィルタリングがオフになります。

##### 3.7.3.1 障害物回避アルゴリズム

- アプリケーションアルゴリズムを障害物回避アルゴリズムに設定すると、障害物回避関連の設定が表示されます。

・[車体表示] 点群画像上に車体の位置を表示する車種のオン/オフを設定できます。

- [障害物回避インデックス] 障害物回避には複数の構成があり、このオプションで設定できます。

- [カメラ角度ロール (x 軸)] カメラの外部パラメータ、ロール角度、デフォルトのカメラは 180 です

- [カメラ角度ピッチ (y 軸)] カメラの外部パラメータ、ピッチ角度、デフォルトの垂直設置は 90 です

- [カメラ角度ヨー (z 軸)] カメラ外部パラメーター、方位角ヨー、デフォルトは 90

- [カメラ位置 (距離 x)] カメラの外部パラメータ、x 方向と原点の間の距離、単位: メートル。

・[カメラ位置(幅y)] カメラ外部パラメータ、y方向と原点間の距離、中央に設置した場合は0となります。単位はメートルです。

- [カメラ位置 (高さ z)] カメラの外部パラメータ、z 方向と原点の間の距離は、地面からのカメラの高さとして理解できます (単位: メートル)。

- [早期警戒距離 (x far)] 障害物回避範囲、車両中心からの x 方向の距離。この距離を超える障害物は安全とみなされます。早期警告距離とアラーム距離の間の距離は早期警告減速度とみなされ、インターフェイスには黄色の領域が表示されます。単位: メートル。

・[警報距離(x中心)]障害物回避範囲、車両中心からのx方向の距離。この距離より下の障害物は警報停止とみなされ、インターフェースには赤い領域が表示されます。事前警告距離とアラーム距離の間の距離は早期警告減速度とみなされ、インターフェイスには黄色の領域が表示されます。単位: メートル。

・[遮蔽距離(x付近)]障害物回避範囲、車両中心からx方向の距離。この距離内の障害物は無視されます。単位: メートル。

・[障害物回避範囲左(y左)] 障害物回避範囲、y方向左側から車両中心までの距離。単位: メートル。

・[障害物回避範囲右(y右)] 障害物回避範囲、y方向右側から車両中心までの距離。単位: メートル。

・[障害物回避高さ低め(z低め)] 障害物回避範囲、地面からの高さ、この高さ未満は処理されません。設定が小さすぎると、測距エラーにより誤警報が発生する可能性があります。設定が大きすぎると、地上の大きな障害物が除外され、誤警報が発生する可能性があります。単位: メートル。

・[障害物回避高さ上限(z上)] 障害物回避範囲、地面からの高さ、これより高い高さは処理されません。単位: メートル。

- [障害物回避出力] 障害物回避結果を出力します。

- [カメラ外部パラメータをリセット] カメラに書き込まれていないすべてのパラメータを復元します

- [外部パラメータを設定] すべてのパラメータをカメラに書き込みます

- 障害物回避の詳細な設定については、障害物回避のドキュメントを参照してください。

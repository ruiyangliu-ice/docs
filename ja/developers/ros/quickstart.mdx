## 序文

この記事では、ビジュアルポジショニングモジュールのソフトウェアアーキテクチャ、ROS SDKの設定方法、基本的なマッピングとポジショニングの操作プロセスを紹介します。

</Frame>

この位置決めモジュールの概略図を上に示します。その中で、カメラに内蔵された測位プログラムは、UDP を使用してホスト コンピューターの ROS SDK と対話します。さらに、測位プログラムには、測位の視覚化を実現できる構成可能な WEB フロントエンドが付属しています (主流のブラウザを使用して開くことができ、デフォルトのアクセス IP は 192.168.100.201:9998 です)。フロントエンドには 2 つの主な機能があります。

1. 完全なデモを実行して、マッピングと位置決めのプロセスを完了します。

2. ROS SDK と通信するためのターゲット ID とポートを設定します。同様に、ROS SDK を起動する前に、ターゲット カメラの IP とポートを設定する必要があります。

上記の設定を完了すると、マッピング、再配置、マップのダウンロードなどの WEB フロントエンド インターフェイスのすべての機能を ROS SDK を通じて呼び出すことができます。

## コンパイル

### プログラム実行環境の構成

1.QTをインストールする

ホスト コンピュータに QT5 がインストールされていることを確認してください。 QT5 にはコア、ネットワーク、その他の機能パッケージが必要です。 Linux のバージョンが異なるとインストール方法が異なる場合があります。インストール例は次のとおりです。

```bash
sudo apt install qtbase5-dev qtbase5-dev-tools libqt5network5
```

2. libzip 関数パッケージをインストールする

インストールチュートリアル https://blog.csdn.net/iamhuanyu/article/details/129603632

3. ROS 関数パッケージをコンパイルします。 `catkin_make` パス (`/src` と同じレベル) で `catkin_make` を実行します。

### デバッグ

1\。 ROSのデバッグ方法

新しいターミナルを開き、`/V2-ROS-SDK-service` パスを入力して `source devel/setup.bash` を実行し、デバッグを開始します。

ａ．メッセージを購読する

例: 現在占有されているカメラのストレージ容量を取得する

デバッグに入ったら、ターミナルで `rostopic echo /lx_localization_node/LxCamera_Capacity_Status` を実行します。

次のメッセージを受信した場合は、通信リンク全体が正常であることを意味します。同時に、このメッセージは 2hz の周波数のハートビート パケットとしても使用できます。

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8092fa65-76da-46ef-8756-b0e37d673e76.avif" alt="デバッグ関連のアイコン" />
</Frame>

2\。サービスインターフェースを呼び出す

例: 現在選択されているマップ名の表示

ターミナルで `rosservice call /lx_localization_node/LxCamera_GetCurMapname_srv "get_cur_mapname: true"` を実行します。

次のメッセージを受信した場合は、通信リンク全体が正常であり、返されたメッセージが現在のマップ名であることを意味します。

## 基本的な操作手順

カメラの電源をオンにして、約 30 秒待ってから、データ転送ノードを起動して使用を開始します。

デバッグ端末に入る

***1.マップ作成手順の発行***

rosservice 呼び出し /lx\_localization\_node/LxCamera\_Mapping\_srv 「ステータス: true マップ名: 'test\_map'」

このコマンドは、マッピング タスクを開始するために使用されます。 「test\_map」という名前のマップを作成する準備が始まります。発行に成功した場合は true を返します。このとき、車を引っ張って地図をスキャンできます。

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8a77a9c8-f9df-4c71-a4c9-a4e9d66d6892.avif" alt="基本的な操作手順のイラスト" />
</Frame>

***2.コマンドを発行してマッピングを終了します***

rosservice 呼び出し /lx\_localization\_node/LxCamera\_Mapping\_srv 「ステータス: false マップ名: 'test\_map'」

このコマンドは、マッピング タスクを終了するために使用されます。配信が成功すると true が返されます。このとき、マッピングの完了を待って戻り値を受け取ることができます。

***3.マップの構築が完了したら、しばらく待ってから新しいマップの名前に切り替えます***

rosservice 呼び出し /lx\_localization\_node/LxCamera\_SwitchMap\_srv "map\_name:"test\_map'"

＊＊＊４．写真を切り取ったら、移転メッセージを送信してください***

### よくある質問と解決策

***1\.ROS*** プログラムをコンパイルできません

\-依存ライブラリ環境をチェックして、すべての依存ライブラリが完全にインストールされているかどうかを確認します

***2.ROS SDK*** メッセージを送信しても応答がありません

\-カメラとマシンが同じネットワーク セグメント上にあること、またはカメラのホスト コンピューターを使用してカメラを開いて画像を表示できるかどうかを確認します。

\-カメラ設定ファイルに問題がないことを確認し、ROS設定ファイルに問題がないことを確認してください。

***3.ROS*** デバッグメッセージを送信できません

\-ROS メッセージには厳密な形式要件があります。デバッグ時は指示書のフォーマットに従い、値の変更のみを行ってください。

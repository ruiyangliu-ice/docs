## 1. **概要**

このドキュメントは主に、カメラ デバイス アクセスに関連する開発者を対象としています (ソフトウェア開発能力が必要です)。現在、M4/M4 Mega/M4 Pro/V1Pro/S2/S2/H3 シリーズ製品をサポートしています。このドキュメントの主な目的は、SDK が提供する API の使用方法を説明することです。カメラ SDK は、x86 ベースの Windows オペレーティング システム、および X86 および Arm ベースの Linux オペレーティング システムでの実行をサポートします。

カメラ SDK は、統合されたアクセス制御およびパラメータ構成インターフェイスを提供し、デバイス データ (深度画像、強度画像、点群画像、RGB 画像を含む) の同期または非同期取得をサポートします。一部のカメラ モデルは、障害物回避、ドッキング、位置決め、その他のデータをサポートしています。

このドキュメントは最大限の開発支援を提供するものであり、提供されるコンテンツには欠陥があるか不完全である可能性があります。

##### 2.2.3.2 Linux システムのインストール:

1. インストールスクリプトを解凍して実行します。

2. インストールには root 権限が必要なため、現在のユーザーが root でない場合は、root ユーザーに切り替える必要があります。

インストール スクリプトは、SDK を /opt/Lanxin-MRDVS ディレクトリにインストールし、環境変数を設定します。スクリプトに設定される環境変数は現在のユーザーの環境変数であることに注意してください。実行中のプログラムが root ユーザーではない場合は、install.sh スクリプトを再実行して環境変数を設定するか、現在のユーザーの ~/.bashrc の末尾に import LD\_LIBRARY\_PATH=/opt/Lanxin-MRDVS/lib/:$LD\_LIBRARY\_PATH を手動で追加する必要があります。

インストール前にターミナル セッションが開かれている場合は、プログラムを実行する前に、source ~/.bashrc を実行して LD\_LIBRARY\_PATH 環境変数を更新する必要があります。

## 4. **カメラ SDK 呼び出しに関するメモ**

### 4.1 カメラ SDK インターフェイス呼び出しの一般原則

1. LX\_INT というプレフィックスが付いた列挙型の場合、DcSetIntValue または DcGetIntValue を使用して INT 型パラメータを設定または取得します。

2. LX\_FLOAT というプレフィックスが付いた列挙型の場合は、DcSetFloatValue または DcGetFloatValue を使用して FLOAT 型パラメータを設定または取得します。

3. LX\_BOOL というプレフィックスが付いた列挙型の場合は、DcSetBoolValue または DcGetBoolValue を使用して BOOL 型パラメータを設定または取得します。

4. LX\_STRING というプレフィックスが付いた列挙型の場合は、DcSetStringValue または DcGetStringValue を使用して文字列型パラメータを設定または取得します。

5. DcSetCmd を使用して、LX\_CMD というプレフィックスが付いた列挙型の命令を実行します。

6. LX\_PTR というプレフィックスが付いた列挙型の場合は、DcGetPtrValue を使用してポインター型の結果を取得します。

### 4.2 カメラ SDK インターフェース部分呼び出しの前提条件

**1.フローが開始されている場合、または作業モード (LX\_INT\_WORK\_MODE) が通常オープン モードである場合、次の関数は設定できません。**

- DcSetRoi

- LX\_INT\_3D\_BINNING\_MODE;

- LX\_INT\_2D\_BINNING\_MODE;

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH;

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR;

- LX\_BOOL\_ENABLE\_MULTI\_MACHINE;

- LX\_INT\_TRIGGER\_MODE;

- LX\_INT\_CALCULATE\_UP;

- LX\_BOOL\_ENABLE\_HDR;

- LX\_BOOL\_ENABLE\_TOF\_DEPTH\_CALIBE;

- LX\_BOOL\_ENABLE\_SYNC\_FRAME

- LX\_INT\_SAVE\_PARAMS\_GROUP

**2.作業モード (LX\_INT\_WORK\_MODE) が通常開いている場合でも、設定は許可されません:**

- LX\_INT\_TOF\_FREQMODE<br />

**3. 3D 自動露出 (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) がオンになっている場合、次の関数列挙操作は許可されません:**

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR ;

- LX\_INT\_FIRST\_EXPOSURE;

- LX\_INT\_SECOND\_EXPOSURE;

- LX\_INT\_THIRD\_EXPOSURE;

- LX\_INT\_GAIN;<br />

**4.3D 自動露出 (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) がオフになっている場合、次の関数列挙は動作できません。**

- LX\_INT\_3D\_AUTO\_EXPOSURE\_LEVEL

3D 自動露出の最大値 (LX\_INT\_3D\_AUTO\_EXPOSURE\_MAX) と最小値の設定は許可されていません

- (LX\_INT\_3D\_AUTO\_EXPOSURE\_MIN)

**5. 2D 自動露出 (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) がオンになっている場合、次の関数列挙操作は許可されません:**

- LX\_INT\_2D\_MANUAL\_EXPOSURE

<br />**6. 2D 自動露出 (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) がオフになっている場合、次の関数列挙は動作できません:**

- LX\_INT\_2D\_AUTO\_EXPOSURE\_LEVEL;

<br />7.**特殊フィルタリング モードでは、次の設定は許可されません:**

- LX\_INT\_FILTER\_SMOOTH\_LEVEL LX\_INT\_FILTER\_NOISE\_LEVEL

- LX\_INT\_FILTER\_TIME\_LEVEL

<br />8.**2D アンチディストーションをオンにする場合は、最初に RGBD アライメントをオンにする必要があります:**

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH

<br />9.**S2 タイプ以外のデバイスの場合、組み込みアルゴリズムをオンにする場合は、最初に tof アルゴリズムを設定する必要があります。アルゴリズム パラメーターは、組み込みアルゴリズム (LX\_INT\_ALGORITHM\_MODE) がオンになっている場合にのみ設定できます。**

- (LX\_STRING\_ALGORITHM\_PARAMS)

<br />10.**点群データ (LX\_PTR\_XYZ\_DATA) を取得するには、まず深度マップを開く必要があります**

- (LX\_BOOL\_ENABLE\_3D\_DEPTH\_STREAM)

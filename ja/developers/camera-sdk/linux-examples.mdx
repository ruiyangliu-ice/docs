## 1. **概要**

Linux 用 MRDVS カメラ SDK には、C、Java HTTP サーバー、ROS、および ROS 2 のサンプルが含まれています。この例では、コア SDK ワークフローと基本機能を示します。実装をアプリケーションに適応させます。このガイドでは、ROS と ROS 2 について詳しく説明し、必要な環境をセットアップして各サンプルを実行する方法を説明します。

パート 1 からパート 3 までを優先順に読むことをお勧めします。パート 1 ～ 3 を順番に読んだ後、さまざまなニーズに合わせて関連するパートに直接ジャンプできます。各章はそれぞれ独立しており、読むことに影響を与えません。

最後に、この記事では、インストール方法と、関連するアプリケーションや環境の注意事項について説明します。必要に応じて付録を参照してください。

#### 3.2.1 C および C++ のサンプル コード

C ディレクトリに入ります。ディレクトリの内容は次のとおりです。

<Frame>
  <img src="/images/legacy/e3b253f3-6743-4945-b738-b8e9b8dfc7dc/9a255d61-eadf-4076-847d-65b29fe4eace.avif" alt="CおよびC++のサンプルコードに関するイラスト" />
</Frame>

9 つのフォルダーと 1 つの Cmakelist ファイルが含まれており、各フォルダーには cpp ファイルが含まれています

各 cpp に実装される機能は大まかに以下のとおりです。

- application\_obstacle: カメラ ストリーム データを取得し、障害物回避アルゴリズムを有効にし、アルゴリズムの結果を出力します

- application\_pallet: カメラ ストリーム データを取得し、パレット アルゴリズムを有効にし、アルゴリズムの結果を出力します

- arm\_local\_camera: カメラ[ARM]内のSDKを呼び出し、ストリームデータを取得します

- Frame\_callback: コールバック メソッドを使用してカメラ ストリーム データを取得します

- multi\_cameras: 複数のカメラを開いてストリーミング データを取得します

- single\_camera: 単一のカメラを開いてストリーミング データを取得します

- single\_camera2: 構造体を通じてフレーム データを取得します

さまざまなニーズに応じて、テストや変更のためにさまざまなサンプル コードを選択できます。

### 5.3 ROS1 と ROS2

ros の説明では、ドライバーのソース コードについては説明しませんが、起動ファイルの構成に対応する内容、構成の調整方法、メッセージの送信とサブスクライブの方法について説明します。提供されている ros ノードにはすでにほとんどの機能が含まれているため、ドライバーのソース コードを直接変更することはお勧めできません。特別なニーズがある場合は、まず C++ サンプルのコード フローを理解してから、ドライバーのソース コードを変更できます。

**ros ドライバーを実行するときは、システムのデフォルト パス内の opencv と pcl のバージョンが一意であることを確認する必要があります。そうしないと、ros ライブラリのリンク時にエラーが発生し、コンパイルの失敗や実行時のクラッシュが発生する可能性があります。**

### 6.3 メモリの詳細

メモリの詳細には、さまざまな動作条件下でノードが占有するメモリ リソースの量が表示されます。テストは **AMD64 [8 コア]、8G メモリ** ハードウェア環境に基づいており、テスト カメラは **M4PRO**、テスト アルゴリズムは **トレイ アルゴリズム**、**2D の幅と高さは 1280\*960、3D の幅と高さは 640\*480**、**2D アンチディストーションはオン、3D アンチディストーションはオフ、ポイント 110 - 3000、深度 0 ～ 8000、TOF フレーム レート 14、ストリーミング モード、スムージング レベル 1、ノイズ レベル 1、タイム ドメイン レベル 0**、統計コマンドは **TOP** です。テスト ソフト環境は **VM 仮想マシン**であるため、物理マシンで表示されるデータがドキュメント内のデータと比較して変動するのは正常です。 S2 などの一部のカメラでは、一部のアルゴリズムがホスト コンピューター上で計算されるため、大量のリソースが使用されますが、これは正常な現象です。 <br />

**1.一部の名詞は次のように説明されています。**

-単一ストリーム: RGB、深度、強度の任意のストリームをオンにすることを指します。

- デュアル ストリーム: RGB、深度、強度の 2 つのストリームをオンにすることを指します。

- 3 つのストリーム: RGB、深度、および強度を指し、3 つのストリームすべてがオンになります。

- 4 つのストリーム: RGB、深度、強度、点群がすべてオンになることを意味します

**2.このアルゴリズムは、いくつかのストリームとバインディング関係を持っています。バインディング関係は次のとおりです。**

- パレットアルゴリズムを有効にするには、深さを有効にする必要があります

- 障害物回避アルゴリズムをオンにするには、深度をオンにする必要があります

- 位置決めアルゴリズムをオンにするには、RGB をオンにする必要があります

3. 表内の水色の背景は点群が有効になっている実行コンフィギュレーションであり、無色の背景は点群が有効になっていない実行コンフィギュレーションです。 **以下のすべてのテスト データでは、デフォルトでアルゴリズムが下にシフトされているため、S2 シリーズ カメラの場合、すべてのテスト値が小さすぎる可能性があります (S2 シリーズ カメラの一部のアルゴリズムは強制的に上に移動されます)。**

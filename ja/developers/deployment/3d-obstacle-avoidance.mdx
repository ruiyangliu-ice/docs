## 1. 概要

視覚障害物回避は、Meyer Micro Vision が自社開発した S シリーズ カメラと TOF 原理に基づく RGBD カメラに基づいています。ディープ ラーニングと従来の 3D ビジョン アルゴリズムを統合して、豊かなシーンに適した、正確でリアルタイム、単一マシンで構成可能、複数マシンで協調し、ある程度の主導性と予測可能性を備えた障害物回避システムを作成します。

このシステムには、リアルタイムの動的検出、適応型地上検出、検出対象の動的追跡、複数の信号出力のサポート、複数のモード切り替えのサポート、5 つのモードのサポート、および複数マシンの共同作業のサポートという特徴があります。

## 7. 通信方法

1. さまざまな通信方法を理解し、使用する前に、**非 API 通信方法**を使用して異なる障害物回避パラメータを切り替える前に、**当社が提供するホスト コンピュータを使用して障害物回避パラメータの内容を 1 つずつ設定する必要があります**。 **注記：**

**① RS485 通信は M4Pro に固有であるなど、一部の通信プロトコルは特定のモデルに固有です。 **

② API メソッドを使用して障害物回避結果を取得する場合は、まず障害物回避構成が正常に切り替えられたかどうかを確認することをお勧めします。つまり、最初に現在の障害物回避構成エリア番号を問い合わせます。

③ UDP および CAN の障害物回避通信方式を使用して、障害物回避エリアを確認します。詳細については、各通信方式の詳細紹介をご覧ください。

2. S2 Max/S10 pro のすべての障害物回避関連パラメータは、ホスト コンピュータと当社が提供する API を通じて設定できますが、他の通信方式 (IO / UDP / CAN / 485) の場合は、障害物回避パラメータ インデックスのみを変更できます。 S10 の障害物回避関連パラメータは、弊社が提供するホストコンピュータを通じて設定できますが、すべての通信方法は障害物回避パラメータインデックスの変更に限定されます。

|カメラの種類 |サポートされている障害物回避モジュールの通信方式 |
| --- | --- |
| S2 | API、UDP、IO |
| S2マックス | API、UDP、CAN |
| S2Max V2.0 | API、UDP、IO、CAN |
| M4プロ | API、UDP、RS485 |
| M4 | API、UDP、CAN |
| M4メガ | API、UDP、IO、CAN |

|カメラの種類 |障害物回避パラメータインデックスの変更 |障害物回避詳細パラメータ設定変更対応 |
| --- | --- | --- |
| S2 |すべての通信方法は変更可能です |弊社が提供するホスト コンピューターを使用してのみ変更できます。
| S2 Max、S2 Max2.0、M4、M4Pro、M4 メガ |すべての通信方法は変更可能です |ホスト コンピューターと API のみを変更できます。

**注: 障害物回避パラメータ インデックス** は、構成 0、1、2 などのパラメータ グループ インデックスを指します。 **障害物回避詳細パラメータ** とは、カメラ角度、警告距離など、障害物回避パラメータを除くすべてのパラメータを指します。

|優先順位 |通信方法 |オブジェクト |
| --- | --- | --- |
| 1 | IO通信 |パラメータはすぐに有効になり、他の手段によって干渉されることはありません。フローティングのままの場合、または IO 通信をサポートしていない場合は、デフォルトのパラメータまたは一時パラメータが使用されます。 |
| 2 |ホストコンピュータとAPIの設定 |デフォルトのパラメータが設定されており、停電の影響を受けません。 |
| 2 |その他（485等） |設定は一時的なパラメータです。このパラメータは、停電やその他の通信が発生した場合に変更されます。これは、指示に従ってデフォルトのパラメータとして保存できます。

3. 障害物回避パラメータを変更する場合、カメラは優先順位のロジックに厳密に従います。

4. 一時パラメータとデフォルトパラメータの最大の違いは、電源オフ後に保存されるかどうかです。どのパラメータが使用されるかは、最後に使用された通信方法によって異なります。

5. IO 通信中（IO 停止時ではない）、優先度 2 の通信方式では現在のパラメータの変更はできませんが、デフォルトパラメータと一時パラメータの変更は可能です。 IO 通信が切断された (IO が一時停止された) 場合、最後の設定方法に従って、特定のデフォルトパラメータまたは一時パラメータが使用されます。最後に 485 または UDP 設定が使用されるときは、一時パラメータが使用されます。最後に API 呼び出しまたはホスト コンピューター設定を使用するときは、デフォルトのパラメーターが使用されます。ロジックは次のとおりです。

### 7.1 API呼び出し方法

API呼び出し方式はC++、C#、JAVA、ROS1、ROS2などのWindows環境、Linux、armなどの環境をサポートできます。 Windows 環境の SDK がホスト コンピューター ソフトウェアにインストールされると、SDK ファイルとサンプル プログラム コードはすべてインストール パス (例: D:\\Program Files\\Lanxin-MRDVS) に配置されます。 Linux 環境 SDK は、Meier Microvision が提供する関連テクニカル サポートまたは公式 Web サイトからダウンロードできます。 S10 カメラのパラメータを切り替えるためにこの方法を使用することはお勧めできません。パラメータの切り替えには UDP または IO を使用することをお勧めします。

- Document フォルダーには、SDK およびホスト コンピューターの使用法に関するドキュメントが含まれています。

- FirmWare フォルダーは、カメラのファームウェア パッケージが保存されているディレクトリです。

・Sampleフォルダはサンプルコードのソースコードが格納されるディレクトリであり、開発環境に応じて選択できます。

・ SDKフォルダは、SDKライブラリファイルが格納されるディレクトリです。開発環境に応じて環境変数を設定できます。

- 障害物回避アプリケーションは、Sampl ディレクトリ内の C/application\_obstacled の障害物回避 API ソース コード プログラムをパラメーター化できます。

### 7.5 CAN通信方式

1. CAN 通信は S2-MAX カメラのみをサポートします。通信プロトコルはCanOpenを使用します。 CanOpen プロトコルは、標準 CAN バス上で実行されるアプリケーション層プロトコルです。その通信モードは、産業用通信プロトコルで一般的に使用される「マスター/スレーブ」モード通信でもあります。つまり、ネットワーク内にマスター局 + 複数のスレーブ局が存在することになります。スレーブステーションは相互に直接通信しません。すべての通信はマスター ステーションとスレーブ ステーションの間で行われます。マスターステーションは「クライアント」とも呼ばれ、スレーブステーションは「サーバー」とも呼ばれます。

2. 基礎となる通信方法は CAN 標準フレーム形式を使用します。つまり、CAN ID は 11 ビット (0x000 ～ 7FF)、データは 8 バイトです。

3. CAN ID の使用規則: 11 ビット ID を 4 ビットの機能コードと 7 ビットのノード ID に分割します。CAN ID は COB ID とも呼ばれます。

</Frame>

4. ボーレートが可能: デフォルトは 250K で、アプリケーションに応じて設定できます。

5. ノード番号: デフォルトは 0x212 で、アプリケーションに応じて設定できます。

6. SDO 通信メッセージは、CanOpen の基本プロトコル形式です。 CAN メッセージ内の 8 バイトのデータは、通信プロトコルによって次の形式として定義されます。

4. 次の手順に従ってテストする必要があります。

- マルチメータを使用して、24 V までの産業用コンピュータ出力のインピーダンスを測定します。&lt;50 kΩ の場合、プルアップ抵抗は必要ありません。 50kΩ以上の場合はプルアップ抵抗を付加してください。抵抗値 10 kΩ、電力 1 W 以上のスルーホール金属皮膜抵抗器を使用してください。一時的なテストには別の抵抗を使用できますが、長期間使用すると信頼性に影響する可能性があります。抵抗は 3 kΩ 以上である必要があり、出力を 24 V に直接接続しないでください。

- 接続を確認した後、産業用コンピュータの出力端子を High に引き上げ、出力端子とアース間の電圧をマルチメータを使用して測定します。電圧が 2V 以下の場合は、カメラ回路を切断し、産業用コンピュータが電圧を高くできるかどうかを確認します。電圧が 2V を超える場合は、出力端子を再度 Low にして、S2 カメラの入力に変化があるかどうかを確認します。変化がある場合は、通常どおり使用できることを意味します。変化がない場合は、産業用コンピュータの出力端子を Low にして、出力端子とアース間の電圧を測定します。電圧が1Vより大きい場合は、プルアップの強さが強すぎてプルダウンするのが難しいことを意味します。プルアップ抵抗の抵抗値を大きくし、プルダウン後に S2 カメラの入力に変化があるかどうかを測定する必要があります。電圧が 1V 未満の場合は、産業用コンピュータを取り外し、カメラの IN\_\* を抵抗を介して 24V と GND に接続し、カメラの入力に変化があるかどうかを確認します。変更があった場合は、産業用コンピュータとカメラ間の接続が正しいかどうかを確認してください。変化がない場合は、テクニカル サポートに連絡して問題のトラブルシューティングを行ってください。

## 8. よくある質問

### 8.1 黒い物体

TOF 原理により、S シリーズ カメラは黒いパレットの反射率の影響を受けます。黒いパレットの反射率が 5% であるため、S2max カメラを使用して 2m の範囲内のイメージングを実現できます。デバッグ中に、ホスト コンピューター ソフトウェアの高い統合時間と低い信号しきい値によって、黒いパレットのイメージング効果を最適化することもできます。障害物回避シーンに黒いフォークアームがある場合は、フォーク先端の前の小さな部分に他の色の塗装面をスプレーするなど、障害物回避を支援するためにフォーク先端に高い反射率を追加することをお勧めします。

### 8.2 低地物体

地上の物体は障害物を避けることができません。障害物回避アプリケーションでは、通常、外部パラメータ設定の基礎として地面を使用します。地上で低い障害物に遭遇したときに障害物を回避できない場合は、次の解決策を参照してください。

- ホスト コンピュータの点群画像では、障害物が選択されていないことがわかります。障害物回避高さ（下）が正の値であるかどうかを確認する必要があります。正の値の場合は、障害物が検出できるかどうかを判断するために一時的に 0 に設定することをお勧めします。それでも検出できない場合は、カメラの設置高さを調整することをお勧めします。

・ホストコンピュータの点群画像では障害物を選択できますが、障害物回避結果の出力が0となります。この現象は、カメラの設置高さの設定が正しくないことが原因である可能性があります。カメラの設置高さはユーザーが調整することをお勧めします。

### 8.3 障害物回避パラメータが有効にならない

- パラメータを設定するときに [セットアップ完了] をクリックしませんでした。

・不正なパラメータを変更したため、設定に失敗しました。

- カメラの接続が不安定で切断され、パラメータの配信に失敗する。

### 8.4 設置構造の干渉

- 設置時にカメラの周囲にカメラの視野を遮る構造物があると、カメラデータが異常になります。

- 構造設計上、カメラのデザインが後方視界に干渉し、カメラデータに異常が発生する可能性があります。

- カメラミラーの保護フィルムが剥がされていないため、カメラデータに異常が発生しています。

- カメラのミラーに他の保護ガラスが取り付けられているため、データに異常が発生する可能性があります。

## インターフェースの説明

赤いメニューバー: クリックして、マッピングの開始、再配置、マップ管理など、対応する機能操作の詳細を入力します。

青いボックス: ロードされた 2D 等高線マップと自車位置表示アイコンを使用して表示されます。

緑色のボックス: トップビュー カメラのリアルタイム RGB 画像を表示します。

黄色のボックス: 再配置信号強度のリアルタイム表示 (緑: 優れた強度、青: 良好な強度、赤: 低強度)。車の現在位置座標、車の現在の移動速度、マウスの位置座標をリアルタイム表示

マッピングを開始するページです

次に、AGV を移動して、操作エリア内の地図をスキャンします。スキャンが完了したら、メニュー バーをクリックしてマップの構築を終了します。しばらくお待ちください (待ち時間はスキャン領域のサイズに関係します)。

マッピングが成功すると、マップ管理項目にマッピングデータが表示されます。

インクリメンタルマッピング

機能：地図の拡張や死角の補完など、既存の地図をベースに新しい地図エリアを追加

前提条件: 増分マップを構築するときは、Web ページの再配置が完了し、追加する必要があるマップに切り替えたことを確認する必要があります。

(当面は手動でHTTPリクエストを発行します)

アドレス [http://192.168.100.201:9998/tool/rob/sendCMD](http://192.168.100.201:9998/tool/rob/sendCMD "http://192.168.100.201:9998/tool/rob/sendCMD")

パラメータJson

```json
{

    "cmd":905, // 904 starts incremental mapping; 905 stops incremental mapping

    "attach":{

        "sub_command":0,

        "param":""

}

}
```

## 2. マップ管理

![2. 地図管理に関するイラスト](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/9b855ff4-d2b8-4a0b-9031-3b1343a848df.avif) 図2 マップ管理の概念図

マップの作成が完了したら、メニュー バーの [**マップ管理**] をクリックします。作成したばかりのマップの名前がホスト マップ リストに表示されます。 「適用」ボタンをクリックしてマップに切り替えます。同様に、切り替える地域地図が複数ある場合も地図管理機能を利用して切り替える必要があります。対応するマップを削除するには、「削除」ボタンをクリックします。 **マップの適用:** [適用] をクリックした後、マップが適用されて有効になる前に必ずホストを再起動してください。そうしないと、再配置ができなくなります。

## 3. 移転

再配置をクリックした後、マウスをドラッグして、再配置用のレーザー点群とレーザー等高線マップの一致するプロンプトに基づいて、おおよその再配置ポーズを与えることができます。

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/2e74ee5f-e192-4fce-8e8e-8f402988dcad.avif" alt="3. 移転関連アイコン" />
</Frame>

図4 再配置機能の概念図

再配置が成功すると、**測位タイムアウト** 例外は消えます。

QRコードの移設

マップを構築するときに、再配置する場所に Apritag コードを貼り付けることを選択できます。完全な QR コードがデバイスの視点に表示されると、コードがどの地図と位置にあるかを自動的に識別し、自動的に地図を切り替えて位置を変更します。

QRコード生成サイト [https://chaitanyantr.github.io/apriltag](https://chaitanyantr.github.io/apriltag "https://chaitanyantr.github.io/apriltag")

## 7. ファームウェアのアップデート

ローカルのパッケージを表示: 更新する必要があるローカル ファームウェア (シャーシ、アクチュエーター、カメラのファームウェア) をインポートします。

パッケージを復元: 最終更新前のファームウェア バージョンを記録します。戻る必要がある場合は、クリックしてください。

![7. ファームウェアアップデート関連のアイコン](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/db55e5fb-c925-4105-bef1-84ac12f05997.avif) 図 7 ファームウェアのアップデート

カメラホストファームウェアのアップグレード

通常のホスト コンピュータを使用してファームウェアをアップグレードできます。このアップグレード方法はロールバックできないため、慎重にアップグレードする必要があります。 ![7. ファームウェアアップデート関連アイコン2](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/4ec2ec9c-5285-4a4d-b91e-0fea462d9bed.avif)

## 8. メインアプリケーションを停止して再起動します。

適切なコントロールをクリックします。 LxCameraViewerをカメラに接続する前に、ロボットのメインアプリケーションを停止してください。それ以外の場合、ロボットはカメラの制御を保持し、LxCameraViewer が接続できなくなります。 ![メインアプリケーションコントロール](/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/6a00c674-61ac-4cde-8af7-c7e9389ccc6d.avif)

## 1. Einführung

### 1.1 Zweck und Umfang

Dieses Dokument beschreibt die Spezifikationen der S10-Kamera und ausgewählte Designdetails für Entwickler, die das Produkt integrieren und verwenden.

### 1.2 Terminologie

| RGBD-Kamera | Eine Kamera, die synchronisierte und räumlich ausgerichtete RGB- und Tiefenbilder ausgibt |
| --- | --- |
| Tiefe | Ein Videostream, der einem Farb- oder Monochrom-Stream ähnelt, wobei jedes Pixel seinen Abstand von der Kamera darstellt |
| Sichtfeld | Sichtfeld: Der Winkelbereich einer von einer Kamera erfassten Szene, einschließlich horizontaler (HFOV), vertikaler (VFOV) und diagonaler (DFOV) Sichtfelder |
| IR-Kamera | Eine Infrarotkamera; In diesem Produkt wird hauptsächlich das Nahinfrarotband erfasst |
| RGB-Kamera | Eine Farbkamera |
| Tiefenkamera | Eine Kamera, deren Tiefenmodul typischerweise einen Punktprojektor, eine IR-Kamera und einen Tiefenprozessor umfasst; sein Farbmodul ist typischerweise eine RGB-Kamera |
| SoC | System auf einem Chip zur Bildverarbeitung, Tiefenberechnung und Anwendungsalgorithmen |
| DTOF | Abstandserkennungstechnologie basierend auf Flugzeitmessung |

### 1.3 Funktionsprinzip

Bei diesem Produkt handelt es sich um eine DTOF-3D-Tiefenkamera mit großer Reichweite, guter Genauigkeit, geringen Kosten, starker Störfestigkeit und stabilen Daten. Ein integriertes RGB-Modul erfasst Farbbilder. Die Kamera richtet RGB- und Tiefenbilder intern aus.

## 2. Produktübersicht

Diese kompakte industrielle RGBD-Kamera liefert Tiefendaten von 0,3 m bis 8 m. Es richtet RGB- und Tiefenbilder räumlich und zeitlich aus. Sein integrierter SoC gibt Ergebnisse zu Tiefe, Intensität, RGB, Punktwolke und Hinderniserkennung aus. Zusätzlich zu Ethernet schalten zwei IO-Eingänge und zwei IO-Ausgänge zwischen vier Hindernisvermeidungszonen um und geben Notstopp-, Verlangsamungs- und Sicherheitszustandssignale aus.

### 2.2 Funktionen

- Niedriger DTOF-Stromverbrauch und kompakte Größe

- Integrierte Verarbeitung, die Hinderniserkennungsergebnisse für den Plug-and-Play-Betrieb ausgibt

- Starke Anpassungsfähigkeit an die Umwelt: Widersteht 100.000 Lux Sonnenlicht und funktioniert von der Dunkelheit bis zum hellen Licht, drinnen und draußen

- Großer Messbereich von 0,3–8 m

## 6. Schnellstart

### 6.1 Verbindungstest

<Frame>
  <img src="/images/legacy/cb764c04-73c2-4cdd-9381-98fe1c201737/24cb2351-298f-4381-8170-c34cf1444269.avif" alt="Verbindungstest" />
</Frame>

1. Schließen Sie die Kamera an eine 24V/2A-Gleichstromversorgung an. Eine langsam blinkende blaue Betriebsanzeige bestätigt das normale Einschalten.

2. Schließen Sie die Kamera mit einem Ethernet-Kabel der Kategorie 6 an den Computer an.

3. Die werkseitig voreingestellte IP-Adresse der Kamera lautet 192.168.100.82.

4. Stellen Sie die Computer-IP auf das Subnetz der Kamera ein und deaktivieren Sie die Computer-Firewall.

5. Führen Sie die Host-Software aus, um die Kamerabilder anzuzeigen.

6. Wenden Sie sich an den MRDVS-Vertrieb oder den technischen Support, um die Host-Software zu erhalten.

## 8. Fehlerbehebung

| Nein. | Problem | Auflösung |
| --- | --- | --- |
| 1 | Kamera reagiert beim Öffnen nicht | Stellen Sie sicher, dass die Firewall deaktiviert ist |
| 2 | Host-Software erkennt mehrere IP-Adressen | Ein anderes Gerät im LAN verwendet dieselbe IP; Wählen Sie die gewünschte IP-Adresse aus |
| 3 | Kameradaten sind nach dem Öffnen instabil | Verwenden Sie ein Gigabit-Ethernet-Kabel. Daten können über Fast Ethernet zunächst instabil sein |
| 4 | Software-Installationsort | Eine Standardinstallation auf Laufwerk C kann von den Berechtigungseinstellungen | betroffen sein

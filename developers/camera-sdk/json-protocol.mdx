## 概述

相机作为 TCP Socket 服务端，默认监听端口为 `5501`。报文为文本格式，支持 ASCII 字符；建议使用 JSON 编码。

<Info>
  本协议用于机器人或 PLC 触发 3D 检测，并接收相机返回的检测结果。
</Info>

## 请求报文

向相机发送以下报文以触发 3D 视觉任务：

```json
{"action":"3D"}
```

## 响应报文

相机完成检测后返回 JSON 对象。

<ResponseField name="result" type="integer" required>
  检测状态。值为 `0` 表示定位失败、无结果；其他整数表示识别成功，数值为返回结果数量。
</ResponseField>

<ResponseField name="pos" type="number[]" required>
  被测物位姿数组。每组结果依次为 `X`、`Y`、`Z`、`RX`、`RY`、`RZ`，采用机械臂坐标系。位置单位为 mm，角度单位为 °，且 `−180° ≤ RZ ≤ 180°`。检测失败时，各数值均为 `0`。
</ResponseField>

### 响应示例

<Tabs>
  <Tab title="检测成功">

```json
{"result":2,"pos":[100,107.3,1530,179,89,89,112,432.1,1500,179,89,75]}
```

  </Tab>
  <Tab title="检测失败">

```json
{"result":0,"pos":[0,0,0,0,0,0]}
```

  </Tab>
</Tabs>

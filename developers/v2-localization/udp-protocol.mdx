V2 Pro 可通过 UDP 与客户端交换控制和定位数据。在 Web 上位机中配置目标设备地址：portName 为目标设备 IP 地址，portCode 为目标设备监听端口。V2 Pro 监听 8001 端口接收外部报文。

## 报文约定

| 项目 | 约定 |
| --- | --- |
| 字节序 | 大端序：高字节在前，低字节在后。 |
| 帧结构 | 帧头、长度、命令字、数据和校验码。 |
| 校验码 | 1 字节；对除帧头和校验码本身外的全部字节进行异或。 |
| 长度字段 | 不包含帧头和校验码；部分报文不含长度字段。 |
| 地图名 | 仅包含英文字母、数字和下划线，且必须以英文字母开头。 |

<Note>
  以下字节编号从 Byte 0 开始。多字节整数均按大端序传输。
</Note>

## V2 Pro 接收的报文

### 里程计信息

| 字节范围 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0x0A |
| Byte 4–7 | V3 V2 V1 V0，4 字节整型速度 |
| Byte 8–11 | Theta3 Theta2 Theta1 Theta0，4 字节整型角度 |
| Byte 12 | 校验码 |

速度 v 的单位为 mm/s；角度 theta 的单位为 0.01°，例如 90° 发送为 9000。

### 重定位

| 字节范围 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0x02 |
| Byte 4–7 | X3 X2 X1 X0，4 字节整型 X 坐标 |
| Byte 8–11 | Y3 Y2 Y1 Y0，4 字节整型 Y 坐标 |
| Byte 12–15 | Theta3 Theta2 Theta1 Theta0，4 字节整型角度 |
| Byte 16 | 校验码 |

x、y 的单位为 mm；theta 的单位为 0.01°，例如 90° 发送为 9000。

### 开始或结束建图

| 字节范围 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0x02 |
| Byte 4 | 状态：0x01 开始建图，0x00 结束建图 |
| Byte 5–34 | 地图名，固定 30 字节 ASCII；不足部分以 0x00 填充 |
| Byte 35 | 校验码 |

一次建图任务仅发送一次开始建图指令；不要连续下发。

### 切换地图

| 字节范围 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0x0C |
| Byte 4–33 | 地图名，固定 30 字节 ASCII；不足部分以 0x00 填充 |
| Byte 34 | 校验码 |

下发前确认目标地图已完成建图。

## V2 Pro 发送的报文

### 定位结果

| 字节范围 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0x01 |
| Byte 4–7 | X3 X2 X1 X0，X 坐标 |
| Byte 8–11 | Y3 Y2 Y1 Y0，Y 坐标 |
| Byte 12–15 | Theta3 Theta2 Theta1 Theta0，姿态角 |
| Byte 16–23 | Timestamp7 至 Timestamp0，时间戳 |
| Byte 24 | 校验码 |

X、Y 的单位为 mm，THETA 的单位为 0.001 rad。重定位成功后，V2 Pro 会持续发送该定位报文。

### 任务执行结果

开始建图、结束建图和切换地图均通过以下结果报文返回状态。

| 字节 | 内容 |
| --- | --- |
| Byte 0–1 | 帧头：AC ED |
| Byte 2 | 长度：Length |
| Byte 3 | 命令：0xFF |
| Byte 4 | 任务 ID |
| Byte 5 | 状态：0x01 成功，0x00 失败 |
| Byte 6 | 校验码 |

| 任务 ID | 操作 |
| --- | --- |
| 0x01 | 开始建图 |
| 0x02 | 结束建图 |
| 0x03 | 切换地图 |

<Info>
  结束建图为长耗时操作。完成时间与实际采集时长相关，通常约为采集时长的三分之一；等待任务结果报文后再执行后续操作。
</Info>

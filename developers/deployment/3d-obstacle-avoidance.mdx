3D 视觉避障在相机内执行目标检测、地面适应和避障状态输出。完成相机外参和区域配置后，可通过上位机、API 或设备支持的现场总线切换避障参数组并读取结果。

<Info>
  本页中的示例参数仅用于说明配置关系。实际值取决于车型、安装位置和工作区域，应在现场验证后下发。
</Info>

## 部署前准备

### 相机安装

| 车辆类型 | 安装建议 |
| --- | --- |
| 潜伏式 AMR | 将相机安装在行驶方向前表面；安装结构不得遮挡视野。推荐使用 S10。 |
| 叉车 | 将相机安装在门架，镜头向下约 `30°–55°`；视野边缘应落在车体前表面，安装高度建议不超过 `2 m`。推荐使用 S10。 |

### 连接相机

<Steps>
  <Step title="配置本机网络">
    相机出厂地址为 `192.168.100.82`。将电脑网口配置为同一网段。
  </Step>
  <Step title="连接并确认成像">
    打开 **LxCameraViewer**，扫描并打开相机，确认深度图和点云正常显示。
  </Step>
  <Step title="按需修改相机地址">
    在 **其他工具 → 网络相关 → IP 配置工具** 中修改 IP。修改后相机会重启；如跨网段，请同步调整电脑网口地址。
  </Step>
</Steps>

<Warning>
  不要为此永久关闭生产网络的防火墙。仅在受控的调试网络中，按企业安全策略放行必要的程序和端口。
</Warning>

### 调整相机成像

在 **3D 设置** 和 **滤波设置** 中确认成像质量。旧版避障配置可参考高曝光 `650`、低曝光 `200` 和低信号阈值 `10–20`；S10 通常只需将炫光等级设为 `1`。非标准安装方式请先联系技术支持确认。

## 配置避障算法

<Steps>
  <Step title="启用算法">
    在 **应用算法** 中选择常开模式。S10 选择 **避障算法**；其他支持机型选择 **避障算法 2.0**。
  </Step>
  <Step title="选择区域类型">
    根据车型选择矩形或扇形区域。S10 不支持扇形区域，使用后可能导致结果偏差。
  </Step>
  <Step title="设置外参与区域">
    先完成外参设置，再配置避障区域。更新外参后，其余区域会同步使用当前外参。
  </Step>
  <Step title="下发并验证">
    下发参数后，在点云视图和输出状态中验证预警、报警与屏蔽区。
  </Step>
</Steps>

### 相机外参

| 参数 | 示例值 | 说明 |
| --- | ---: | --- |
| Roll（X 轴） | `180°` | 相机姿态参数；默认正装示例值。 |
| Pitch（Y 轴） | `90°` | 相机姿态参数；默认垂直安装示例值。 |
| Yaw（Z 轴） | `90°` | 相机姿态参数。 |
| X 位置 | `430 mm` | 相机相对 AMR 前后方向的位置。 |
| Y 位置 | `0 mm` | 相机相对 AMR 左右方向的位置；居中安装时为 `0`。 |
| Z 位置 | `120 mm` | 相机距地高度。 |

先调整安装高度，再依次微调滚转与俯仰，使地面在点云视图中保持水平。完成后点击 **下发参数**。

### 避障区域

| 参数 | 示例值 | 作用 |
| --- | ---: | --- |
| 预警距离（X 远） | `2000 mm` | 超出该距离视为安全；与报警距离之间为预警减速区。 |
| 报警距离（X 中） | `1500 mm` | 与屏蔽距离之间为报警停止区。 |
| 屏蔽距离（X 近） | `600 mm` | 近端盲区；该区域不参与避障。 |
| 避障范围左 / 右（Y） | `−810 / 1270 mm` | 车辆中心线两侧的检测边界。 |
| 避障高度下 / 上（Z） | `10 / 1500 mm` | 参与检测的高度范围。 |

<Tip>
  区域的 X 值相对于车辆中心解释。实际保护距离需要同时考虑相机 X 位置，例如预警距离减去相机前后偏移。
</Tip>

支持的区域组数如下：

| 切换方式 | 支持的区域组 |
| --- | ---: |
| S2 / S2 Max 2.0 的 I/O | `0–3` |
| SDK、UDP、RS485、CAN | `0–19`（取决于机型） |

可在 `Tools\\params\\<相机 ID>\\avoidance.json` 备份已验证的避障参数，并通过 **从文件加载配置** 应用于同类车辆。

## 通信与状态输出

在使用现场总线或网络协议前，先通过上位机完成避障区域和外参配置。除 API 外，大多数通信方式仅用于切换参数组；详细参数应使用上位机或 API 设置。

| 相机类型 | 支持的通信方式 |
| --- | --- |
| S2 | API、UDP、I/O |
| S2 Max | API、UDP、CAN |
| S2 Max 2.0 | API、UDP、I/O、CAN |
| M4 Pro | API、UDP、RS485 |
| M4 | API、UDP、CAN |
| M4 Mega | API、UDP、I/O、CAN |

<Note>
  I/O 的优先级最高。I/O 有效时，其他方式不能改变当前参数组，但仍可更新默认参数或临时参数。上位机和 API 设置默认参数；UDP、CAN、RS485 等通常设置临时参数。断电后的行为取决于最后一次保存方式。
</Note>

### API

Windows、Linux 和 ARM 环境可通过 SDK 调用避障接口；安装 Windows 上位机后，SDK 和示例位于安装目录。避障示例位于 `Sample\\C\\application_obstacled`。S10 不建议通过 API 高频切换参数组，宜使用 UDP 或 I/O。

### RS485（仅 M4 Pro）

使用 Modbus RTU，建议波特率为 `9600`。读取输入寄存器（功能码 `0x04`）的地址 `108`（`0x6C`）起始的两个寄存器可获取当前避障结果；写单寄存器（功能码 `0x06`）地址 `106`（`0x6A`）可切换参数组。

| 引脚 | 信号 | 线色 |
| ---: | --- | --- |
| 1 | GND | 蓝 |
| 2 | RS485_A | 灰 |
| 3 | RS485_B | 白 |
| 4 | VIN+ | 棕 |
| 5 | VIN− | 黑 |

### UDP 与 TCP

使用十六进制报文传输。UDP 首次通信先发送建立连接报文；建立后，相机按图像帧率发送结果。参数切换和结果报文如下。

| 方向 | 报文 | 含义 |
| --- | --- | --- |
| 相机 → 控制器 | `ACED0410XX000000` | 无障碍；`XX` 为当前参数组。 |
| 相机 → 控制器 | `ACED0410XX000001` | 预警减速。 |
| 相机 → 控制器 | `ACED0410XX000002` | 报警停止。 |
| 控制器 → 相机（UDP） | `ACED040000000000` | 建立 UDP 连接。 |
| 控制器 → 相机 | `ACED041A00000000` 至 `ACED041A00000013` | 切换参数组 `0–19`。 |
| 控制器 → 相机 | `ACED041B00000000` | 保存当前参数组为默认参数。 |

UDP 默认远程端口为 `6688`。相机切换参数需要加载时间，避免高频切换；频繁切换会降低图像帧率和避障结果刷新率。

### CAN 与 I/O

CAN 可用于支持机型的参数组切换。I/O 输出包含无障碍、减速和急停状态；输出类型可能为推挽或开漏，接线前应确认控制器的输入类型。

| 推挽输出 OUT1 | 推挽输出 OUT2 | 状态 |
| --- | --- | --- |
| `24 V` | `24 V` | 无障碍 |
| `24 V` | `0 V` | 减速 |
| `0 V` | `24 V` | 急停 |
| `0 V` | `0 V` | 未启用、初始化或异常 |

<Warning>
  S10 的 `24 V−` 应连接控制器电源地，`IN_G` 和 `OUT_G` 应连接控制器信号地。若控制器未区分两类地线，可按现场接地方案共地。接线前确认控制器为 PNP 或 NPN 输出，并进行万用表验证。
</Warning>

## 排查

| 现象 | 检查项 |
| --- | --- |
| 黑色或低反射物体漏检 | 确认点云覆盖；可调整积分时间和低信号阈值。黑色叉尖可增加高反射辅助表面。 |
| 低矮物体未触发 | 检查 Z 下限；可临时设为 `0` 验证。若仍无检测，检查安装高度与点云覆盖。 |
| 参数未生效 | 确认已下发；检查参数合法性、网络连接和当前参数组优先级。 |
| 数据异常 | 检查安装结构遮挡、镜头保护膜和额外保护玻璃。 |

本页说明 MRDVS 人体检测、库位检测和体积检测算法的部署、标定、参数配置与结果集成方式。算法在相机端运行；算法上位机用于配置、升级和查看结果。

<Info>
  算法固件和算法上位机按项目交付。刷写算法固件前，确认相机已使用 MDS 版本固件；请向技术支持获取与项目匹配的安装包。
</Info>

## 支持范围

### 系统组成

| 组件 | 作用 |
| --- | --- |
| 3D 相机 | 采集 RGB 图像和点云，并在相机端执行检测算法。 |
| 算法上位机 | 配置算法、标定外参、查看结果、升级算法固件。 |
| 工控机或上位系统 | 运行上位机软件，或通过网络接口读取检测结果。 |
| LiDAR | `mid360`，用于原始方案中的雷达配置。 |

### 算法与相机

| 算法 | 支持的相机 |
| --- | --- |
| 人体检测 | M4 Pro |
| 库位检测 | S10 Pro |
| 体积检测 | S10、S10 Pro |

S10 与 S10 Pro 的产品资料请参阅 [S10 3D 相机产品页](/products/s-series/s10-overview)。

### 上位机运行环境

| 项目 | 建议配置 |
| --- | --- |
| 操作系统 | Windows 10 或更高版本 |
| CPU | Intel® Core™ i5（第 7 代）或更高版本 |
| 内存 | 8 GB 或更高 |
| GPU | Intel® UHD Graphics 或更高性能的核显 / 独立显卡 |

## 硬件与网络部署

### 安装相机

使用刚性支架固定相机，避免安装完成后发生位移或振动。人体检测通常采用水平安装，或倾角不超过 30° 的安装方式。

  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/ed7d4ff9-ebc7-456e-9382-c1bbad17757e.avif" alt="相机支架安装示意图" />
</Frame>

### 连接设备

<Steps>
  <Step title="完成供电和网络连接">
    使用 24 V 稳压直流电源为相机供电，并使用超六类或七类网线连接相机与千兆交换机或主控设备。
  </Step>
  <Step title="部署多台相机">
    按 CAD 规划的安装高度与位置固定相机。多台相机通过千兆交换机接入网络，并分别确认供电与链路状态。
  </Step>
  <Step title="准备现场目标">
    在相机视野内放置待检测的人员、托盘或货物，再进行外参标定和检测区域配置。
  </Step>
</Steps>

### 配置主机网络

将运行上位机的计算机与相机配置到同一网段。原始示例使用以下地址规划：

| 项目 | 示例值 |
| --- | --- |
| 主机 IP 地址 | `192.168.100.xxx` |
| 子网掩码 | `255.255.255.0` |
| 默认网关 | `192.168.100.1` |

<Steps>
  <Step title="打开网络适配器设置">
    在 Windows 中依次打开 **控制面板** → **网络和共享中心** → **更改适配器设置**，选择连接相机的以太网适配器。
  </Step>
  <Step title="配置 IPv4">
    打开 **Internet 协议版本 4 (TCP/IPv4)**，选择 **使用下面的 IP 地址**，填写与相机同网段且未被占用的静态地址。
  </Step>
  <Step title="确认连接">
    保存设置后，确认网卡为千兆链路，并使用上位机搜索或打开相机。
  </Step>
</Steps>

## 安装和升级软件

### 安装上位机

1. 从 [软件下载](https://zcnt5g3o2989.feishu.cn/wiki/Q8IYwGiGaiIvIyktiavczymanOe?from=from_copylink "软件下载") 获取相机上位机软件。
2. 安装相机上位机时，选择系统盘以外的目录。
3. 将项目交付的算法上位机文件复制到相机上位机安装目录下的 `Tools` 文件夹；替换同名文件后，双击可启动程序。

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/c8922d3a-2b74-4582-9b3d-119ba6be460d.avif" alt="相机上位机软件安装流程" />
</Frame>

算法上位机仅升级时，替换交付的 `.exe` 文件即可。

### 升级算法固件

<Warning>
  刷写过程中相机会重启。不要断开电源、网络或关闭上位机；升级包必须与相机型号、MDS 固件版本和项目算法匹配。
</Warning>

<Tabs>
  <Tab title="相机上位机">
    1. 启动 `LxCameraViewer.exe` 并打开相机。
    2. 依次选择 **基础工具** → **功能设置** → **固件升级**。
    3. 选择交付的检测算法文件，点击 **执行** 开始刷写。
  </Tab>
  <Tab title="算法上位机">
    1. 在左侧设备列表选择目标相机。
    2. 点击 **升级固件**，选择检测算法文件并打开。
    3. 等待相机重新出现在设备列表中。
  </Tab>
</Tabs>

相机重启后再次出现在设备列表中，表示刷写完成。算法上位机中相机背景显示为绿色，表示相机已连接且算法可用。

| 功能 | 说明 |
| --- | --- |
| RGB | 显示 RGB 图像。 |
| 点云 | 显示点云。 |
| 停止算法 | 停止当前算法；在下次相机上电前不会自动启动。 |

若设备背景为黄色，或背景为绿色但图像不更新，可启用自动刷新以恢复图像获取。

## 外参标定

### 相机配置项

| 配置项 | 说明 |
| --- | --- |
| 相机名称 | 用于区分多台相机。 |
| 仿真模式 | 内部功能，用户无需启用。 |
| 存图模式 | 在相机端保存算法图像数据。 |
| 取图模式 | `MDS` 不占用相机访问权限；`SDK` 会占用权限，算法运行时无法同时打开相机上位机。 |
| `x`、`y`、`z` | 相机外参位置；`z` 为相机距地面的高度。 |
| `roll`、`pitch`、`yaw` | 相机外参姿态角。 |
| 理想地面 | 保持默认值即可。 |
| 从相机读取配置 | 读取当前相机配置。 |
| 保存并下发 | 将修改写入相机；未下发的参数不生效。 |
| 外参标定 | 标定相机高度与姿态角。 |

## 部署算法

切换到点云模式后，将鼠标置于图像区域并滚动滚轮，可显示点云数据。

### 人体检测

<Steps>
  <Step title="完成水平安装标定">
    将 `X`、`Y` 轴与地面点云重合，`Z` 轴垂直地面并朝上。
  </Step>
  <Step title="设置全局参数">
    打开 **人体检测**。通常只需设置 **旋转检测**；相机顺时针方向仅支持 `0°`、`90°`、`180°` 和 `270°`。
  </Step>
  <Step title="创建检测区域">
    点击 **增加检测区域**，设置最小 / 最大 `X`、`Y`、`Z`。原始示例创建了 `1000 × 1000 × 2000 mm` 的区域。使用鼠标将区域移动到目标位置。
  </Step>
  <Step title="保存并验证">
    点击 **保存并下发**，在区域内进行人员通行测试，并通过状态栏或历史回放验证结果。
  </Step>
</Steps>

| 匹配规则 | 无人 | 有人 | 有障碍物 | 人与障碍物 |
| --- | --- | --- | --- | --- |
| 仅深度 | `0` | `2` | `2` | `2` |
| 语义加深度 | `0` | `1` | `0` | `1` |

仅深度模式会报告检测区域内的所有物体；语义加深度模式只报告语义分类为人的聚类。

### 库位检测

<Steps>
  <Step title="标定相机">
    按照外参标定步骤，将 `X`、`Y` 轴与地面重合，并使 `Z` 轴垂直向上。
  </Step>
  <Step title="配置全局参数">
    根据现场需求设置 **对齐校验**、**对齐角度** 和 **最小点数量**。若货物必须按指定角度放置，启用对齐校验。
  </Step>
  <Step title="创建库位">
    在点云模式下，点击 **增加库位**，切换到俯视图，按键盘 `X` 开始或结束绘制；拖动矩形框覆盖托盘并调整 `X`、`Y`、`Z` 边界。绘制后必须再次按 `X` 退出绘制模式。
  </Step>
  <Step title="保存并验证">
    点击 **保存并下发**。可使用 2D RGB 或 3D 点云方式配置库位，并检查状态栏中的 `Result`。
  </Step>
</Steps>

  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/846b9c20-37c5-4847-b10d-82b766443aab.avif" alt="点云模式下绘制库位检测区域" />

| 库位状态 | `Result` |
| --- | --- |
| 无物体 | `0` |
| 有物体，或有物体且对齐 | `1` |
| 有物体且角度超过对齐阈值 | `2` |
| 有物体且超出库位区域 | `3` |

若库位布局规则，可使用 **拷贝当前框** 按固定方向和间距批量创建库位。需要二维码校验时，使用 **QR Code 标定** 配置库位尺寸与深度范围。

### 体积检测

<Steps>
  <Step title="完成垂直安装标定">
    体积检测从上向下工作。完成标定后，`Z` 轴应垂直向下，`X`、`Y` 轴组成的平面应与地面平行。
  </Step>
  <Step title="设置检测平面">
    货物直接放在地面时，将检测平面设置为地面高度；只计算托盘上货物时，使检测平面与托盘点云重合。
  </Step>
  <Step title="设置最小尺寸与检测范围">
    设置最小 `X`、`Y`、`Z` 以过滤小目标；再根据检测平面和货物尺寸设置区域的 `X`、`Y`、`Z` 边界。
  </Step>
  <Step title="保存并验证">
    点击 **保存并下发**，在检测区域内放置单个或多个物体，确认返回的长、宽、高和数量。
  </Step>
</Steps>

### 传输方式

| 协议 | 端口 | 请求内容 | 用途 |
| --- | --- | --- | --- |
| TCP | `14951` | `RESULT_UPDATE` | 获取当前算法结果。 |
| UDP | `14950` | `RESULT_UPDATE` | 获取当前算法结果。 |
| HTTP | `14952` | `RESULT_UPDATE` | 获取 JSON 格式的算法结果。 |
| TCP / UDP | 对应结果端口 | `TM_TRIGGER` | 在体积检测的触发模式下执行一次检测并返回结果。 |

### Modbus TCP（库位检测）

检测结果按库位 ID 排序，最多返回前 16 个库位。

| 配置项 | 值 |
| --- | --- |
| 端口 | `16502` |
| 监听地址 | `0.0.0.0`（所有网络接口） |
| 寄存器数量 | `16` |
| 起始地址 | `0x0100` |
| 字节序 | 大端序（Big Endian） |

| 寄存器 | 内容 | 数据格式 |
| --- | --- | --- |
| `0x0100 + ID` | 库位检测结果与标记物检测结果 | 2 字节 `uint16_t` |
| 高 8 位 | 库位状态码 | `0–2` |
| 低 8 位 | 标记物状态码 | `0–2`；该功能暂未开放，当前含义与库位状态码一致。 |

| 库位状态码 | 含义 | 触发条件 |
| --- | --- | --- |
| `0` | 未占用 | 无货物或托盘。 |
| `1` | 正常占用 | 货物按要求放置。 |
| `2` | 异常占用 | 货物尺寸或位置偏移。 |

| 标记物状态码 | 含义 |
| --- | --- |
| `0` | 未检测到标记物。 |
| `1` | 检测到标记物。 |
| `2` | 标记物检测异常。 |

#### 读取第 4 个库位

以下示例使用事务标识符 `0x0001`，读取地址 `0x0104` 的一个保持寄存器。响应中的状态数据为 `0x0002`。

```text
请求：00 01 00 00 00 06 FF 03 01 04 00 01
响应：00 01 00 00 00 05 FF 03 02 00 02
```

| 字段 | 请求值 | 说明 |
| --- | --- | --- |
| 事务标识符 | `00 01` | 客户端递增序号；响应应保持一致。 |
| 协议标识符 | `00 00` | Modbus/TCP 固定值。 |
| 长度 | `00 06` | 请求中后续字段的字节数。 |
| 单元标识符 | `FF` | 广播模式或设备地址。 |
| 功能码 | `03` | 读取保持寄存器。 |
| 起始地址 | `01 04` | `0x0100 + 4`。 |
| 寄存器数量 | `00 01` | 读取 1 个寄存器。 |

## 排查要点

| 现象 | 优先检查项 |
| --- | --- |
| 找不到相机 | 主机与相机是否同网段、网卡是否为千兆链路、供电与交换机链路是否正常。 |
| 参数未生效 | 是否在修改后点击 **保存并下发**。 |
| 无算法结果 | 算法固件是否正确刷写、设备是否显示为绿色、算法是否已启用。 |
| 点云为空或不更新 | 是否已在点云区域滚动鼠标显示点云；必要时启用自动刷新。 |
| 库位或体积结果异常 | 重新检查外参坐标方向、检测平面和区域边界，再使用现场目标复测。 |

体积测量算法在相机端运行，用于静态或动态物体的长、宽、高测量。部署时需要完成固定安装、网络连接、外参标定和检测区域配置；多相机系统通过千兆交换机接入同一网络。

<Info>
  本页为隐藏的历史部署文档。算法固件和专用上位机按项目交付；部署前请确认相机、固件和软件版本匹配。
</Info>

## 适用范围

该流程适用于使用 M 系列相机的物流、快递、仓储和散料堆体等体积测量项目。算法可测量检测区域中的单个或多个物体；物体识别、重量采集和业务系统关联由上层系统实现。

产品资料请参阅：[M 系列 ToF 相机](/products/m-series/overview "M系列ToF相机")。

## 安装与系统准备

### 硬件安装

- 使用刚性支架固定 M4 相机，使镜面尽量与检测平面平行并朝向下方。
- 使用 `24 VDC` 供电；使用超六类或七类网线连接相机与主机或千兆交换机。
- 多相机部署时，为每台相机确认独立的网络地址和供电状态。

### 上位机要求

| 项目 | 建议配置 |
| --- | --- |
| 操作系统 | Windows 10 或更高版本 |
| CPU | Intel® Core™ i5（第 7 代）或更高版本 |
| 内存 | 8 GB 或更高 |
| GPU | Intel® UHD Graphics 或更高性能的核显 / 独立显卡 |

先安装 [LxCameraViewer](/developers/software/lxcameraviewer "软件下载")，再将项目交付的体积测量上位机放入 `Lanxin-MRDVS\\Tools` 目录。若系统提示替换文件，按交付说明完成替换。

<Warning>
  升级固件会使相机重启。仅使用项目交付的算法包，并在升级前备份已验证的配置。
</Warning>

## 部署流程

### 1. 配置网络

将上位机与相机配置到同一网段。原始部署示例使用：

| 项目 | 示例值 |
| --- | --- |
| 主机 IP | `192.168.100.xxx` |
| 子网掩码 | `255.255.255.0` |
| 默认网关 | `192.168.100.1` |

在 Windows 的以太网适配器属性中选择 **Internet 协议版本 4 (TCP/IPv4)** 并设置静态地址。调试网络的防火墙规则应允许软件通信；不要为此长期关闭生产网络的防火墙。

### 2. 标定外参

<Steps>
  <Step title="恢复待标定状态">
    打开 **相机配置**。确认 `x`、`y`、`z`、`roll`、`pitch` 和 `yaw` 均为 `0`；如有历史值，归零后点击 **保存并下发**。
  </Step>
  <Step title="执行标定">
    点击 **外参标定**，确认软件返回成功提示，并记录标定得到的 `z` 值。
  </Step>
  <Step title="保存姿态参数">
    将 `z` 改回 `0` 后点击 **保存并下发**。本流程中，实际相机高度将在下一步作为检测平面距离设置。
  </Step>
</Steps>

### 3. 设置检测平面和范围

在 **体积检测** 页面，将标定记录的 `z` 值填入 **检测平面距离**，然后保存并下发。接着按点云视图设置检测区域边界：

| 参数 | 含义 |
| --- | --- |
| 最小 / 最大 X | 检测框的 X 方向边界 |
| 最小 / 最大 Y | 检测框的 Y 方向边界 |
| 最小 / 最大 Z | 检测框的 Z 方向边界 |

检测平面距离决定地面参考；Z 上限应略高于该距离。检测下限应覆盖需要测量的物体高度。设置完成后点击 **保存并下发**。

### 4. 验证结果

在 **基础工具** 中查看体积状态。将物体放入检测区域后，确认物体数量和长、宽、高输出符合现场情况。多物体时可切换体积盒查看各对象结果。

若使用单次测量，在 **体积检测** 中启用触发模式并保存，然后在 **基础工具** 中点击 **单次触发** 获取一帧图像和结果。

## 关键界面与参数

### 体积检测参数

| 参数 | 说明 |
| --- | --- |
| 触发模式 | 开启后，算法仅在收到触发时输出结果。 |
| 向上检测 | 改变检测方向；当前默认流程为向下检测。 |
| 检测频率 | 图像与结果的刷新周期。 |
| 检测平面距离 | 相机到检测平面的距离，单位为 `mm`。 |
| 邻域数量 / 距离阈值 | 点云滤波参数；通常保留项目默认值。 |
| 法向量滤波 / 距离阈值 | 边缘与点云滤波参数；改动前应保留原配置。 |
| 最小点数量 | 物体被判定为有效所需的最少点数。 |
| 最小 X / Y / Z | 可检测物体的最小尺寸，单位为 `mm`。 |

### 相机配置

| 项目 | 说明 |
| --- | --- |
| 相机名称 | 用于区分多相机设备。 |
| 存图模式 | 在相机内保存算法图像。 |
| 取图模式 | `MDS` 不占用相机权限；`SDK` 会占用相机权限，运行时无法同时打开上位机。 |
| 外参 | `x`、`y`、`z`、`roll`、`pitch`、`yaw`。 |
| 保存并下发 | 将当前外参和算法配置写入相机。 |

## 结果接口

TCP、UDP 和 HTTP 均使用命令 `RESULT_UPDATE` 请求体积结果。

| 协议 | 端口 | 用途 |
| --- | ---: | --- |
| TCP | `14951` | 请求体积检测结果 |
| UDP | `14950` | 请求体积检测结果 |
| HTTP | `14952` | 请求 JSON 格式的体积检测结果 |

<ParamField body="command" type="string" required default="RESULT_UPDATE">
  请求相机返回当前体积检测结果。
</ParamField>

HTTP 响应的顶层字段：

<ResponseField name="IP" type="string" required>
  数据来源相机的 IP 地址。
</ResponseField>

<ResponseField name="TM" type="object[]" required>
  检测到的物体列表。
</ResponseField>

<ResponseField name="TM_DATA_TIME" type="string">
  体积检测数据提取时间。
</ResponseField>

<ResponseField name="TM_RES_TIME" type="string">
  结果输出时间。
</ResponseField>

<ResponseField name="nick_name" type="string">
  相机名称。
</ResponseField>

<ResponseField name="ret_code" type="integer" required>
  返回码；`0` 表示成功。
</ResponseField>

每个 `TM` 对象可包含：

<ResponseField name="angle" type="number">
  物体框选角度。
</ResponseField>

<ResponseField name="cx" type="number">
  物体中心 X 坐标。
</ResponseField>

<ResponseField name="cy" type="number">
  物体中心 Y 坐标。
</ResponseField>

<ResponseField name="cz" type="number">
  物体中心 Z 坐标。
</ResponseField>

<ResponseField name="height" type="number">
  物体高度。
</ResponseField>

<ResponseField name="length" type="number">
  物体长度。
</ResponseField>

<ResponseField name="width" type="number">
  物体宽度。
</ResponseField>

<ResponseField name="valid" type="boolean">
  结果有效性；置信度过低时为 `false`。
</ResponseField>

## 排查

| 现象 | 处理建议 |
| --- | --- |
| 点云视图为空或难以查看 | 在点云区域滚动和拖动视图；确认相机已正常出图。 |
| 标定后检测异常 | 确认外参中的 `z` 已归零，且检测平面距离已填入记录的标定高度。 |
| 未检出物体 | 检查检测框边界、最小尺寸、最小点数量和物体是否完整进入视野。 |
| 无法打开 LxCameraViewer | 检查是否选择了 `SDK` 取图模式；该模式会占用相机权限。 |

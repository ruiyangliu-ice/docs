## Prefacio

Este artículo presenta la arquitectura de software del módulo de posicionamiento visual, el método de configuración del SDK de ROS y el proceso básico de operación de mapeo y posicionamiento.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/7ed197c5-c599-4d93-af40-94afbc0a5507.avif" alt="Iconos relacionados con el prefacio" />
</Frame>

El diagrama esquemático de este módulo de posicionamiento se muestra arriba. Entre ellos, el programa de posicionamiento integrado de la cámara interactúa con el SDK ROS de la computadora host mediante UDP. Además, el programa de posicionamiento viene con una interfaz WEB configurable que puede realizar la visualización del posicionamiento (se puede abrir usando cualquier navegador convencional, la IP de acceso predeterminada es 192.168.100.201:9998). El front-end tiene dos funciones principales:

1. Ejecute la DEMO completa para completar el proceso de mapeo y posicionamiento;

2. Configure el ID de destino y el PUERTO para la comunicación con ROS SDK. De la misma manera, la IP y el PUERTO de la cámara de destino deben configurarse antes de iniciar ROS SDK.

Después de completar la configuración anterior, todas las funciones de la interfaz web front-end, como mapeo, reubicación y descarga de mapas, se pueden llamar a través del SDK de ROS.

## compilar

### Configuración del entorno de ejecución del programa

1. Instalar QT

Asegúrese de que QT5 se haya instalado en la computadora host. Necesita Core, Network y otros paquetes funcionales en QT5. Los métodos de instalación de diferentes versiones de Linux pueden ser diferentes. Aquí hay un ejemplo de instalación:

```bash
sudo apt install qtbase5-dev qtbase5-dev-tools libqt5network5
```

2. Instale el paquete de funciones libzip

Tutorial de instalación https://blog.csdn.net/iamhuanyu/article/details/129603632

3. Compile el paquete de funciones ROS. Ejecute `catkin_make` debajo de la ruta `/V2-ROS-SDK-service` (mismo nivel que `/src`).

### Depuración

1\. Método de depuración de ROS

Abra una nueva terminal, ingrese la ruta `/V2-ROS-SDK-service` y ejecute `source devel/setup.bash`, luego comience a depurar.

a. Suscríbete a mensajes

Ejemplo: ocupar el espacio de almacenamiento actual de la cámara

Después de ingresar a la depuración, ejecute `rostopic echo /lx_localization_node/LxCamera_Capacity_Status` en la terminal.

Recibir el siguiente mensaje significa que todo el enlace de comunicación es normal. Al mismo tiempo, este mensaje también se puede utilizar como un paquete de latidos con una frecuencia de 2 Hz.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8092fa65-76da-46ef-8756-b0e37d673e76.avif" alt="Iconos relacionados con la depuración" />
</Frame>

2\. Llame a la interfaz de servicio

Ejemplo: ver el nombre del mapa seleccionado actualmente

Ejecute `rosservice call /lx_localization_node/LxCamera_GetCurMapname_srv "get_cur_mapname: true"` en la terminal.

Recibir el siguiente mensaje significa que todo el enlace de comunicación es normal y el mensaje devuelto es el nombre del mapa actual.

## Proceso de operación básico

Encienda la cámara, espere unos 30 segundos y luego inicie el nodo de reenvío de datos para comenzar a usarla.

Ingrese a la terminal de depuración

***1. Emitir instrucciones de construcción de mapas***

llamada de rosservice /lx\_localization\_node/LxCamera\_Mapping\_srv "estado: verdadero nombre del mapa: 'prueba\_map'"

Este comando se utiliza para iniciar una tarea de mapeo. Se preparará para crear un mapa con el nombre "test\_map". Si se emite correctamente, devolverá verdadero. En este momento, puedes tirar del coche para escanear el mapa.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/8a77a9c8-f9df-4c71-a4c9-a4e9d66d6892.avif" alt="Ilustraciones relacionadas con procedimientos operativos básicos." />
</Frame>

***2. Emita el comando para finalizar el mapeo***

llamada de rosservice /lx\_localization\_node/LxCamera\_Mapping\_srv "estado: nombre de mapa falso: 'prueba\_map'"

Este comando se utiliza para finalizar una tarea de mapeo. Volverá verdadero después de que la entrega sea exitosa. En este momento, puede recibir el valor de retorno después de esperar a que se complete el mapeo.

<Frame>
  <img src="/images/legacy/c450a63a-7b1c-4291-9e40-4cb352361b6e/9c93beee-a975-4c4a-93d0-31c500a676f4.avif" alt="Ilustraciones relacionadas con los procedimientos operativos básicos 2" />

***3. Una vez completada la construcción del mapa, espere un momento y cambie al nombre del nuevo mapa***

llamada de rosservice /lx\_localization\_node/LxCamera\_SwitchMap\_srv “mapa\_nombre:“prueba\_mapa'”

***4. Después de cortar la imagen, envíe el mensaje de reubicación***

llamada de rosservice /lx\_localization\_node/LxCamera\_UploadReloc\_srv “pose:

Entre ellos, x, y, z, w en orientación son la representación cuaternión del ángulo. Preste atención al formato.

***5. Obtenga la ubicación de posicionamiento actual en tiempo real***

eco rostopico /lx\_localization\_node/LxCamera\_LocationPoseResult

### Preguntas frecuentes y soluciones

***1\.ROS*** El programa no se puede compilar

\-Verifique el entorno de la biblioteca dependiente para confirmar si todas las bibliotecas dependientes están completamente instaladas

***2.ROS SDK*** No hay respuesta después de enviar el mensaje

\-Confirme que la cámara y la máquina estén en el mismo segmento de red, o si puede usar la computadora host de la cámara para abrir la cámara y ver imágenes.

\ -Confirme que no hay ningún problema con el archivo de configuración de la cámara y confirme que no hay ningún problema con el archivo de configuración de ROS.

***3.ROS*** No se puede enviar el mensaje de depuración

Los mensajes \-ROS tienen requisitos de formato estrictos. Al depurar, siga completamente el formato del documento de instrucciones y solo modifique el valor.

Cambiar la depuración.

## 1. Descripción general

La evitación visual de obstáculos se basa en las cámaras de la serie S y las cámaras RGBD de desarrollo propio de Meyer Micro Vision basadas en el principio TOF. Integra aprendizaje profundo y algoritmos tradicionales de visión 3D para crear un sistema para evitar obstáculos que es adecuado para escenas ricas, preciso, en tiempo real, configurable por una sola máquina, colaborativo entre varias máquinas y tiene un cierto grado de iniciativa y previsibilidad.

El sistema tiene las siguientes características: detección dinámica en tiempo real, detección terrestre adaptativa, seguimiento dinámico de objetivos de detección, admite múltiples salidas de señal, admite conmutación de múltiples modos, puede admitir 5 modos y admite trabajo colaborativo con varias máquinas.

## 7. Método de comunicación

1. Antes de comprender o utilizar varios métodos de comunicación, **debe usar la computadora host que proporcionamos para configurar el contenido de los parámetros para evitar obstáculos uno por uno** antes de poder usar **métodos de comunicación que no sean API** para cambiar diferentes parámetros para evitar obstáculos. **Nota:**

**① Algunos protocolos de comunicación son exclusivos de un determinado modelo, como la comunicación RS485 es exclusiva de M4Pro; **

② Cuando se utiliza, entre otros, el método API para obtener resultados para evitar obstáculos, se recomienda confirmar primero si la configuración para evitar obstáculos se cambió correctamente, es decir, primero consultar el número del área de configuración para evitar obstáculos actual;

③ Utilice los métodos de comunicación para evitar obstáculos UDP y CAN para verificar el área para evitar obstáculos. Para obtener más información, consulte la introducción detallada de cada método de comunicación.

| --- | --- |
| T2 | API、UDP、IO |

| S2Max V2.0 | API、UDP、IO、CAN |

| --- | --- | --- |
| T2 | Todos los métodos de comunicación se pueden modificar | Solo se puede modificar utilizando la computadora host que proporcionamos |
| S2 Max, S2 Max2.0, M4, M4Pro, M4 Mega | Todos los métodos de comunicación se pueden modificar | Solo se pueden modificar la computadora host y la API |

**Nota: el índice de parámetros para evitar obstáculos** se refiere al índice del grupo de parámetros de configuración 0, 1, 2, etc.; **Parámetros detallados para evitar obstáculos** se refiere a todos los parámetros excepto los parámetros para evitar obstáculos, como el ángulo de la cámara, la distancia de advertencia, etc.

| Prioridad | Método de comunicación | Objeto |
| --- | --- | --- |
| 1 | comunicación IO | Los parámetros entran en vigor inmediatamente y no pueden ser alterados por otros medios. Si se dejan flotantes o no admiten comunicación IO, se utilizarán los parámetros predeterminados o los parámetros temporales. |
| 2 | Configuración de API y computadora host | Los parámetros predeterminados están configurados y no se ven afectados por cortes de energía. |
| 2 | Otros (como 485, etc.) | Las configuraciones son parámetros temporales. Este parámetro se modificará si hay un corte de energía u otras comunicaciones. Se puede guardar como parámetros predeterminados mediante instrucciones, |

3. Para la modificación de los parámetros de evitación de obstáculos, la cámara seguirá estrictamente la lógica de prioridad.

4. La mayor diferencia entre los parámetros temporales y los parámetros predeterminados es si se guardan después del apagado. El parámetro que se utiliza depende del método de comunicación que se utilizó la última vez.

5. Durante la comunicación IO (IO no está suspendida), los parámetros actuales no se pueden modificar utilizando el método de comunicación de prioridad 2, pero se pueden modificar los parámetros predeterminados y los parámetros temporales. La última vez que se utilizan las configuraciones 485 o UDP, se utilizan parámetros temporales. 6. Para la cámara S10, se necesitan entre 200 y 300 ms para cambiar los parámetros hasta que se complete la carga. Por lo tanto, S2 realiza una operación de filtrado en la operación del parámetro de conmutación y se ignorará la última configuración dentro de 250 ms. Ya sea S10 o S2Max/S10pro, no se recomienda cambiar los parámetros con demasiada frecuencia, ya que esto aumentará el uso de recursos de la cámara y reducirá la velocidad de fotogramas de la imagen. Debido a que la frecuencia de actualización de los resultados de evasión de obstáculos está relacionada con la velocidad de fotogramas, también hará que se reduzca la frecuencia de actualización de los resultados de evasión de obstáculos. La frecuencia de actualización del resultado para evitar obstáculos es igual a la frecuencia de cuadros de la imagen.

### 7.5 Método de comunicación CAN

1. La comunicación CAN solo admite cámaras S2-MAX. El protocolo de comunicación utiliza CanOpen. El protocolo CanOpen es un protocolo de capa de aplicación que se ejecuta en el bus CAN estándar. Su modo de comunicación es también el modo de comunicación "maestro-esclavo" comúnmente utilizado en los protocolos de comunicación industriales. Es decir, habrá una estación maestra + múltiples estaciones esclavas en la red. Las estaciones esclavas no se comunicarán directamente entre sí. Todas las comunicaciones se realizan entre la estación maestra y las estaciones esclavas. La estación maestra también se denomina "cliente" y la estación esclava también se denomina "servidor".

2. El método de comunicación subyacente utiliza el formato de trama estándar CAN, es decir, el ID CAN es de 11 bits (0x000 ~ 7FF) y los datos son 8 bytes.

3. La convención de uso de CAN ID: divida la ID de 11 bits en un código de función de 4 bits y una ID de nodo de 7 bits, y la ID de CAN también se denomina ID de COB:

## 8. Preguntas frecuentes

### 8.1 Objetos negros

Debido al principio TOF, las cámaras de la serie S se ven afectadas por la reflectividad de la paleta negra. Con una reflectividad del 5 % de la paleta negra, la cámara S2max se puede utilizar para lograr imágenes dentro de un rango de 2 m. Durante la depuración, el efecto de imagen de la paleta negra también se puede optimizar mediante el alto tiempo de integración y el bajo umbral de señal del software del ordenador host. Si hay un brazo de horquilla negro en la escena para evitar obstáculos, se recomienda que el usuario agregue una mayor reflectividad a la punta de la horquilla para ayudar a evitar obstáculos, como rociar una pequeña sección delante de la punta de la horquilla con una superficie de pintura de otros colores.

### 8.2 Objetos de terreno bajo

Los objetos en el suelo no pueden evitar los obstáculos. En aplicaciones para evitar obstáculos, generalmente utilizamos el suelo como base para la configuración de parámetros externos. Si no se pueden evitar obstáculos al encontrar obstáculos bajos en el suelo, puede consultar las siguientes soluciones:

- La imagen de la nube de puntos del ordenador host muestra que no se ha seleccionado ningún obstáculo. Debe confirmar si la altura para evitar obstáculos (abajo) es un valor positivo. Si es un valor positivo, se recomienda configurarlo temporalmente en 0 para determinar si se puede detectar el obstáculo. Si aún no se puede detectar, se recomienda ajustar la altura de instalación de la cámara.

- La imagen de la nube de puntos de la computadora host puede seleccionar el objeto obstáculo pero el resultado para evitar obstáculos es 0. Este fenómeno puede deberse a una configuración incorrecta de la altura de instalación de la cámara. Se recomienda que el usuario ajuste la altura de instalación de la cámara.

### 8.3 Los parámetros para evitar obstáculos no tienen efecto

- No hice clic en Configuración completa al configurar los parámetros.

- La modificación de parámetros ilegales provocó que la configuración fallara.

- La conexión de la cámara es inestable y está desconectada, lo que provoca que falle la entrega de parámetros.

### 8.4 Interferencia en la estructura de instalación

- Durante la instalación, hay estructuras alrededor de la cámara que bloquean el campo de visión de la cámara, lo que genera datos anormales en la cámara.

- Durante el diseño estructural, el diseño de la cámara interfiere con la vista trasera, provocando datos anormales en la cámara.

- La película protectora del espejo de la cámara no se ha quitado, lo que genera datos anormales en la cámara.

- Hay otro cristal protector instalado en el espejo de la cámara, lo que provoca anomalías en los datos.

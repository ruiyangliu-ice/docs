## 1. **Descripción general**

El SDK de cámara MRDVS para Linux incluye ejemplos de C, servidor HTTP Java, ROS y ROS 2. Los ejemplos demuestran los flujos de trabajo principales del SDK y las características básicas. Adapte la implementación a su aplicación. Esta guía explica cómo configurar el entorno requerido y ejecutar cada ejemplo, con una cobertura detallada de ROS y ROS 2.

Se recomienda leer las Partes 1 a 3 en orden de prioridad. Después de leer las Partes 1 a 3 en secuencia, puede saltar directamente a las partes relevantes para satisfacer diferentes necesidades. Cada capítulo es independiente entre sí y no afecta la lectura.

Finalmente, este artículo proporciona algunos métodos de instalación y precauciones para aplicaciones y entornos relacionados. Puede consultar el apéndice si es necesario.

#### 3.2.1 Código de muestra de C y C++

Ingrese al directorio C. El contenido del directorio es el siguiente:

<Frame>
  <img src="/images/legacy/e3b253f3-6743-4945-b738-b8e9b8dfc7dc/9a255d61-eadf-4076-847d-65b29fe4eace.avif" alt="Ilustraciones relacionadas con código de muestra C y C++" />
</Frame>

Contiene 9 carpetas y 1 archivo Cmakelist, cada carpeta contiene un archivo cpp

Las funciones implementadas aproximadamente de cada cpp son las siguientes:

- aplicación\_obstacle: obtiene datos de transmisión de la cámara y habilita el algoritmo para evitar obstáculos, genera resultados del algoritmo

- application\_pallet: obtiene datos de transmisión de la cámara y habilita el algoritmo de paleta, genera resultados del algoritmo

- arm\_local\_camera: llame al SDK en la cámara [ARM] y obtenga los datos de la transmisión

- frame\_callback: utiliza el método de devolución de llamada para obtener datos de la transmisión de la cámara

- multi\_cameras: abre varias cámaras y obtiene datos de transmisión

- single\_camera: abre una sola cámara y obtiene datos de transmisión

- single\_camera2: obtiene datos del cuadro a través de la estructura

Se pueden seleccionar diferentes códigos de muestra para probar y modificar según las diferentes necesidades.

### 5.3 ROS1 y ROS2

En la descripción de ros, no hablaremos sobre el código fuente del controlador, sino sobre el contenido correspondiente a la configuración en el archivo de inicio, cómo ajustar la configuración y cómo enviar y suscribirse a mensajes. El nodo ros proporcionado ya contiene la mayoría de las funciones, por lo que no se recomienda modificar directamente el código fuente del controlador. Si tiene necesidades especiales, primero puede comprender el flujo de código en el ejemplo de C++ y luego modificar el código fuente del controlador.

**Al ejecutar el controlador ros, debe asegurarse de que las versiones de opencv y pcl en la ruta predeterminada del sistema sean únicas; de lo contrario, se producirá un error al vincular la biblioteca ros, lo que puede causar fallas en la compilación o fallas en el tiempo de ejecución.**

El entorno de cada computadora es diferente y los problemas encontrados pueden no ser consistentes. Si encuentra problemas que no se muestran en este apéndice, busque soluciones usted mismo.

### 1. Método de instalación de la versión especificada de Cmake

Vaya a https://cmake.org/download/ y descargue la versión de CMake requerida. Extraiga el paquete descargado con `tar -zxvf cmake-XXX.tar.gz`. Ingrese al directorio extraído, luego compílelo y configúrelo:

./bootstrap

hacer

sudo hacer instalar

Verifique la versión de Cmake para ver si la instalación se realizó correctamente:

cmake --versión

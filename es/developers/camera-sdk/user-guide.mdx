## 1. **Descripción general**

Este documento está destinado principalmente a desarrolladores relacionados con el acceso a dispositivos de cámara (requiere capacidades de desarrollo de software). Actualmente es compatible con productos de la serie M4/M4 Mega/M4 Pro/V1Pro/S2/S2/H3. El objetivo principal de este documento es explicar cómo utilizar la API proporcionada por el SDK. Camera SDK admite la ejecución en sistemas operativos Windows basados ​​en x86 y en sistemas operativos Linux basados ​​en X86 y Arm.

Camera SDK proporciona una interfaz de configuración de parámetros y control de acceso unificado, que admite la adquisición sincrónica o asincrónica de datos del dispositivo (incluidas imágenes de profundidad, imágenes de intensidad, imágenes de nubes de puntos e imágenes RGB). Algunos modelos de cámaras admiten la evitación de obstáculos, el acoplamiento, el posicionamiento y otros datos.

Este documento proporciona asistencia para el desarrollo en la mayor medida posible y el contenido proporcionado puede ser defectuoso o incompleto.

##### 2.2.3.2 Instalación del sistema Linux:

1. Descomprima y ejecute el script de instalación.

- tar xvf Lanxin-MRDVS-xxx.tar.gz

- cd Lanxin-MRDVS

2. Debido a que la instalación requiere permisos de root, si el usuario actual no es root, debe cambiar al usuario root.

- sudo su raíz

3. Luego realice la instalación.

- chmod +x instalar.sh

- ./install.sh

El script de instalación instalará el SDK en el directorio /opt/Lanxin-MRDVS y configurará las variables de entorno. Cabe señalar que las variables de entorno establecidas en el script son las variables de entorno del usuario actual. Si el programa en ejecución no es el usuario root, debe volver a ejecutar el script install.sh para configurar las variables de entorno o agregar manualmente export LD\_LIBRARY\_PATH=/opt/Lanxin-MRDVS/lib/:$LD\_LIBRARY\_PATH al final de ~/.bashrc del usuario actual.

Para las sesiones de terminal que se abrieron antes de la instalación, debe ejecutar source ~/.bashrc para actualizar la variable de entorno LD\_LIBRARY\_PATH antes de ejecutar el programa.

## 4. **Notas de llamadas del SDK de la cámara**

### 4.1 Principios generales de la llamada a la interfaz Camera SDK

1. Para tipos de enumeración con el prefijo LX\_INT, utilice DcSetIntValue o DcGetIntValue para establecer u obtener parámetros de tipo INT;

2. Para los tipos de enumeración con el prefijo LX\_FLOAT, utilice DcSetFloatValue o DcGetFloatValue para establecer u obtener parámetros de tipo FLOAT;

3. Para tipos de enumeración con el prefijo LX\_BOOL, utilice DcSetBoolValue o DcGetBoolValue para establecer u obtener parámetros de tipo BOOL;

4. Para tipos de enumeración con el prefijo LX\_STRING, utilice DcSetStringValue o DcGetStringValue para establecer u obtener parámetros de tipo de cadena;

5. Utilice DcSetCmd para ejecutar instrucciones para tipos de enumeración con el prefijo LX\_CMD;

6. Para tipos de enumeración con el prefijo LX\_PTR, utilice DcGetPtrValue para obtener resultados de tipo puntero;

### 4.2 Condiciones previas de llamada de la parte de la interfaz del SDK de la cámara

**1. Cuando se inicia el flujo o el modo de trabajo (LX\_INT\_WORK\_MODE) es normalmente abierto, no se permite configurar las siguientes funciones:**

-DcSetRoi

- LX\_INT\_3D\_BINNING\_MODE;

- LX\_INT\_2D\_BINNING\_MODE;

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH;

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR;

- LX\_BOOL\_ENABLE\_MULTI\_MACHINE;

- LX\_INT\_TRIGGER\_MODE;

- LX\_INT\_CALCULATE\_UP;

- LX\_BOOL\_ENABLE\_HDR;

- LX\_BOOL\_ENABLE\_TOF\_DEPTH\_CALIBE;

- LX\_BOOL\_ENABLE\_SYNC\_FRAME

- LX\_INT\_SAVE\_PARAMS\_GROUP

**2. Las configuraciones no están permitidas incluso cuando el modo de trabajo (LX\_INT\_WORK\_MODE) está normalmente abierto:**

- LX\_INT\_TOF\_FREQMODE<br />

**3. Cuando la exposición automática 3D (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) está activada, no se permiten las siguientes operaciones de enumeración de funciones:**

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR ;

- LX\_INT\_FIRST\_EXPOSICIÓN;

- LX\_INT\_SECOND\_EXPOSICIÓN;

- LX\_INT\_TERCERA\_EXPOSICIÓN;

- LX\_INT\_GAIN;<br />

**4.Cuando la exposición automática 3D (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) está desactivada, no se permite operar la siguiente enumeración de funciones:**

- LX\_INT\_3D\_AUTO\_EXPOSICIÓN\_LEVEL

No se permite configurar el valor máximo de exposición automática 3D (LX\_INT\_3D\_AUTO\_EXPOSURE\_MAX) y el valor mínimo

- (LX\_INT\_3D\_AUTO\_EXPOSICIÓN\_MIN)

**5. Cuando la exposición automática 2D (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) está activada, no se permiten las siguientes operaciones de enumeración de funciones:**

- LX\_INT\_2D\_MANUAL\_EXPOSICIÓN

<br />**6. Cuando la exposición automática 2D (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) está desactivada, no se permite operar la siguiente enumeración de funciones:**

- LX\_INT\_2D\_AUTO\_EXPOSURE\_LEVEL;

<br />7.**En el modo de filtrado extraordinario, no se permiten las siguientes configuraciones:**

- LX\_INT\_FILTER\_SMOOTH\_LEVEL LX\_INT\_FILTER\_RUIDO\_LEVEL

- LX\_INT\_FILTER\_TIME\_LEVEL

<br />8.**Al activar la antidistorsión 2D, primero se debe activar la alineación RGBD:**

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH

<br />9.**Para dispositivos que no sean de tipo S2, al activar el algoritmo integrado, primero debe configurar el algoritmo tof. Solo se permite configurar los parámetros del algoritmo cuando el algoritmo integrado (LX\_INT\_ALGORITHM\_MODE) está activado.**

- (LX\_STRING\_ALGORITHM\_PARAMS)

<br />10.**Para obtener datos de la nube de puntos (LX\_PTR\_XYZ\_DATA), primero debe abrir el mapa de profundidad**

- (LX\_BOOL\_ENABLE\_3D\_DEPTH\_STREAM)

## Preguntas frecuentes

<AccordionGroup>
<Accordion title="LX_ERROR = -1">

**Posibles razones**

error desconocido

**Resolución**

El resultado es anormal y es necesario extraer el registro del dispositivo y enviarlo a MRDVS para su procesamiento.

</Accordion>

<Accordion title="LX_E_NOT_SUPPORT = -2">

**Posibles razones**

Función no compatible

**Resolución**

El dispositivo actual no admite esta función

</Accordion>

<Accordion title="LX_E_NETWORK_ERROR = -3">

**Posibles razones**

Error de comunicación de red

**Resolución**

Falla de comunicación de la red, pérdida de paquetes de transmisión de señales y otros problemas, verifique la configuración de IP, verifique si los cables y conmutadores de la red son normales y solucione problemas de pérdida de paquetes

</Accordion>

<Accordion title="LX_E_INPUT_ILLEGAL = -4">

**Posibles razones**

Es ilegal tomar ginseng

**Resolución**

Los parámetros pasados ​​son incorrectos. Debe verificar los parámetros de la interfaz de llamada.

</Accordion>

<Accordion title="LX_E_RECONECTANDO = -5">

**Posibles razones**

Reconexión del dispositivo

**Resolución**

1. El dispositivo se está reconectando, espere un breve retraso e inténtelo nuevamente hasta que la reconexión sea exitosa<br />2. Compruebe si la cámara está encendida normalmente y si la conexión de red es normal.

</Accordion>

<Accordion title="LX_E_DEVICE_ERROR = -6">

**Posibles razones**

Fallo del dispositivo o fallo de respuesta del dispositivo

**Resolución**

Anomalía en el dispositivo, reinicie el programa o la cámara y vuelva a intentarlo

</Accordion>

<Accordion title="LX_E_DEVICE_NEED_UPDATE = -7">

**Posibles razones**

La versión del dispositivo es demasiado baja y es necesario actualizarla.

**Resolución**

Actualizar la versión del firmware

</Accordion>

<Accordion title="LX_E_API_NEED_UPDATE = -8">

**Posibles razones**

La versión de API es demasiado baja

**Resolución**

Actualizar SDK

</Accordion>

<Accordion title="LX_E_CTRL_PERMISS_ERROR = -9">

**Posibles razones**

Error en el permiso de control exclusivo

**Resolución**

La cámara solo admite una conexión de cliente de autoridad principal. Cuando la cámara se cierra de manera anormal, debe esperar de 3 a 5 segundos para liberar los permisos.

</Accordion>

<Accordion title="LX_E_GET_DEVICEINFO_ERROR = -10">

**Posibles razones**

No se pudo obtener información del dispositivo

**Resolución**

Confirme la versión del firmware del dispositivo o comuníquese con el soporte de MRDVS

</Accordion>

<Accordion title="LX_E_IMAGE_SIZE_ERROR = -11">

**Posibles razones**

El tamaño de la imagen no coincide y es necesario volver a abrirla.

**Resolución**

El tamaño de la imagen obtenida no coincide con el tamaño del flujo de código real. Debe cerrar el dispositivo y volver a abrirlo.

</Accordion>

<Accordion title="LX_E_IMAGE_PARTITION_ERROR = -12">

**Posibles razones**

Error al analizar la imagen (el formato de píxeles es incorrecto), debe intentar abrirla nuevamente

**Resolución**

Vuelva a encender el dispositivo

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_CONNECTED = -13">

**Posibles razones**

Cámara no conectada

**Resolución**

Es un tipo ilegal de entrada de parámetros. Si utiliza la cámara cuando no está encendida, se devolverá este error. Debe verificar la lógica del código de llamada para asegurarse de que la cámara se haya encendido correctamente.

</Accordion>

<Accordion title="LX_E_DEVICE_INIT_FAILED = -14">

**Posibles razones**

Falló la inicialización de la cámara

**Resolución**

Vuelve a encender la cámara

</Accordion>

<Accordion title="LX_E_DEVICE_NOT_FOUND = -15">

**Posibles razones**

Cámara no encontrada (o no se encontró ninguna cámara coincidente)

**Resolución**

Un tipo de parámetros de entrada ilegales. Este código de error generalmente se devuelve cuando el identificador de la cámara que se pasa a la interfaz es ilegal. Debe verificar la lógica del código de llamada.

</Accordion>

<Accordion title="LX_E_FILE_INVALID = -16">

**Posibles razones**

Error de archivo (nombre de archivo, tipo o formato incorrecto o error al abrir el archivo)

**Resolución**

Si la ruta/tipo del archivo es incorrecto o el archivo no existe o no se puede abrir, debe verificar si la ruta del archivo de parámetros entrante se puede abrir manualmente.

</Accordion>

<Accordion title="LX_E_CRC_CHECK_FAILED = -17">

**Posibles razones**

Error en la comprobación del archivo crc o md5

**Resolución**

La verificación de crc o md5 falló al cargar o descargar archivos, debe intentarlo nuevamente

</Accordion>

<Accordion title="LX_E_TIME_OUT = -18">

**Posibles razones**

se acabó el tiempo

**Resolución**

Intentar otra vez. Puede deberse a motivos de red por lo que hay una gran pérdida de paquetes o la velocidad de fotogramas de la cámara es demasiado baja.

</Accordion>

<Accordion title="LX_E_FRAME_LOSS = -19">

**Posibles razones**

Marcos faltantes

**Resolución**

La pérdida de paquetes y de tramas suele ocurrir en el modo de devolución de llamada. En un pequeño número de casos, no se requiere ningún procesamiento especial; en una gran cantidad de casos, es necesario verificar el ancho de banda de la red, el caché, etc.

</Accordion>

<Accordion title="LX_E_ENABLE_ANYSTREAM_FAILED = -20">

**Posibles razones**

No se pudo abrir ninguna transmisión

**Resolución**

Antes de la versión 2.4.9, no estaba permitido cerrar todos los datos de transmisión. Este código de error se ha eliminado en versiones posteriores.

</Accordion>

<Accordion title="LX_E_NOT_RECEIVE_STREAM = -21">

**Posibles razones**

Datos de transmisión no recibidos

**Resolución**

No se recibieron datos de secuencia de código, la cámara de referencia está abierta y no hay datos.

</Accordion>

<Accordion title="LX_E_PARSE_STREAM_FAILED = -22">

**Posibles razones**

La transmisión se inició correctamente pero no se pudieron analizar los datos de la transmisión.

**Resolución**

No se pudo analizar el flujo de código. Confirme la versión del firmware o comuníquese con el soporte de MRDVS.

</Accordion>

<Accordion title="LX_E_PROCESS_IMAGE_FAILED = -23">

**Posibles razones**

Error al procesar el cálculo de la imagen

**Resolución**

Confirmar la versión de la biblioteca de algoritmos de imágenes "Dataprocess"

</Accordion>

<Accordion title="LX_E_SETTING_NOT_ALLOWED = -24">

**Posibles razones**

No se permite la configuración en modo normalmente abierto

**Resolución**

En el modo de trabajo, normalmente abierto, no se permiten operaciones como configurar ROI o configurar múltiples cámaras/alineación. Es necesario ajustar la lógica de llamada. Primero configure el modo de funcionamiento de la cámara y luego realice la operación de configuración.

</Accordion>

<Accordion title="LX_E_LOAD_DATAPROCESSLIB_ERROR = -25">

**Posibles razones**

Error al cargar la biblioteca de algoritmos de procesamiento de imágenes

**Resolución**

Asegúrese de que la biblioteca de algoritmos de imágenes Dataprocess (biblioteca de vínculos dinámicos/archivo de objetos compartidos) exista y esté incluida en las variables de entorno cargadas por la biblioteca. Ejecutar: fuente install.sh

</Accordion>

<Accordion title="LX_E_FUNCTION_CALL_LOGIC_ERROR = -26">

**Posibles razones**

Error de lógica de llamada de función

**Resolución**

Si la secuencia de llamada de función o la lógica previa son incorrectas, debe informar un error de acuerdo con el registro del SDK y ajustar la lógica de llamada. Los problemas lógicos comunes son los siguientes:<br />1. La configuración de parámetros hará que el tamaño de la imagen cambie, por lo que primero es necesario detener el flujo;<br />2. En el modo normalmente activado, no se permite cambiar el tamaño de la imagen;<br />3. Mover el algoritmo hacia arriba es mutuamente excluyente con activar el algoritmo de aplicación integrado;<br />4. No se permite activar la exposición automática cuando HDR está habilitado;<br />5. Los parámetros relacionados con la exposición automática están relacionados con el estado de explosión de luz automática;<br /> />6. Los parámetros relacionados con el disparador están relacionados con el modo de disparo.

</Accordion>

<Accordion title="LX_E_IPAPPDR_UNREACHABLE_ERROR = -27">

**Posibles razones**

IP inalcanzable o error de configuración de red

**Resolución**

La red no está disponible. Debe asegurarse de que el host del SDK pueda hacer ping al dispositivo visual.

</Accordion>

<Accordion title="LX_E_FRAME_ID_NOT_MATCH = -28">

**Posibles razones**

Error de cuadro fuera de sincronización dentro del rango de tiempo de espera

**Resolución**

Cuando la función de sincronización de cuadros RGBD está activada, este error se informa cuando los cuadros de datos tof y rgb no están sincronizados en el flujo de código real. Puede optar por descartar los datos del marco o ignorar el error según el escenario de uso real (alta prioridad para sincronización o alta prioridad para datos en tiempo real).

</Accordion>

<Accordion title="LX_E_FRAME_MULTI_MACHINE = -29">

**Posibles razones**

Señal de interferencia de varias máquinas detectada en el marco

**Resolución**

Dependiendo del escenario de uso real, elija descartar los datos del marco o ignorar el informe de errores.

</Accordion>

<Accordion title="código de error: -25 o error de carga de LxDataProcess">

**Posibles razones**

(Aplicable únicamente al entorno del sistema operativo Linux) La biblioteca de algoritmos no está configurada para variables de entorno

**Resolución**

Ejecutar fuente install.sh

</Accordion>

<Accordion title="Tome tres tercios de la imagen transmitida">

**Posibles razones**

Después de adquirir la imagen, la relación de aspecto mostrada es anormal.

</Accordion>

<Accordion title="ros2 entra en conflicto con la biblioteca SDK">

**Resolución**

El SDK de la cámara está encapsulado en la carpeta util y se vincula indirectamente a la biblioteca del SDK de la cámara.

</Accordion>

<Accordion title="Cómo determinar si la cámara es una versión Lite o una versión estándar con RGB a nivel de software">

**Resolución**

DcGetBoolValue(handle, LX_BOOL_ENABLE_2D_STREAM, &test_rgb) Si la cámara es una versión Lite, devolverá no compatible

</Accordion>

<Accordion title="Método de sincronización horaria">

**Resolución**

SDK y PTP están sincronizados. Cada llamada a la interfaz se sincroniza automáticamente sin activación adicional.

</Accordion>

</AccordionGroup>

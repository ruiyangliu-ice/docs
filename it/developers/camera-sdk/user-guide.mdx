## 1. **Panoramica**

Questo documento è destinato principalmente agli sviluppatori relativi all'accesso al dispositivo fotocamera (richiede funzionalità di sviluppo software). Attualmente supporta i prodotti della serie M4/M4 Mega/M4 Pro/V1Pro/S2/S2/H3. Lo scopo principale di questo documento è spiegare come utilizzare l'API fornita dall'SDK. Camera SDK supporta l'esecuzione su sistemi operativi Windows basati su x86 e sistemi operativi Linux basati su X86 e Arm.

Camera SDK fornisce un controllo unificato degli accessi e un'interfaccia di configurazione dei parametri, supportando l'acquisizione sincrona o asincrona dei dati del dispositivo (comprese immagini di profondità, immagini di intensità, immagini di nuvole di punti, immagini RGB). Alcuni modelli di fotocamera supportano l'elusione degli ostacoli, l'aggancio, il posizionamento e altri dati.

Questo documento fornisce la massima assistenza allo sviluppo e il contenuto fornito potrebbe essere difettoso o incompleto.

##### 2.2.3.2 Installazione del sistema Linux:

1. Decomprimere ed eseguire lo script di installazione

- tar xvf Lanxin-MRDVS-xxx.tar.gz

-cd Lanxin-MRDVS

2. Poiché l'installazione richiede i permessi di root, se l'utente corrente non è root, è necessario passare all'utente root

- sudo su root

3. Quindi eseguire l'installazione

- chmod +x install.sh

- ./install.sh

Lo script di installazione installerà l'SDK nella directory /opt/Lanxin-MRDVS e imposterà le variabili di ambiente. Va notato che le variabili d'ambiente impostate nello script sono le variabili d'ambiente dell'utente corrente. Se il programma in esecuzione non è l'utente root, è necessario eseguire nuovamente lo script install.sh per impostare le variabili di ambiente o aggiungere manualmente export LD\_LIBRARY\_PATH=/opt/Lanxin-MRDVS/lib/:$LD\_LIBRARY\_PATH alla fine di ~/.bashrc dell'utente corrente.

Per le sessioni terminale aperte prima dell'installazione, è necessario eseguire source ~/.bashrc per aggiornare la variabile di ambiente LD\_LIBRARY\_PATH prima di eseguire il programma.

## 4. **Note sulle chiamate dell'SDK della fotocamera**

### 4.1 Principi generali di chiamata dell'interfaccia Camera SDK

1. Per i tipi di enumerazione con prefisso LX\_INT, utilizzare DcSetIntValue o DcGetIntValue per impostare o ottenere parametri di tipo INT;

2. Per i tipi di enumerazione con prefisso LX\_FLOAT, utilizzare DcSetFloatValue o DcGetFloatValue per impostare o ottenere i parametri di tipo FLOAT;

3. Per i tipi di enumerazione con prefisso LX\_BOOL, utilizzare DcSetBoolValue o DcGetBoolValue per impostare o ottenere i parametri del tipo BOOL;

4. Per i tipi di enumerazione con prefisso LX\_STRING, utilizzare DcSetStringValue o DcGetStringValue per impostare o ottenere i parametri del tipo stringa;

5. Utilizzare DcSetCmd per eseguire istruzioni per i tipi di enumerazione con prefisso LX\_CMD;

6. Per i tipi di enumerazione con prefisso LX\_PTR, utilizzare DcGetPtrValue per ottenere risultati del tipo puntatore;

### 4.2 Parte dell'interfaccia SDK della fotocamera che richiede precondizioni

**1. Quando il flusso viene avviato o la modalità di lavoro (LX\_INT\_WORK\_MODE) è normalmente la modalità aperta, non è consentito impostare le seguenti funzioni:**

-DcSetRoi

- LX\_INT\_3D\_BINNING\_MODE;

- LX\_INT\_2D\_BINNING\_MODE;

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH;

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR;

- LX\_BOOL\_ENABLE\_MULTI\_MACHINE;

- LX\_INT\_TRIGGER\_MODE;

- LX\_INT\_CALCULATE\_UP;

-LX\_BOOL\_ENABLE\_HDR;

- LX\_BOOL\_ENABLE\_TOF\_DEPTH\_CALIBE;

- LX\_BOOL\_ENABLE\_SYNC\_FRAME

- LX\_INT\_SAVE\_PARAMS\_GROUP

**2. Le impostazioni non sono consentite anche quando la modalità di lavoro (LX\_INT\_WORK\_MODE) è normalmente aperta:**

-LX\_INT\_TOF\_FREQMODE<br />

**3. Quando l'esposizione automatica 3D (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) è attivata, le seguenti operazioni di enumerazione delle funzioni non sono consentite:**

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR ;

- LX\_INT\_PRIMA\_ESPOSIZIONE;

- LX\_INT\_SECONDO\_ESPOSIZIONE;

- LX\_INT\_TERZO\_ESPOSIZIONE;

-LX\_INT\_GAIN;<br />

**4.Quando l'esposizione automatica 3D (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) è disattivata, la seguente enumerazione delle funzioni non è consentita:**

- LX\_INT\_3D\_AUTO\_ESPOSIZIONE\_LEVEL

L'impostazione del valore massimo dell'esposizione automatica 3D (LX\_INT\_3D\_AUTO\_EXPOSURE\_MAX) e del valore minimo non è consentita

- (LX\_INT\_3D\_AUTO\_ESPOSIZIONE\_MIN)

**5. Quando l'esposizione automatica 2D (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) è attivata, le seguenti operazioni di enumerazione delle funzioni non sono consentite:**

- LX\_INT\_2D\_MANUALE\_ESPOSIZIONE

<br />**6. Quando l'esposizione automatica 2D (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) è disattivata, la seguente enumerazione delle funzioni non è consentita:**

- LX\_INT\_2D\_AUTO\_ESPOSIZIONE\_LIVELLO;

<br />7.**In modalità di filtraggio straordinario, le seguenti impostazioni non sono consentite:**

- LX\_INT\_FILTER\_SMOOTH\_LEVEL LX\_INT\_FILTER\_NOISE\_LEVEL

- LX\_INT\_FILTER\_TIME\_LEVEL

<br />8.**Quando si attiva l'anti-distorsione 2D, è necessario attivare prima l'allineamento RGBD:**

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH

<br />9.**Per i dispositivi di tipo non S2, quando si attiva l'algoritmo integrato, è necessario impostare prima l'algoritmo tof. È possibile impostare i parametri dell'algoritmo solo quando l'algoritmo integrato (LX\_INT\_ALGORITHM\_MODE) è attivato.**

- (LX\_STRING\_ALGORITMO\_PARAM)

<br />10.**Per ottenere i dati della nuvola di punti (LX\_PTR\_XYZ\_DATA), è necessario prima aprire la mappa di profondità**

- (LX\_BOOL\_ENABLE\_3D\_DEPTH\_STREAM)

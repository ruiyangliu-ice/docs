## 1. **Обзор**

Этот документ в основном предназначен для разработчиков, связанных с доступом к устройствам камеры (требуются возможности разработки программного обеспечения). В настоящее время поддерживает продукты серий M4/M4 Mega/M4 Pro/V1Pro/S2/S2/H3. Основная цель этого документа — объяснить, как использовать API, предоставляемый SDK. Camera SDK поддерживает работу в операционных системах Windows на базе x86, а также операционных системах Linux на базе X86 и Arm.

Camera SDK предоставляет унифицированный интерфейс управления доступом и конфигурацией параметров, поддерживающий синхронный или асинхронный сбор данных устройства (включая изображения глубины, изображения интенсивности, изображения облаков точек, изображения RGB). Некоторые модели камер поддерживают объезд препятствий, стыковку, позиционирование и другие данные.

Этот документ в наибольшей степени предоставляет помощь в разработке, и предоставленное содержание может быть дефектным или неполным.

##### 2.2.3.2 Установка системы Linux:

1. Разархивируйте и выполните скрипт установки.

- tar xvf Lanxin-MRDVS-xxx.tar.gz

- компакт-диск Lanxin-MRDVS

2. Поскольку для установки требуются права root, если текущий пользователь не является пользователем root, вам необходимо переключиться на пользователя root.

- sudo su корень

3. Затем выполните установку

- chmod +x install.sh

- ./install.sh

Сценарий установки установит SDK в каталог /opt/Lanxin-MRDVS и установит переменные среды. Следует отметить, что переменные среды, установленные в скрипте, являются переменными среды текущего пользователя. Если запущенная программа не является пользователем root, вам необходимо повторно запустить сценарий install.sh, чтобы установить переменные среды, или вручную добавить экспорт LD\_LIBRARY\_PATH=/opt/Lanxin-MRDVS/lib/:$LD\_LIBRARY\_PATH в конце ~/.bashrc текущего пользователя.

Для сеансов терминала, которые были открыты перед установкой, вам необходимо выполнить source ~/.bashrc, чтобы обновить переменную среды LD\_LIBRARY\_PATH перед запуском программы.

## 4. **Примечания к вызову Camera SDK**

### 4.1 Общие принципы вызова интерфейса Camera SDK

1. Для типов перечисления с префиксом LX\_INT используйте DcSetIntValue или DcGetIntValue для установки или получения параметров типа INT;

2. Для типов перечисления с префиксом LX\_FLOAT используйте DcSetFloatValue или DcGetFloatValue, чтобы установить или получить параметры типа FLOAT;

3. Для типов перечисления с префиксом LX\_BOOL используйте DcSetBoolValue или DcGetBoolValue для установки или получения параметров типа BOOL;

4. Для типов перечисления с префиксом LX\_STRING используйте DcSetStringValue или DcGetStringValue для установки или получения параметров строкового типа;

5. Используйте DcSetCmd для выполнения инструкций для типов перечисления с префиксом LX\_CMD;

6. Для типов перечисления с префиксом LX\_PTR используйте DcGetPtrValue для получения результатов типа указателя;

### 4.2 Предварительные условия вызова части интерфейса Camera SDK

**1. Когда поток запущен или рабочий режим (LX\_INT\_WORK\_MODE) является нормально открытым, следующие функции не могут быть установлены:**

- ДкСетРой

- LX\_INT\_3D\_BINNING\_MODE;

- LX\_INT\_2D\_BINNING\_MODE;

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH;

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR;

- LX\_BOOL\_ENABLE\_MULTI\_MACHINE;

- LX\_INT\_TRIGGER\_MODE;

- LX\_INT\_CALCULATE\_UP;

- LX\_BOOL\_ENABLE\_HDR;

- LX\_BOOL\_ENABLE\_TOF\_DEPTH\_CALIBE;

- LX\_BOOL\_ENABLE\_SYNC\_FRAME

- LX\_INT\_SAVE\_PARAMS\_GROUP

**2. Настройки не допускаются, даже если рабочий режим (LX\_INT\_WORK\_MODE) обычно открыт:**

- LX\_INT\_TOF\_FREQMODE<br />

**3. Когда автоматическая 3D-экспозиция (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) включена, следующие операции перечисления функций не допускаются:**

- LX\_BOOL\_ENABLE\_MULTI\_EXPOSURE\_HDR ;

- LX\_INT\_FIRST\_EXPOSURE;

- LX\_INT\_SECOND\_EXPOSURE;

- LX\_INT\_THIRD\_EXPOSURE;

- LX\_INT\_GAIN;<br />

**4.Когда автоматическая 3D-экспозиция (LX\_BOOL\_ENABLE\_3D\_AUTO\_EXPOSURE) отключена, не допускается работа следующего перечисления функций:**

- LX\_INT\_3D\_AUTO\_EXPOSURE\_LEVEL

Установка максимального значения автоматической экспозиции 3D (LX\_INT\_3D\_AUTO\_EXPOSURE\_MAX) и минимального значения не допускается.

- (LX\_INT\_3D\_AUTO\_EXPOSURE\_MIN)

**5. Когда автоматическая 2D-экспозиция (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) включена, следующие операции перечисления функций не допускаются:**

- LX\_INT\_2D\_MANUAL\_EXPOSURE

<br />**6. Когда автоматическая 2D-экспозиция (LX\_BOOL\_ENABLE\_2D\_AUTO\_EXPOSURE) отключена, не допускается работа следующего перечисления функций:**

- LX\_INT\_2D\_AUTO\_EXPOSURE\_LEVEL;

<br />7.**В режиме экстраординарной фильтрации следующие настройки не допускаются:**

- LX\_INT\_FILTER\_SMOOTH\_LEVEL LX\_INT\_FILTER\_NOISE\_LEVEL

- LX\_INT\_FILTER\_TIME\_LEVEL

<br />8.**При включении функции подавления искажений 2D сначала необходимо включить выравнивание RGBD:**

- LX\_BOOL\_ENABLE\_2D\_TO\_DEPTH

<br />9.**Для устройств, отличных от типа S2, при включении встроенного алгоритма необходимо сначала установить алгоритм tof. Параметры алгоритма разрешено устанавливать только при включенном встроенном алгоритме (LX\_INT\_ALGORITHM\_MODE).**

- (LX\_STRING\_АЛГОРИТМ\_ПАРАМС)

<br />10.**Чтобы получить данные облака точек (LX\_PTR\_XYZ\_DATA), необходимо сначала открыть карту глубины**

- (LX\_BOOL\_ENABLE\_3D\_DEPTH\_STREAM)

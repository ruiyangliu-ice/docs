## Предисловие

В этой статье представлена ​​архитектура программного обеспечения модуля визуального позиционирования, метод настройки ROS SDK и базовый процесс картографирования и позиционирования.

</Frame>

Принципиальная схема этого модуля позиционирования показана выше. Среди них встроенная программа позиционирования камеры взаимодействует с ROS SDK главного компьютера с помощью UDP. Кроме того, программа позиционирования поставляется с настраиваемым веб-интерфейсом, который может реализовать визуализацию позиционирования (можно открыть с помощью любого обычного браузера, IP-адрес доступа по умолчанию — 192.168.100.201:9998). Передняя часть выполняет две основные функции:

1. Запустите полную ДЕМО-версию, чтобы завершить процесс картографирования и позиционирования;

2. Настройте целевой идентификатор и ПОРТ для связи с ROS SDK. Таким же образом IP и ПОРТ целевой камеры должны быть настроены до запуска ROS SDK.

После завершения вышеуказанной настройки все функции веб-интерфейса, такие как сопоставление, перемещение и загрузка карт, можно вызывать через ROS SDK.

## Компилировать

### Конфигурация среды выполнения программы

1. Установите КТ

Убедитесь, что QT5 установлен на главном компьютере. Вам нужны Core, Network и другие функциональные пакеты в QT5. Способы установки разных версий Linux могут отличаться. Вот пример установки:

```bash
sudo apt install qtbase5-dev qtbase5-dev-tools libqt5network5
```

2. Установите пакет функций libzip.

Руководство по установке https://blog.csdn.net/iamhuanyu/article/details/129603632

3. Скомпилируйте пакет функций ROS. Запустите `catkin_make` по пути `/V2-ROS-SDK-service` (тот же уровень, что и `/src`).

### Отладка

1\. Метод отладки ROS

Откройте новый терминал, введите путь `/V2-ROS-SDK-service` и запустите `source devel/setup.bash`, затем начните отладку.

а. Подписаться на сообщения

Пример: получение текущего места для хранения данных камеры.

После входа в отладку запустите в терминале `rostopic echo /lx_localization_node/LxCamera_Capacity_Status`.

Получение следующего сообщения означает, что вся линия связи в порядке. В то же время это сообщение также можно использовать как пакет пульса с частотой 2 Гц.

2\. Вызов интерфейса сервиса

Пример: просмотр текущего выбранного названия карты.

Запустите `rosservice call /lx_localization_node/LxCamera_GetCurMapname_srv "get_cur_mapname: true"` в терминале.

Получение следующего сообщения означает, что весь канал связи работает нормально, а возвращаемое сообщение представляет собой текущее имя карты.

## Основной процесс работы

Включите камеру, подождите около 30 секунд, а затем запустите узел пересылки данных, чтобы начать его использовать.

Войдите в терминал отладки

***1. Выдать инструкцию по построению карты***

вызов rosservice /lx\_localization\_node/LxCamera\_Mapping\_srv «статус: true имя_карты: ‘test\_map’»

Эта команда используется для запуска задачи сопоставления. Он будет готовиться к созданию карты с именем «test\_map». Если он успешно выдан, он вернет true. В это время вы можете вытащить машину для сканирования карты.

</Frame>

***2. Введите команду для завершения сопоставления***

вызов rosservice /lx\_localization\_node/LxCamera\_Mapping\_srv «статус: ложное имя карты: ‘test\_map’»

Эта команда используется для завершения задачи сопоставления. Он вернется true после успешной доставки. В это время вы можете получить возвращаемое значение, дождавшись завершения сопоставления.

### Часто задаваемые вопросы и решения

***1\.ROS*** Программа не может быть скомпилирована

\-Проверьте среду зависимых библиотек, чтобы убедиться, что все зависимые библиотеки полностью установлены.

***2.ROS SDK*** Нет ответа после отправки сообщения.

\-Убедитесь, что камера и устройство находятся в одном сегменте сети, или можете ли вы использовать главный компьютер камеры, чтобы открыть камеру для просмотра изображений.

\-Убедитесь, что с файлом конфигурации камеры нет проблем, и убедитесь, что с файлом конфигурации ROS нет проблем.

***3.ROS*** Невозможно отправить отладочное сообщение.

Сообщения \-ROS имеют строгие требования к формату. При отладке полностью следуйте формату инструкции и изменяйте только значение.

Изменить отладку.

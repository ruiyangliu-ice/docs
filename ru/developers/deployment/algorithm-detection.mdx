## 1. Введение

В этой статье в основном рассказывается о развертывании и использовании алгоритмов приложений Meyer Micro Vision.

## 2. Состав системы обнаружения

Система включает в себя камеру, программное обеспечение хост-компьютера и промышленный компьютер.

Схема установки

Функция обнаружения человека обычно устанавливается под наклонным (менее 30°) или горизонтальным углом.

### Подключение камеры

1. Камера питается от сети 24 В и подключается через сетевой кабель категории 6e/категории 7. После установки и развертывания камеры откройте программное обеспечение главного компьютера для определения местоположения склада через ноутбук, чтобы откалибровать внешние параметры камеры и установить диапазон обнаружения. Алгоритм работает внутри камеры в режиме реального времени.

2. При развертывании нескольких камер их необходимо соединить через гигабитный коммутатор. В соответствии с высотой установки, запланированной в CAD, камера 3D Vision устанавливается в фиксированном положении с помощью монтажного кронштейна, чтобы обеспечить надежную фиксацию камеры. После установки камеры подключите ее к регулируемому источнику питания постоянного тока напряжением 24 В и используйте соответствующий сетевой кабель для подключения ее к переключателю или главному элементу управления.

3. После завершения подготовки разместите товар на складе в поле зрения камеры и приступайте к развертыванию программного обеспечения.

### Глобальные функции

Убедитесь, что алгоритм активирован в камере и что камера выходит из строя после настройки параметров через главный компьютер камеры при нормальных условиях использования. Функцию автоматического обновления следует включать в следующих ситуациях.

(1) Если фон соответствующей камеры в списке устройств отображается желтым цветом, это означает, что алгоритм столкнулся с блокировкой в ​​процессе получения изображения.

(2) Если фон соответствующей камеры в списке устройств зеленый, но изображение не обновляется, это также означает, что алгоритм заблокирован в процессе получения изображения. Хотя связь нормальная, система продолжает отправлять кэшированные данные.

|  |  |
| --- | --- |
| Статус | Результаты |
| Нет в наличии | 0 |
| В наличии | 1 |
| В наличии и выровнены (включите проверку выравнивания) | 1 |

### modbus-tcp (определение местоположения библиотеки)

Результаты обнаружения автоматически сортируются по идентификатору местоположения, и будут возвращены только первые 16 результатов местоположения.

#### Параметры конфигурации ведомого устройства

##### Базовая конфигурация связи

1. \****Номер порта***\*: 16502.

2. \****Адрес прослушивания***\*: Все сетевые интерфейсы (0.0.0.0).

3. \****Количество регистров***\*: 16

4. \****Зарегистрируйте начальный адрес***\*: 0x100

5. \****Порядок байтов***\*: режим Big-Endian (Big-Endian)

##### Регистрация правил хранения

|  |  |  |
| --- | --- | --- |
| Адрес регистрации**** | Содержание хранилища**** | Формат данных**** |
| *0x100 + идентификатор* | Результат определения местоположения + результат обнаружения маркера | *2 байта (uint16_t)* |
| *Правила разделения* | | |
| *Байт 0 (старшие 8 бит)* | Код статуса местоположения | *0-2 (см. определение ниже)* |
| *Байт 1 (младшие 8 бит)* | Код состояния маркера (пока не открыт, содержимое соответствует коду состояния местоположения) | *0-2 (см. определение ниже)* |

#### Определение кода состояния

##### Код статуса местоположения

|  |  |  |
| --- | --- | --- |
| Кодирование | Физический смысл | Условия срабатывания |
| *0* | Незанятый | *Нет товаров/поддонов* |
| *1* | Нормальная заполняемость | *Стандартные товары размещены правильно* |
| *2* | Аномальная занятость | *Размер/смещение груза* |

##### Код состояния маркера

|  |  |  |
| --- | --- | --- |
| Кодирование**** | Физический смысл**** | Условие срабатывания**** |
| *0* | Маркер не обнаружен | |
| *1* | Маркер обнаружен | |
| *2* | Обнаружение аномалий | |

- `type`: Семантический тип.
- `valid`: действительный ли результат обнаружения.
